# 73神话 Mac 迁移说明

## 目标

明天切到 Mac 时，优先保证 `73神话` 的主数据完整可用，不在 Windows 端做大清洗。

## 第一批：必须先拷

这些是主数据，优先完整复制：

- `wiki/`
- `raw/`
- `data/`
- `.llm-wiki/`
- `.obsidian/`
- `scripts/`
- `73wiki.config.json`
- `73wiki.system.json`
- `purpose.md`
- `schema.md`
- `extract_l1.py`

## 第二批：建议再拷

这些是运行状态和恢复辅助信息，Mac 上继续接管时有用：

- `.system/ingest-registry.jsonl`
- `.system/codex-raw-watch-queue.jsonl`
- `.system/codex-raw-watch-state.json`
- `.system/raw-queue-consumer-state.json`
- `.system/werss-api-state.json`
- `.system/werss-feeds.json`
- `.system/werss-health.json`
- `.system/werss-private-link-state.json`
- `.system/werss-private-mp-seeds.json`
- `.system/youzi-learning-state.json`
- `.system/youzi-author-refresh-targets.json`
- `.system/postgres-data/`

## 第三批：可后拷

这些不是首日必须，可在 Mac 稳定后再补：

- `.system/logs/`
- `raw/**/*_assets/`
- `.system/postgres-logs/`
- `.system/backups/`

## 不建议现在花时间处理的

- 不要为了迁移临时重构目录
- 不要先删 `raw/` 历史内容
- 不要先删 `.llm-wiki/`
- 不要先删 `.system/postgres-data/`
- 不要先改名旧文件夹

## 最稳的拷贝顺序

1. 先整包复制第一批
2. 再复制第二批
3. 到 Mac 上先验证目录完整
4. 如果空间和时间够，再补第三批

## 到 Mac 后优先检查

- `wiki/` 是否完整
- `raw/` 是否完整
- `.llm-wiki/` 是否完整
- `data/` 是否完整
- `.system/postgres-data/` 是否完整
- `scripts/` 是否完整

## 一句话原则

先把主数据完整带走，再处理日志、图片和平台适配。
