# 发给龙虾和老虎：板块概念分析事实层执行规则

从现在开始，板块概念分析由龙虾和老虎竞争交付。谁能稳定、完整、准时、字段规范，谁做事实层主干。

## 一、分工边界

龙虾和老虎只做事实层，不做最终交易结论。

你们负责：

1. 拉全板块数据。
2. 拉全板块成分股。
3. 拉全涨停、首板、连板、热榜、龙虎榜等事实。
4. 写清每只票归属哪个主板块。
5. 写清板块强弱、板块内部分化、领涨领跌、资金和技术状态。

Codex 负责：

1. 判断主线、支线、轮动、防守、退潮。
2. 给板块强度、持续性、次日扩散概率、风险等级打分。
3. 结合持仓、作战室、首板、连板天梯、热榜、消息催化做交易判断。
4. 做 D+1/D+3/D+5 验证和权重修正。

## 四、个股板块归属统一口径

涨停票、首板票、连板票、热榜票、作战室候选票都必须使用同一套中文字段：

```text
交易题材
主板块
主板块代码
板块来源
备选板块
行业归属
概念标签
板块归属证据
```

主板块选择优先级：

1. 当日涨停原因直接指向的催化板块。
2. 当日市场正在扩散的主线题材。
3. 个股主营最相关的产业链板块。
4. 通达信、同花顺、东方财富中最强且最相关的板块。
5. 不确定就写“未确认”，并写清数据缺口。

禁止只写“机器人、人工智能、国企改革”这种泛标签当主板块，除非当天催化明确就是它。

## 五、板块状态事实字段

每个重点板块必须写：

```text
板块状态：新高 / 趋势上行 / 横盘 / 破位 / 连续杀跌 / 反弹
内部分化：普涨 / 龙头带动 / 龙头独强 / 普跌 / 龙头带崩
资金状态：放量流入 / 缩量轮动 / 放量流出 / 无明显资金
涨停扩散：强扩散 / 弱扩散 / 无扩散 / 退潮
风险点：高位补跌 / 后排大面 / 消息兑现 / 持仓暴露 / 数据缺口
```

这些仍是事实层描述，不是最终买卖结论。

## 六、首板涨停最低标准

首板涨停文件必须做到：

1. 全量首板，不得只写前 30。
2. 主板、创业板 20cm、科创板 20cm、北交所 30cm 都要覆盖。
3. 每只首板必须写中文板块字段。
4. 每只首板必须写催化来源，不能整列写“数据缺口”。
5. 每只首板必须交叉同花顺 Top100 和通达信 Top100；通达信拿不到时，先用同花顺，不允许写“热榜无法拉取”。
6. 每只首板必须写明日验证点。
7. 事件催化不能截断，不能出现半句话。
8. Markdown 可以只展示前 30，但必须清楚写完整 JSON 路径和全量数量。

## 七、明日验证点怎么写

明日验证点分两层。龙虾和老虎只写事实层验证点，Codex 负责判断层。

禁止写：

```text
明日验证点：需 Codex 定性
扩散可能：需 Codex 定性
```

必须写成可拉数据验证的客观条件：

```text
竞价验证：是否一字、是否高开、是否低于预期
封单验证：封单是否增强、是否撤单、开板后是否快速回封
板块验证：同主板块是否继续涨停，涨停家数是否增加，板块排名是否上升
扩散验证：是否出现 20cm/30cm 扩散，是否有补涨首板
连板验证：同主板块是否有 2板以上带队
热榜验证：是否进入通达信热榜；通达信没有该接口时写“通达信接口无此字段”
消息验证：昨日催化是否有新公告、政策、互动易、研报或财联社继续发酵
```

示例：

```text
雷赛智能：
事实层明日验证点：
1. 竞价是否继续一字或高开 5%以上。
2. 人形机器人主板块是否至少 3 只涨停。
3. 是否有创业板 20cm 机器人票跟随。
4. 通达信热榜如可取，是否进入前 30；如接口无字段，写接口无此字段。
5. 优必选/特斯拉机器人消息是否有新公告或主流媒体继续发酵。
6. 若开板，是否快速回封，封单是否强于昨日。
```

Codex 判断层再决定：

```text
是否升权
是否进入作战室
是主线、支线、轮动还是一日游
是否有交易权限
仓位和风险怎么处理
```

## 八、验收评分

总分 100：

| 项目 | 分值 |
|---|---:|
| 全量覆盖 | 20 |
| 中文字段规范 | 20 |
| 板块代码/主板块可对齐 | 15 |
| 催化来源完整 | 15 |
| Top100/热榜交叉验证 | 10 |
| 明日验证点 | 10 |
| 数据缺口诚实且可修复 | 5 |
| 文件路径和 JSON/MD 双格式 | 5 |

判定：

```text
90分以上：可作为事实层主干
75-89分：可用，但需要补交
60-74分：只能参考，不能做主干
60分以下：不合格
```
