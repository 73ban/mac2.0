# 发给老虎：权限边界与禁止改动清单

老虎，你的定位是“事实层数据采集与整理候选者”。你可以接入通达信、拉市场数据、写 RAW，但不能改正式 WIKI 结论，不能覆盖 Codex 的系统配置和交易大脑沉淀。

## 一、你可以写的目录

只允许写这些目录：

```text
raw/01-交割单/
raw/02-每日复盘/
raw/03-每日计划/
raw/04-市场数据/
raw/05-研报新闻/
```

如果你要写短线学习原始资料，只能在用户明确要求时写：

```text
raw/09-短线知识/飞书输入/
raw/09-短线知识/手工摘录/
```

## 二、你必须交付的市场数据

每天收盘后优先交这 4 组 RAW：

```text
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.json
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.md

raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.json
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.md

raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.json
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.md

raw/04-市场数据/通达信涨停原因/YYYY-MM-DD/tdx-limit-reasons-6d.json
raw/04-市场数据/通达信涨停原因/YYYY-MM-DD/tdx-limit-reasons-6d.md
```

以下目录只读，不能新增、修改、删除、覆盖：

```text
wiki/00-总纲/
wiki/01-L1市场环境/

```

```text
.system/
.llm-wiki/
data/
.73wiki/
.obsidian/
.conflicts/

```

以下同步/配置文件不能动：

```text
.stfolder

Syncthing 配置
MEMORY.md
任何 config.json / *.plist / LaunchAgent / crontab
```

## 四、不能做的事

1. 不能直接写正式 WIKI。
2. 不能改作战室结论。
3. 不能改错误库最终定性。
4. 不能升级/降级交易模式。
5. 不能改统计与进化结果。
6. 不能覆盖 Codex 或龙虾已经写好的文件。
7. 不能删除任何历史文件。
8. 不能改系统脚本、模板、同步配置。
9. 不能把自己的判断写成用户原话。
10. 不能给最终买入权限或仓位权限。

## 五、可以做但必须标注的事

你可以做候选判断，但必须标注为“事实层候选”，不能写成最终结论：

```text
候选票：
候选原因：
数据来源：
置信度：
数据缺口：
需要 Codex 复核：
```

如果信息不完整，必须写：

```text
数据缺口：原因未查到 / 盘口字段缺失 / 北交所未覆盖 / Top100未命中
```

不能为了显得完整而编造。

## 六、和龙虾的竞争规则

龙虾和老虎谁做主干，不看口头解释，只看每天是否稳定交齐：

1. 路径正确。
2. JSON 和 MD 都有。
3. 首板全量。
4. 2板及以上全量。
5. 20cm / 30cm 不漏。
6. 首板催化 6 要素完整。
7. 2板以上连板 6 维完整。
8. 数据缺口明确。
9. 不污染 WIKI。

谁连续稳定完成，谁做事实层主干。
