# 发给龙虾和老虎：每日闭环验收与交接规则

## 分工

龙虾/老虎是事实层，只负责拉数据、写 RAW、标注缺口。

Codex 是分析层，负责：

- 从 RAW 提取衍生产物。
- 生成首板催化、涨停原因事实源、连板天梯、热榜和成交额分析。
- 发布交割单和复盘到 WIKI。
- 生成作战室、D+验证、错误账本、统计与进化。
- 把需要外部事实层读取的内容回传到 `raw/11-Codex分析产物/回传外部事实层/日期/`。

## 事实层每日必须交付

1. 交割单
   - `raw/01-交割单/YYYY-MM-DD/交割单.md`
   - 原始委托、持仓表也放同目录。

2. 口述复盘
   - `raw/02-每日复盘/YYYY-MM-DD-口述原文.md`
   - 或 `raw/02-每日复盘/YYYY-MM-DD-复盘.md`

3. 市场数据
   - `raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-daily-limit.json`
   - `raw/04-市场数据/通达信热榜/YYYY-MM-DD/通达信热榜TOP100.json`
   - `raw/04-市场数据/通达信成交额排名/YYYY-MM-DD/tdx-成交额Top100.json`
   - `raw/04-市场数据/龙虎榜全量/YYYY-MM-DD/龙虎榜全量日报.json`
   - `raw/04-市场数据/板块强度/YYYY-MM-DD/tdx-board-strength.json`
   - `raw/04-市场数据/板块成分股/YYYY-MM-DD/tdx-board-members.json`
   - `raw/04-市场数据/板块成分股/YYYY-MM-DD/tdx-stock-boards.json`

## Codex 每日闭环命令

Mac 端运行：

```bash
cd /Users/qixinchaye/wiki/73神话
python3 raw/07-系统脚本/codex_run_daily_closed_loop.py --date YYYY-MM-DD
```

运行后检查：

- `wiki/09-统计与进化/YYYY-MM-DD-每日闭环运行报告.md`
- 报告中必须显示：步骤 OK、必要文件缺失为 0。

## 外部事实层读取位置

外部事实层端只读这个回传目录：

```text
raw/11-Codex分析产物/回传外部事实层/YYYY-MM-DD/
```

如果需要手工复制回传包：

- 把回传包里面的 `raw` 和 `wiki` 两个目录复制到对端 Wiki 根目录。
- 不要复制到对端 `raw` 子目录。
- 如果复制到对端 `raw` 子目录，就会错误生成 `raw/raw`。
- 正常情况下优先使用 Syncthing 自动同步，不手工复制。

里面按原始路径打包，重点看：

- `raw\01-交割单\YYYY-MM-DD\`
- `raw\02-每日复盘\YYYY-MM-DD-复盘.md`
- `raw\04-市场数据\首板涨停催化\YYYY-MM-DD\`
- `raw\04-市场数据\通达信涨停原因\YYYY-MM-DD\`
- `raw\04-市场数据\通达信热榜\YYYY-MM-DD\`
- `raw\04-市场数据\通达信成交额排名\YYYY-MM-DD\`
- `raw\11-Codex分析产物\YYYY-MM-DD\`
- `wiki\09-统计与进化\YYYY-MM-DD-每日闭环运行报告.md`

## 同步方向

- 龙虾/老虎写事实层：外部事实层 -> Mac。
- Codex 写分析层：Mac -> 外部事实层。
- 不要把 `raw/01`、`raw/02`、`raw/04` 改成双向乱写。
- Codex 回传统一走 `raw/11-Codex分析产物`。

## 验收口径

如果外部事实层端看到文件空壳，但 Mac 闭环报告显示完整，优先判断为 Syncthing 未同步完成或接收目录未接受。

不要用“对端没看到”直接判定 Codex 没生成。先看：

1. Mac 闭环报告。
2. Syncthing completion。
3. 外部事实层端是否接受 `raw-09`、`raw-10`、`raw-11` 三个目录。
