# 发给龙虾和老虎：板块四龙事实层数据规则

## 一、你们不需要写四龙结论

先锋龙、资金龙、中军、趋势龙的最终候选由 Mac 端 Codex Pro 生成。

龙虾/老虎只需要把事实数据写全，不能为了像分析报告而消耗 DeepSeek 写判断。

## 二、每天必须交的数据

### 1. 板块强度

路径：

```text
raw/04-市场数据/板块强度/YYYY-MM-DD/
```

必须包含：

- 板块涨幅前10。
- 板块跌幅前10。
- 板块代码。
- 板块名称。
- 涨跌幅。
- 数据时间。
- 数据缺口。

有能力时补充：

- 板块成交额。
- 板块上涨家数。
- 板块下跌家数。
- 板块涨停家数。
- 板块领涨股。

### 2. 每日涨停全景

路径：

```text
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/
```

必须包含每只非ST涨停票：

- 代码。
- 名称。
- 涨跌幅。
- 首次涨停。
- 开板次数。
- 板型。
- 封单金额。
- 最大封单额。
- 涨停成交额。
- 连板天数。
- 几天几板。
- 涨停原因。
- 原因揭秘。
- 资金流向字段。

### 3. 成交额Top100

路径：

```text
raw/04-市场数据/通达信成交额排名/YYYY-MM-DD/
```

必须包含：

- 股票代码。
- 股票名称。
- 成交额。
- 现价。
- 涨跌幅。
- 换手率。
- 流通市值。
- 总市值或自由流通市值。

### 4. 热榜Top100

路径：

```text
raw/04-市场数据/通达信热榜/YYYY-MM-DD/
```

必须包含：

- 排名。
- 股票代码。
- 股票名称。
- 热度分或人气值。
- 涨跌幅。
- 概念标签。
- 上榜原因。
- 原始接口字段。

## 三、为什么要这些字段

Codex 会按这些字段自动生成：

- 先锋龙：看首次涨停、连板天数、板型、封单金额。
- 资金龙：看成交额、涨停成交额、主力净额、换手率。
- 中军：看流通市值、总市值、成交额、行业地位字段。
- 趋势龙：看几天几板、连板天数、趋势结构字段。

## 四、禁止事项

- 不要把板块四龙写成主观结论。
- 不要把缺失字段编出来。
- 不要用英文 key。
- 不要写 ST 股。
- 不要只交 Top50。
- 不要把成交额Top100覆盖成热榜Top100。

## 五、缺口写法

拿不到的字段写“待补”，并在数据缺口写明原因。

例如：

```text
数据缺口：主板块代码需通达信880xxx板块映射；龙虎榜需收盘后补。
```

这比编一个假的结论更有价值。
