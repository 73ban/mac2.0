# 发给龙虾：09:25 竞价弱转强执行规则

## 任务边界

09:25 竞价弱转强由龙虾独立执行。Codex 不参与盘中实时判断，只做盘后验收和 D+验证。

龙虾必须给出可验证证据，不准只写“关注”“强”“弱转强”。

## 输出路径

文件名和字段全部用中文，不要写英文文件名、英文表头、英文变量名。

```text
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.json
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.md
```

## 必填字段

每只候选票必须写：

| 字段 | 要求 |
|---|---|
| 股票代码 | 6位代码 |
| 股票名称 | 中文名称 |
| 昨日弱的原因 | 烂板、炸板、尾盘回封、放量分歧、断板反包、板块退潮中抗跌等，必须具体 |
| 今日转强证据 | 竞价价格、竞价金额、竞价量、板块共振、同身位对比、隔夜催化 |
| 竞价涨幅 | 数字，单位百分比 |
| 竞价金额 | 数字，单位元或万元必须注明 |
| 竞价量占昨日成交额比例 | 必填；没有则写“缺昨日成交额，无法计算” |
| 是否强于板块 | 是/否/待确认，并写比较对象 |
| 是否强于同身位 | 是/否/待确认，并写比较对象 |
| 隔夜催化 | 公告、互动问答、新闻、热榜、龙虎榜、外围事件；没有写“无明确催化” |
| 个股地位 | 龙头/中军/补涨/后排/独立逻辑/套利票 |
| 操作分级 | A可参与/B观察/C冲高卖或回避/D骗炮风险 |
| 买点 | A/B才允许写；C/D写“不买” |
| 止损 | 必须可执行 |
| 骗炮风险 | 必须写；不能空 |
| 飞书提醒等级 | 立即提醒/普通提醒/不提醒 |

## 判断硬规则

1. 高开 5% 不等于弱转强。
2. 低开不一定弱，必须结合昨日分歧、竞价金额、板块共振和个股地位。
3. 没有昨日弱的原因，不允许叫弱转强。
4. 没有今日转强证据，不允许叫弱转强。
5. 板块没有跟随，只能降级为“个股异动”，不能直接给 A。
6. 昨日一字板今天继续一字，不叫弱转强，叫强更强或一致加速。
7. ST 股不纳入。
8. 操作建议必须分级，不能只写“关注”。

## 飞书提醒

09:25 飞书只发最重要的 3-5 只，不要刷屏。

提醒格式：

```text
09:25 竞价弱转强：
1. 股票：名称 代码
2. 分级：A/B/C/D
3. 昨日弱因：
4. 今日转强证据：
5. 风险：
6. 建议：
```

## 盘后验证

Codex 盘后检查：

- 是否开盘后继续走强。
- 是否晋级涨停。
- 是否强于板块。
- 是否强于同身位。
- 是否出现高开低走骗炮。
- 龙虾 09:25 分级是否正确。

验证结果写入：

```text
wiki/09-统计与进化/YYYY-MM-DD-竞价弱转强验收.md
```
