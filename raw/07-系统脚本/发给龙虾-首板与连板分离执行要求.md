# 发给龙虾：每日涨停全景总表与复盘正文分工规则

龙虾，按这个口径执行：每天所有涨停票先放进一个“每日涨停全景总表”，首板和连板都能在一个文件里看到；每日复盘正文只重点展开 2板及以上连板票，首板只写摘要、主题聚类和重点异动。

## 零、每天必须先交一个涨停全景总表

总表路径：

```text
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.json
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.md
```

总表必须包含：

| 区块 | 要求 |
|---|---|
| 全市场统计 | 剔除 ST 后的涨停总数、首板数、2板及以上数、主板10cm、创业板20cm、科创板20cm、北交所30cm |
| 首板涨停 | 全量首板票，含题材、涨停原因、催化来源、Top100重合、明日验证点 |
| 连板天梯 | 剔除 ST 后的全量 2板及以上票，含板数、市场、题材、封板质量 |
| 2板以上6维 | 每只 2板及以上必须有题材归属、事件催化、产业逻辑、个股地位、盘口质量、明日验证点 |
| ST口径 | ST 直接剔除，不写明细、不单列、不补全；数据源未覆盖要写数据缺口 |

这个总表是 Codex 和龙虾共同读取的主入口。下面的首板日报、连板天梯、6维原因仍然保留为结构化分表，方便脚本训练和验收。

## 一、每天首板单独拉、单独写

首板涨停票每天单独写 RAW/JSON：

```text
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.json
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.md
```

首板日报用于发现新题材、新消息催化、预期差和次日补涨，不进入连板天梯。

首板必须同时进入“每日涨停全景总表”的首板区。每日复盘正文不需要把全部首板逐票铺开，只写主题聚类、Top100重合、最强预期差和明日需要重点验证的首板样本。

每只首板至少写：

| 字段 | 要求 |
|---|---|
| 代码/名称 | 必填 |
| 市场/涨停幅度 | 主板10cm、创业板20cm、科创板20cm、北交所30cm |
| 题材 | 不要只写行业，尽量写具体催化方向 |
| 涨停原因 | 通达信原因原文 + 你能查到的补充证据 |
| 催化来源 | 财联社/公告/研报/互动易/公众号/通达信原因/未知 |
| Top100 重合 | 是否进入通达信热榜 Top100，排名多少 |
| 明日验证点 | 次日竞价、强弱、是否扩散、是否补涨 |
| 数据缺口 | 查不到就写缺口，不能编 |

## 二、连板天梯只写 2板及以上

连板天梯路径：

```text
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.json
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.md
```

天梯只包含剔除 ST 后的 2板及以上票，必须覆盖：

```text
主板10cm
创业板20cm
科创板20cm
北交所30cm
```

ST 直接剔除，不写明细、不单列、不补全。

2板及以上必须同时进入“每日涨停全景总表”的连板区，并且进入每日复盘正文重点分析。

## 三、每只 2板及以上必须有 6 维度

每只 2板及以上必须拆成：

| 代码 | 名称 | 题材归属 | 事件催化 | 产业逻辑 | 个股地位 | 盘口质量 | 明日验证点 |
|---|---|---|---|---|---|---|---|

盘口质量至少写：

```text
板型、首次封板、开板次数、封单、成交额、换手
```

缺字段就写“数据缺失”，不要省略。

## 五、验收不过的硬错误

1. 没有首板涨停催化日报。
2. 没有每日涨停全景总表。
3. 把首板混入连板天梯。
4. 连板天梯漏掉 2板。
5. 漏掉创业板20cm、科创板20cm、北交所30cm。
6. 2板及以上没有逐票 6 维度。
7. 只写“上榜原因”，不写催化来源和明日验证点。
