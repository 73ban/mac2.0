# 发给龙虾：22点公告全量 RAW 执行规则

```yaml
type: 执行规则
date: 2026-07-04
target: 龙虾
mode: 事实层RAW
```

## 总原则

22:00 公告任务只做原始数据落盘，不做主观筛选，不写交易建议。

你只负责用 `wenda_notice_query` 按关键词分 4 批拉公告，把原始字段写入 RAW。Codex 后续负责关联持仓、作战室、热榜、涨停票，并判断哪些公告有交易价值。

## 输出位置

统一写入：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.json
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.md
```

如果工具失败，写：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements-error.md
```

## 4批关键词

| 批次 | 关键词 | 分类标签 | 重点 |
|---:|---|---|---|
| 1 | 业绩预告、业绩快报、业绩修正 | 业绩 | 超预期、亏损、修正、增长、扭亏 |
| 2 | 减持、增持、回购 | 增减持 | 大比例减持、实控人增持、股份回购 |
| 3 | 异动公告、问询函、关注函、立案调查 | 监管 | 高标风险、出监管、问询、立案、澄清 |
| 4 | 重大合同、中标、定增、重组、并购 | 重组 | 合同催化、资产注入、并购重组、定增投向 |

## 必须保留的原始字段

每条公告至少写这些中文字段：

```text
公告标题
公司名称
股票代码
公告日期
分类标签
查询关键词
公告摘要
原始来源
原始字段
数据缺口
```

如果 `wenda_notice_query` 返回字段是英文，也要映射成中文：

| 原字段 | 中文字段 |
|---|---|
| title | 公告标题 |
| name | 公司名称 |
| code | 股票代码 |
| date | 公告日期 |
| brief | 公告摘要 |
| url/link | 公告链接 |

`原始字段` 里可以保留完整返回对象，方便 Codex 复核。

## 去重规则

同一条公告可能被多个关键词命中，不能重复写多条。

去重键：

```text
股票代码 + 公告标题 + 公告日期
```

如果同一公告命中多个关键词，把关键词合并到：

```text
命中关键词
分类标签
```

## 不允许做

- 不允许只拉并购重组。
- 不允许只拉标题不写股票代码。
- 不允许用模型写公告影响判断。
- 不允许漏写失败批次。
- 不允许把公告摘要改写成交易结论。

## 交付后 Codex 做什么

Codex 会读取：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.json
```

然后交叉：

- 当前持仓。
- 作战室候选。
- 同花顺热榜Top100。
- 通达信热榜Top100。
- 成交额Top100。
- 每日涨停全景。
- 连板天梯。
- 龙虎榜。

最后输出公告催化初筛和次日验证清单。
