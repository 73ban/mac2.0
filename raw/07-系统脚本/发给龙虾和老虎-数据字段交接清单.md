# 发给龙虾和老虎：数据字段交接清单

## 一、目的

事实层不能只把 JSON 丢给 Codex，还要明确每个字段应该用于什么分析。

Codex 看到字段后必须使用；如果没有使用，盘后验收要标记为“数据利用不足”。

## 二、每日涨停全景字段用途

文件：

```text
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-daily-limit.json
```

字段用途：

| 字段 | 用途 |
|---|---|
| 首次涨停 | 先锋龙排序、首板强度、竞价后抢筹验证 |
| 连板天数 | 连板天梯、趋势龙、主线高度 |
| 几天/几板 | 趋势龙、反包/多日强度识别 |
| 板型 | 一字、T字、换手板质量判断 |
| 开板次数 | 封板质量、分歧程度 |
| 封单金额 | 封板强度、先锋龙辅助 |
| 最大封单额_万 | 封单峰值，判断是否被撤单 |
| 涨停成交额_万 | 资金参与度，资金龙兜底 |
| 涨停原因 | 主题聚类、首板催化、连板主线 |
| 原因揭秘 | 新闻/公告/产业逻辑溯源 |
| 主力净额万元 | 资金龙核心排序 |
| 超大单净额万元 | 大资金确认 |
| 大单净额万元 | 大单承接补充 |

## 三、热榜Top100字段用途

文件：

```text
raw/04-市场数据/通达信热榜/YYYY-MM-DD/tdx-hot-top100.json
```

字段用途：

| 字段 | 用途 |
|---|---|
| 排名 | 人气强度 |
| 人气值/热度分 | 人气变化强度 |
| 涨跌幅 | 热度是否转化为正反馈 |
| 概念标签 | 板块归属辅助 |
| 上榜原因 | 情绪催化 |

必须和成交额Top100、涨停全景交叉，不能单独看热榜排名。

## 四、成交额Top100字段用途

文件：

```text
raw/04-市场数据/通达信成交额排名/YYYY-MM-DD/tdx-成交额Top100.json
```

字段用途：

| 字段 | 用途 |
|---|---|
| 成交额亿 | 资金战场、中军、容量确认 |
| 换手率 | 承接和分歧程度 |
| 流通市值亿 | 中军识别 |
| 自由流通市值亿 | 真实容量 |
| 涨跌幅 | 资金正反馈或负反馈 |

## 五、板块成分股字段用途

文件：

```text
raw/04-市场数据/板块成分股/YYYY-MM-DD/tdx-board-members.json
raw/04-市场数据/板块成分股/YYYY-MM-DD/tdx-stock-boards.json
```

用途：

- 具体板块四龙必须优先使用板块成分股映射。
- 个股板块归属用于热榜、成交额、涨停票补板块字段。
- 主题聚合只能作为辅助，不能替代具体板块映射。

## 六、Codex必须输出的交叉结果

每天盘后 Codex 必须生成：

- 热榜100资金情绪交叉分析。
- 连板天梯与高度情绪分析。
- 首板涨停催化分析。
- 涨停全景交叉复盘。
- 板块四龙候选。
- 次日作战室输入。

每份报告必须至少使用：

- 热榜排名。
- 成交额。
- 涨停状态。
- 主力净额。
- 超大单净额。
- 板块归属。
- 涨停原因。

## 七、验收标准

如果事实层已经提供字段，但 Codex 报告没有使用，判定为：

```text
数据利用不足
```

典型问题：

- 有主力净额但资金龙没用。
- 有板块成分股但板块归属空白。
- 有连板资金流但天梯只列高度。
- 有首板资金流但首板只列涨停时间。
