# 发给龙虾：低成本事实层执行规则

更新时间：2026-07-03

## 结论

可以改成低成本模式。

龙虾不再用 DeepSeek 做 Top100、涨停全景、首板、连板、板块强度的深度分析。龙虾只负责拉原始数据、标准化、写 RAW、标注缺口。

Mac 端 Codex Pro 负责读取 RAW 后做：

1. 6维分析
2. 主线/伪主线判断
3. 股票候选打分
4. 作战室
5. D+验证
6. 错误库和模式进化

## 龙虾必须做

| 数据 | 龙虾交付 | 不允许 |
|---|---|---|
| 通达信热榜 Top100 | 全量 100 只，原始排名、涨跌幅、热度、标签、原因、板块字段 | 只写 Top10/Top20；用 DeepSeek 总结替代原始表 |
| 每日涨停全景 | 剔除 ST 后，首板和 2板及以上总入口 | 只写核心票；漏首板；漏 20cm/30cm |
| 首板涨停事实源 | 全量首板，原始涨停原因、来源、板块、Top100重合、盘口字段 | 写“首板很多”；写主观看法不写原始原因 |
| 连板天梯 | 只写 2板及以上，含高度、类型、市场、板型 | 把首板混进连板天梯 |
| 涨停原因事实源 | 2板及以上逐票原始原因、板型、封单、开板、成交额、换手、来源 | 用 DeepSeek 写空泛6维分析 |
| 板块强度 | 全量板块排名优先，最低 Top100，含板块代码 | 只写涨幅前几名 |
| 数据缺口 | 拉不到什么写清楚 | 假装完整 |

## 龙虾不要做

以下工作不要用 DeepSeek 做：

1. 不写“产业逻辑是否真实受益”的深度判断。
2. 不写“个股地位：龙头/补涨/中军”的最终定性。
3. 不写“明日验证点”的完整交易判断。
4. 不写作战室结论。
5. 不写错误卡最终定性。
6. 不写模式升级/降级。

如果接口已经给出“涨停原因”“概念标签”“板块归属”，必须原样写入；这是事实，不是分析。
