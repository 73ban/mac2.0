## 结论

`tdxrs` 可以作为 73wiki 的通达信实时行情候选接入层，但不能直接替代全部现有数据链路。

适合做：

- 高速 K 线读取。
- 批量实时行情。
- 五档盘口、成交额、总量等盘口行情。
- 当日/历史分时、逐笔成交。
- 本地通达信 `.day` / `.lc5` / `.dat` / 财务文件解析。
- 板块、财务、基金行情补充。

不适合直接承诺：

- 交易下单。
- 逐笔委托。
- 官方授权数据服务。
- 替代通达信客户端或当前龙虾/老虎事实层所有接口。

## 从截图提取的关键信息

1. `tdxrs` 是基于通达信库 `tdxpy` 的 Rust 高性能重构，并封装 Python 调用接口。
2. 作者称本地文件解析性能提升约 9-11 倍。
3. 连接池可复用 TCP 连接，减少每次约 200ms 握手。
4. 高并发场景使用裸连接方案，60 线程取 50 条 K 线约 344ms。
5. 内置自适应限流：15 / 30 / 60 req/s。
6. 评论区确认“有实时数据”，并称可比部分财经官网略快。
7. 评论区未确认逐笔委托能力，不能把它当成交易委托接口。
8. 依赖较轻，作者称约 500k，可解析本地通达信格式数据。

## GitHub README 核验

README 明确给出：

```python
from tdxrs import TdxHqClient
from tdxrs.constants import MARKET_SH, KLINE_DAILY, FQ_QFQ

client = TdxHqClient()
client.connect_to_any()

df = client.get_security_bars_dataframe(KLINE_DAILY, MARKET_SH, "600519", 0, 500)
quotes = client.get_security_quotes([
    (MARKET_SH, "600519"), (0, "000858"), (0, "300750")
])
```

README 声明的能力包括：

- K 线：个股 + 指数，12 种周期。
- 实时行情：五档盘口，含成交额 / 总量。
- 分时数据：当日 + 历史。
- 逐笔成交：当日 + 历史。
- 证券信息：全市场列表 + 数量。
- 财务数据：实时 34 项 + 英文字段财务指标。
- 除权除息：分红 / 送股 / 配股 / 缩股历史。
- 板块数据：行业 / 概念 / 地域分类。
- 基金数据：ETF / LOF / REITs / 分级基金。

## 73wiki 接入建议

先做候选通道，不进入主链：

```text
raw/04-市场数据/tdxrs试验/YYYY-MM-DD/
```

试验数据只写 RAW，不直接写 WIKI 结论。

必须和当前龙虾/老虎数据做对照：

- 同一批股票实时行情字段对照。
- 同一只股票 K 线前复权/不复权对照。
- 同一批涨停股五档盘口字段可用性对照。
- 同一批板块成分和板块涨幅对照。

## 验收标准

可以进入正式链路前，必须满足：

1. 安装稳定，Mac 与 Windows 都能跑。
2. 交易时间内连续运行 2 小时，不崩溃。
3. 批量行情 100 只以内可稳定分批获取。
4. K 线字段与现有通达信/龙虾数据误差可解释。
5. 不触发明显限流或封禁。
6. 输出字段全部中文化，不能直接把英文 API 字段写入 RAW。
7. 所有数据必须写 RAW，Codex 再分析，不让事实层写结论。

## 风险

- 开源项目更新快，接口可能变化。
- README 明确有学习研究用途和数据授权免责，不能当商业授权源。
- 通达信行情服务器稳定性、限流策略不可控。
- 逐笔委托未确认，不得纳入交易执行链。
