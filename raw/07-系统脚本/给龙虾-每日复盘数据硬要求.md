# 给龙虾：每日复盘数据硬要求

这份可以直接按原文执行。你的边界仍然是事实层 RAW，不直接写 WIKI，不做最终错误定性。

硬要求：交割单和每日复盘不能只留在临时目录或聊天里，必须写入 Mac 本机 73wiki 的 RAW。Codex 会自动发布到正式 WIKI；如果缺 RAW，视为当天流程未完成。

## 每天必须写入

1. 交割单：原始凭证、成交摘要、持仓摘要、每笔交易的用户口述理由。
2. 通达信热榜 Top100：必须全量，不要只写 Top10/Top20；同花顺热榜由 Mac 自动抓取，对端只验收同步，不重复爬。
3. 每日涨停全景总表：每天所有涨停票放在一个总入口文件里，首板和 2板及以上都能一眼看到。
4. 连板天梯：只写 2板及以上，主板、创业板 20cm、科创板 20cm、北交所 30cm 都要纳入；首板不得混入天梯。
5. 首板涨停催化日报：每天首板涨停票单独拉数据、单独写 RAW/JSON，作为新题材和预期差训练池。
6. 涨停原因事实源：每只 2板及以上连板票必须写清涨停原因原文、来源、盘口字段和数据缺口；6维深度分析由 Codex Pro 基于 RAW 完成。首板在催化日报里写清涨停原因、催化来源和明日验证点。
7. 板块字段：每只涨停/热榜/候选票必须写 `交易题材`、`主板块`、`板块来源`、`概念标签`、`板块归属证据`。
8. 板块强度：每天写 `raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.json/md`，优先全量，接口受限最低 Top100。
9. 财联社/公众号催化：只保留能刺激题材或个股的高信号内容，并标注对应方向。
10. 明日验证点：核心候选必须写 D+1 验证点。

## 连板天梯必须分清

不要混用口径。连板天梯每天只写 2板及以上，并分开写：

```text
连续涨停高度
N日M板高度
反包板
趋势新高板
```

例如 `3连板` 和 `5天3板` 不能混在同一列。
首板不属于连板天梯，必须进入“每日涨停全景总表”和独立的首板涨停催化日报。每日复盘正文不需要逐票铺开全部首板，只写首板主题聚类、Top100 重合、重点预期差和明日验证样本。

## 涨停原因事实源字段

每只 2板及以上连板票必须尽量交齐以下事实字段：

| 代码 | 名称 | 题材归属 | 事件催化 | 产业逻辑 | 个股地位 | 盘口质量 | 明日验证点 |
|---|---|---|---|---|---|---|---|

盘口质量至少写：

```text
板型、首次封板、开板次数、封单、成交额、换手
```

缺哪项就写“数据缺失”，不要省略。

## 结构化 RAW

Markdown 给人看，JSON 给 Codex 训练。尽量每天同步写：

```text
raw/04-市场数据/通达信热榜/YYYY-MM-DD/tdx-hot-top100.json
raw/04-市场数据/通达信热榜/YYYY-MM-DD/tdx-hot-top100.md
raw/04-市场数据/同花顺热榜/YYYY-MM-DD/ths-hot-top100.json
raw/04-市场数据/同花顺热榜/YYYY-MM-DD/ths-hot-top100.md
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.json
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.md
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.json
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.md
raw/04-市场数据/通达信涨停原因/YYYY-MM-DD/tdx-limit-reasons-6d.json
raw/04-市场数据/通达信涨停原因/YYYY-MM-DD/tdx-limit-reasons-6d.md
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.json
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.md
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.json
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.md
```

目录分工：

- `同花顺热榜`：Codex/Mac 抓取的人气热榜 Top100，必须同时有 JSON 和 MD；龙虾/老虎只检查是否同步成功，不重复爬、不重复分析。
- `通达信热榜`：龙虾/TDX 抓取的 Top100，必须同时有 JSON 和 MD。
- `每日涨停全景`：所有涨停票总入口，首板和 2板及以上都放在一个文件里，给 Codex 和龙虾共同读取。
- `首板涨停催化`：每天全量首板涨停票，单独写 JSON 和 MD，用于新催化、新题材、预期差训练；同时被纳入每日涨停全景。
- `通达信涨停原因`：剔除 ST 后的 2板及以上连板票逐票事实源，单独写 JSON 和 MD；不要混到 `通达信热榜` 目录。事实源必须包含原因原文、原因来源、盘口字段、板块字段和数据缺口；6维深度分析由 Codex Pro 完成。
- `板块强度`：每天全量板块排名，接口受限最低 Top100；另写涨停家数 Top20、主线候选 Top10，用于 Codex 计算个股是否跑赢主板块。

所有个股 JSON 必须有板块字段：

```yaml
交易题材:
主板块:
主板块代码:
板块来源: tdx | ths | eastmoney | manual | unknown
备选板块:
行业归属:
概念标签:
板块归属证据:
```

如果做不到 JSON，复盘正文必须写：

```text
数据缺口：结构化 JSON 未生成
```

JSON 不能只写核心样本却伪装成全量。每天必须在 JSON `元信息` 里写清楚：

```yaml
是否样本: false
选择规则: 剔除 ST 后的全市场 2板及以上连板全量，首板另走首板涨停催化日报
2板及以上总数:
剔除ST后2板及以上总数:
市场范围:
  主板10cm数量:
  创业板20cm数量:
  科创板20cm数量:
  北交所30cm数量:
  ST口径: 已剔除，不写明细、不单列、不补全
  数据缺口:
```

如果 JSON 只写核心样本，必须写：

```yaml
是否样本: true
选择规则: 只保留连板/高辨识度/Top100重合样本
full_source_path: raw/02-每日复盘/YYYY-MM-DD-复盘.md#七、连板天梯与情绪锚
```

## 禁止偷懒写法

以下写法不合格：

1. “今日科技很强，明天继续看科技。”
2. “热榜前排很强。”但不给 Top100。
3. “连板高度不错。”但不列完整天梯。
4. “涨停原因见通达信。”但不写原因原文、来源和盘口字段。
5. “明天观察。”但不给具体 D+1 验证条件。
6. 涨停总数、首板数、2板及以上数这类口径不一致，但不解释差异。
7. 把首板混进连板天梯，或者没有单独首板涨停催化日报。
8. 把创业板 20cm、科创板 20cm、北交所 30cm 从天梯里过滤掉。
9. JSON 只有几十只核心票，但 `元信息.剔除ST后数量` 写 100 多只，却不声明 `是否样本: true`。
10. 只写题材，不写 `主板块`。
11. 不交板块强度 JSON/MD。

## 天梯市场范围硬规则

天梯不是只看 10cm 主板。每天必须单列统计：

```yaml
连板天梯市场范围:
  主板10cm数量:
  创业板20cm数量:
  科创板20cm数量:
  北交所30cm数量:
  纳入口径: 主板 + 创业板20cm + 科创板20cm + 北交所30cm
  ST口径: 已剔除，不写明细、不单列、不补全
  数据缺口:
```

如果北交所当天没有涨停，必须写 `北交所30cm数量: 0`，不能不提。
