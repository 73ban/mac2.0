# 发给龙虾和老虎：板块字段与板块强度执行规则

从现在开始，所有市场数据 RAW 必须支持 Codex 做“个股是否跑赢板块”的 D+验证。只写题材名称不合格，必须写可解析的板块字段。

## 一、适用范围

以下文件里的每只股票都必须带板块字段：

```text
raw/04-市场数据/每日涨停全景/YYYY-MM-DD/tdx-limit-up-panorama.json
raw/04-市场数据/首板涨停催化/YYYY-MM-DD/tdx-first-board-catalyst.json
raw/04-市场数据/通达信连板天梯/YYYY-MM-DD/tdx-limit-ladder.json
raw/04-市场数据/通达信涨停原因/YYYY-MM-DD/tdx-limit-reasons-6d.json
raw/04-市场数据/通达信热榜/YYYY-MM-DD/tdx-hot-top100.json
raw/04-市场数据/同花顺热榜/YYYY-MM-DD/ths-hot-top100.json
raw/02-每日复盘/YYYY-MM-DD-复盘.md
```

## 二、每只股票必填字段

JSON 每只股票必须至少包含以下中文字段：

```json
{
  "代码": "600000",
  "名称": "示例股份",
  "交易题材": "六氟化钨/电子特气",
  "主板块": "电子化学品",
  "主板块代码": "",
  "板块来源": "通达信",
  "备选板块": [
    {
      "板块名称": "电子化学品",
      "板块代码": "",
      "来源": "通达信",
      "排序": 1,
      "归属理由": "涨停原因、主营或通达信板块归属最匹配"
    }
  ],
  "行业归属": "基础化工",
  "概念标签": ["电子特气", "半导体材料", "涨价"],
  "板块归属证据": "通达信板块/同花顺概念/涨停原因/主营信息"
}
```

字段口径：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| `交易题材` | 必填 | 交易题材或催化主题，例如六氟化钨、PCB、CPO、机器人 |
| `主板块` | 必填 | 用于 D+对比的主板块，只能填一个 |
| `主板块代码` | 尽量填 | 如果通达信/同花顺能返回稳定板块代码，必须写；拿不到可以空 |
| `板块来源` | 必填 | 通达信 / 同花顺 / 东方财富 / 手工确认 / 未确认 |
| `备选板块` | 必填 | 多板块时列候选，排序第一必须等于 `主板块` |
| `行业归属` | 尽量填 | 申万/通达信/同花顺一级行业，拿不到写空 |
| `概念标签` | 必填 | 相关概念标签数组，不能只写一句话 |
| `板块归属证据` | 必填 | 为什么归到这个板块，写证据来源 |

## 三、主板块选择规则

一只票可能同时属于行业、概念、题材、地域、国企改革。主板块按这个优先级选：

1. 当日涨停原因直接指向的催化板块。
2. 当日市场正在扩散的主线题材。
3. 个股主营最相关的产业链板块。
4. 通达信/同花顺热榜或板块涨幅排名中最强的相关板块。
5. 如果仍不确定，`主板块` 写“未确认”，`板块来源` 写“未确认”，并在 `板块归属证据` 写清缺口。

禁止把“国企改革、融资融券、华为概念、人工智能”这类泛标签直接当主板块，除非当天涨停原因明确就是该泛标签驱动。

## 五、每天必须新增板块强度文件

每天收盘后必须交付：

```text
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.json
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.md
```

板块强度必须优先全量排名；接口受限时最低 Top100。Top50 只允许作为临时缺口，并必须在 meta.data_gap 说明原因。每个板块必须包含：

```json
{
  "日期": "YYYY-MM-DD",
  "板块排名": 1,
  "板块名称": "电子化学品",
  "板块代码": "",
  "来源": "通达信",
  "涨跌幅": 3.21,
  "涨停家数": 8,
  "上涨家数": 42,
  "下跌家数": 6,
  "成交额": "",
  "领涨股": [
    {"代码": "600000", "名称": "示例股份", "涨跌幅": 10.01}
  ],
  "板块催化摘要": "涨价/政策/订单/事件催化摘要"
}
```

Markdown 里必须有三张表：

1. 板块涨幅全量排名；接口受限时最低 Top100。
2. 涨停家数 Top20。
3. 今日主线候选板块 Top10。

## 六、每日复盘里怎么写

`raw/02-每日复盘/YYYY-MM-DD-复盘.md` 必须新增或保留“主线与板块”段落，至少写：

```text
## 主线与板块

| 排名 | 板块 | 涨幅 | 涨停数 | 领涨股 | 代表催化 | 明日验证 |
|---:|---|---:|---:|---|---|---|
```

这里不是长篇分析，重点是给 Codex 做 D+验证和次日作战室使用。

## 七、验收标准

合格：

1. 每只股票都有 `交易题材`、`主板块`、`板块来源`、`概念标签`、`板块归属证据`。
2. 每天有 `板块强度` JSON 和 MD。
3. `主板块` 能在板块强度文件里找到同名板块，或者明确写“未确认”。
4. 多题材票有 `备选板块`，并说明为什么排序第一。

不合格：

1. 只写“题材：机器人”，没有 `主板块`。
2. 只写“上榜原因”，没有板块字段。
3. 用泛概念糊弄主板块。
4. 没有板块强度文件。
5. 板块字段和涨停原因明显冲突，但没有解释。
