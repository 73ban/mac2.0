# 每日工作流程与关注重点

> 最后更新：2026-07-05
> 此文件概括用户对TdxClaw的日常需求模式，方便新会话快速理解。

## 一、每日复盘（核心任务）

### 触发时机
- 交易日收盘后，用户发送当日交割单时触发

### 复盘模板（13段结构）
1. 赚钱效应：涨停/跌停家数、各指数涨跌、市场情绪判断
2. 盘前背景：昨持仓、昨现金、昨日计划回顾
3. 盘面关键信号：当日板块轮动、题材切换、异动
4. 今日操作：买入/卖出明细表格，金额汇总
5. 热门板块拆解：最强方向+最弱方向
6. 连板天梯：各连板高度+题材+板型
7. 资金流向：主力/北向资金动向
8. 消息面：当日重要催化、政策、新闻
9. 关键反思：操作逻辑对错、纪律执行、模式总结
10. 心态与纪律：当日心理状态+纪律遵守情况
11. 资金概览：总资产/现金/负债/今日盈亏
12. 明日操盘要点：核心策略+风险监控
13. 关注方向：下个交易日的优先级排序

### 数据来源优先级
1. 交割单Excel/截图 → 直接解析
2. 通达信实时行情 → tdx_quotes
3. 涨停/跌停筛选 → tdx_screener
4. 用户口述补充 → 定性分析、心态、逻辑

### 核心规则
- 资金必须与交割单+银证转账完全吻合，不允许矛盾
- 买入逻辑由用户口述，AI不能自行编造
- 不确定的数据标注"数据缺失"，不编造
- 复盘双写：`raw/02-每日复盘/` + `wiki/09-统计与进化/`
- 交割单双写：`raw/01-交割单/YYYY-MM-DD/交割单.md` + `wiki/06-持仓与资金管理/YYYY-MM-DD-交割单.md`
- 每天交割单和复盘必须正式写入 WIKI；只停在 RAW 视为流程未完成。
- 盘后必须刷新 RAW 覆盖、月度统计、D+验证、错误账本和 AI 上下文

### 当前自动命令

```bash
python3 raw/07-系统脚本/codex_daily_workflow.py --date YYYY-MM-DD --phase premarket
python3 raw/07-系统脚本/codex_daily_workflow.py --date YYYY-MM-DD --phase postmarket
python3 raw/07-系统脚本/codex_daily_workflow.py --date YYYY-MM-DD --phase context
```

`postmarket` 阶段现在会自动执行：

1. `codex_prepare_postmarket.py`：生成盘后复盘/统计容器。
2. `codex_audit_raw_coverage.py`：刷新 RAW 交割单/复盘覆盖索引。
3. `codex_build_monthly_trade_stats.py`：刷新月度统计初稿。
4. `codex_build_error_ledger.py`：刷新错误候选 JSONL 和错误成本账本草案。
5. `codex_update_ai_context.py`：刷新 Wiki 对话框/Codex Local 默认读取上下文。
6. `codex_publish_daily_raw_to_wiki.py`：把当日交割单和复盘从 RAW 发布到正式 WIKI，并生成缺口审计。

## 二、用户关心的核心问题

### 交易逻辑类
- 每笔买卖的逻辑是否成立？（分歧买入/一致卖出）
- 当前持仓是否符合主线？（只做主线龙头，不做跟风）
- 是否有过气票逻辑已破但还没走？（炒新不炒旧）

### 市场判断类
- 今日情绪如何？（涨停家数趋势、指数形态）
- 主线/支线结构？（哪个方向是资金主攻、哪个被抛弃）
- 大盘是否见顶？（涨停缩量+指数收阴=见顶信号）

### 纪律执行类
- 是否满仓单调？（理想：1-2只核心票）
- 资金使用是否合理？（满仓满融无容错=警告）
- 是否有计划外交易？

### 持仓审查类
- 每个票的买入逻辑还在不在？
- 逻辑在但资金走了？ → 清仓
- 逻辑在且资金在？ → 持有
- 新题材出现？ → 老题材无条件清仓

## 三、数据工具使用模式

| 场景 | 工具 |
|------|------|
| 查实时行情 | `tdx_quotes`（个股+指数同时查） |

## 四、关键偏好

- **风格**：直接给结论+数据+逻辑，不要废话、不要客服话术
- **速度**：少绕圈子，先做再说
- **格式**：表格优先，结构化输出
- **交互**：复盘需要用户口述的部分主动问（买入逻辑、心态），但不反复追问同一个问题

## 七、常见错误与踩坑

- ❌ 不看昨天复盘直接读交割单 → 买卖方向判反
- ❌ 账户逻辑错误：两个股东代码都是信用账户，只有"融资买入"和"证券买入"的区别，没有"信用vs普通"两套资金池
- ❌ 复盘中"普通账户/信用账户"表述 → 已改为"本金/融资"
- ❌ memory-wiki插件配置被版本升级覆盖，需要重新写入
- ❌ context接近200k上限时响应变慢 → 建议新开会话
- ❌ 只看复盘不看交割单 → 必须先读 [[2026-06-RAW交割单复盘覆盖索引]] 和 [[当前持仓决策]]
- ❌ 自动 A+ 当买入理由 → 必须先过 [[自动核心池评分修正规则]] 和 D+验证
