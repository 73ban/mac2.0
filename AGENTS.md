# 73神话 Codex 工作说明

## 项目定位

这是杨明的 Trading Review Wiki，本机 Mac 是长期主机。

根目录：

```text
/Users/qixinchaye/wiki/73神话
```

云服务器和 Windows 本地电脑均不作为当前工作链路；73wiki 当前只以本机 Mac 为主机和执行环境。

## 核心分工

- `raw/`：原始证据和事实层数据，先保真，再分析。
- `wiki/`：沉淀后的交易规则、复盘、作战室、错误库、模式库、统计。
- `data/`：结构化事实、索引、验证结果。
- `.system/`：Mac 本机定时任务、运行报告、健康检查。
- `.conflicts/`：冲突、隔离、待确认材料。

## 智能体边界

- 龙虾/老虎继续负责事实层 RAW：交割单、每日复盘口述、市场数据、公告、龙虎榜、涨停全景、板块强度等按既定分工交付。
- 龙虾/老虎是长期协作角色，不等于 Windows 或云服务器；当前落盘、验收和分析都以本机 Mac 的 73wiki 为准。
- Codex 负责分析层：正式 WIKI 复盘、今日作战室、D+验证、错误库、模式进化、公告事件验证、晚间线索。
- `raw/05-研报新闻` 是事实层接收目录，Codex 派生产物不要写进去；Codex 分析产物放 `raw/11-Codex分析产物`。
- PaddleOCR 链路保留：图片先入 RAW，OCR 只生成文字侧车，不覆盖原图。
- 淘股吧高手样本库先做准结构，不急着扩大规模；优先保证比赛白名单、千万组前10选手池、公开交割单字段、每日分享字段、D+验证字段。

## 新窗口启动顺序

新 CLI 窗口或上下文不连续时，先读：

```text
MEMORY.md
wiki/10-系统配置/2026-07-05-Codex新窗口交接单.md
wiki/10-系统配置/Codex每日启动检查清单.md
wiki/10-系统配置/数据接口统一总控.md
wiki/10-系统配置/截图OCR自动化规则.md
wiki/10-系统配置/淘股吧实盘赛样本学习规则.md
wiki/10-系统配置/Codex每日工作必需数据与缺口清单.md
```

用户只说“继续”时，默认按最新交接单的 P0 任务推进。

## 硬规则

- 不能把“Mac 没收到文件”直接说成“外部事实层没做”；当前先排查本机任务、RAW入口和用户是否已提供原始资料。
- 不能把 RAW 空壳当成已完成。
- 不能把旧服务器、旧 Windows 路径、旧目录口径当成当前事实或当前任务入口。
- 不能漏掉交割单和复盘必须写入正式 WIKI。
- 不能把事实层原始数据和 Codex 派生产物混写到同一 receiveonly 目录。
- 批量删除、同步改向、定时任务重建、历史口径重写，先隔离或出 dry-run 报告。
- 改完系统文件必须看变更摘要或 diff。

## 每日复盘硬输入

正式复盘至少读取：

```text
raw/01-交割单
raw/02-每日复盘
raw/04-市场数据
raw/05-研报新闻
raw/10-飞书交易沟通
wiki/07-作战室
wiki/09-统计与进化/D+验证待回填总览.md
```

交割单和用户口述先保留 RAW 原文，再写正式 WIKI。

## 交易系统重点

- 今日作战室要有来源、触发条件、失效条件和 D+验证入口。
- 公告、互动问答、重大新闻不能只做摘要；要转成事件样本并跟踪 D+0/D+1/D+3/D+5/D+10。
- 选票必须事后验证，否则不能进化。
- 复盘不能只有叙事，必须能回到数据、交易、错误、模式、次日计划。

## 高风险停问条件

以下情况不要直接最终执行：

- 删除不可恢复资料；
- 覆盖用户原始口述、截图、交割单、公告；
- 改 Syncthing 方向或同步根目录；
- 改历史交易结论口径；
- 涉及费用、账号、权限、微信授权、Docker、sudo；
- 数据不足但会影响买卖判断；
- 新工具要替换已经跑通的数据链路。
