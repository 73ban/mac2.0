# 复盘连续性缺陷

## 总览：3月至5月系统性缺失

从2026年3月初到5月中旬，交易复盘存在**大面积数据缺失**，累计缺失约 **66个交易日**。

## 缺失时间线

| 时间段 | 天数 | 状态 | 备注 |
|--------|------|------|------|
| 3/1 ~ 3/2 | 2天 | ⚠️ 缺失 | |
| 3/6 | 1天 | ✅ 已复盘 | [[个股档案/兆易创新-603986\|兆易创新]]融资买入 |
| 3/7 | 1天 | ⚠️ 缺失 | |
| 3/8 ~ 3/31 | 24天 | ⚠️ 缺失 | 最长连续缺失段之一 |
| **4/1 ~ 4/22** | **22天** | **⚠️ 缺失（新发现）** | **连接3月→5月的关键桥梁断裂** |
| 5/1 ~ 5/17 | 17天 | ⚠️ 缺失 | |
| 5/15 | 1天 | ✅ 持仓快照 | IDC浮亏 + 韶能实亏3.69万 |
| 5/22 ~ 5/23 | 2天 | ✅ 已复盘 | PCB高潮次日追高，模式漂移诊断 |

**缺失天数累计**：2+1+24+22+17 = **66天**

## 4月缺失的特殊重要性（🆕 2026-06-02）

4月1~22日（22天）的缺失是**截至目前发现的最关键的断裂点**：

### 为什么4月是“因果链缺失环”

```
3/6 兆易创新买入 ──▶ 4月全月空白 ──▶ 5/15 IDC持仓浮亏 ──▶ 5/22 PCB追高
     存储线               ???              IDC线              模式漂移
```

- 3/6：融资买入兆易创新（存储/MCU/长鑫催化）
- 4月：**全月空白** — 兆易可能被卖出/加仓，新标的可能被建仓
- 5/15：持仓快照显示IDC两票（数据港/美利云）浮亏、韶能实亏3.69万
- 5/22–23：PCB高潮次日追高，诊断为模式漂移

**核心问题**：5/22的“模式漂移”诊断缺少4月的前置数据。如果4月已在漂移，那5月就不是“突发漂移”而是“漂移延续”——诊断性质将完全不同。

### 无法回答的关键问题

1. 从存储线到IDC线，是主动策略切换还是被动追热点？
2. 韶能是否在4月买入？亏损是4月扩大还是在5月形成？
3. 4月是否已有模式漂移的征兆？
4. 4月的胜率、持仓天数、仓位集中度如何？

## 系统性影响

### 对Wiki目标的冲击

Wiki核心目标：“三个月后能清晰回答核心盈利模式、常见错误、空仓环境”

**现实**：66天缺失（约占3个月窗口的2/3），使这一目标在数据层面失去基础。

### 对已有诊断的影响

- 5/22–23的精复诊断缺少前因
- “模式漂移”的定性可能因缺失数据而改变
- 错误归因缺乏完整因果链支持

## 根本原因探索（待确认）

- 文件在后期被创建但未填充（如4月文件创建于5/24）
- 暗示用户**意识到需要补全**但未执行
- 可能原因：券商数据不可得、补全意愿不足、数据量太大难以处理

## 行动建议

1. 从券商系统导出全部3‑5月交割单
2. 优先补全4月数据（最关键断裂点）
3. 即使不完整，也比完全空白强

### 流程改进
4. 建立“每周至少导出一次交割单”的固定流程
