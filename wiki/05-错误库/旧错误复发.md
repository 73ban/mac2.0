# 旧错误复发：已知错误的系统性重演

## 定义

「旧错误复发」指交易者此前已识别、归档、并理论上“明白”的某类错误，在新交易周期中再次出现。它暴露了 **认知理解 ≠ 行为阻断** 的根本断裂。

## 核心特征

| 特征 | 说明 |
|------|------|
| **已知性** | 该错误在 [[错误/错误案例库]] 中已有归档，交易者能口头描述其危害 |
| **重复性** | 在相似的触发条件下再次出现 |
| **警觉缺失** | 复发时交易者未启动防错机制，甚至未意识到正在重蹈覆辙 |
| **后果放大** | 复发的错误往往叠加新的情境变量，造成比首次更大的损失 |

## 为什么「旧错误复发」比新错误更致命

1. **知识无效化**：它证明 Wiki 的知识积累没有转化为执行层的保护
2. **自我怀疑**：反复犯同样错误会侵蚀“我能改进”的信念
3. **防错机制失效**：表明现有的纪律约束不足以在关键时刻接管行为

## 防复发机制设计（待补全后验证）

为防止旧错误再次复发，需要从三个层次建立防线：

| 层次 | 措施 |
|------|------|
| **规则层** | 开仓 checklist 前置，单板块仓位硬上限 |
| **监控层** | 每日复盘时强制检查是否触发已知错误模式 |
| **惩罚层** | 一旦确认复发，强制空仓 1-3 天以打断惯性 |
