# 当前作战室工作页日用示范样例

说明：以下内容只用于演示怎么填写现行作战室工作页，不代表真实交易日判断，不可直接当成实盘结论。

## 基本信息

```yaml
date: T日盘后
market_state: 分歧后修复，允许低频试错，不允许无逻辑追高
position_permission: 轻中仓
data_gap: 无龙虾辅助竞价数据，需靠盘中强弱二次确认
primary_mode: 分歧转强后的前排确认
```

## 今日结论

```text
今天不是全面进攻日，而是修复确认日。可做，但只做最强方向里最强一只，且必须等分歧后重新转强。主计划有效，备选全部降级观察。
```

## 市场环境

```text
指数层面没有明显系统性风险，但短线资金偏谨慎。高位票分化，低位新题材开始争夺资金。适合聚焦“有催化、有承接、有辨识度”的前排，不适合看到拉升就追。
```

## 主计划

```text
主计划：只做示例前排A的分歧转强回封。
触发条件：早盘分歧不破关键承接，回封时板块同步走强。
禁止条件：板块核心中军走弱、风险锚转负、示例前排A缩量硬拉无承接。
退出条件：首次回封失败且承接明显转弱；或板块核心中军掉头。
仓位：试错仓，确认后再考虑加。
实际结果：触发一次，给到低吸+回封确认机会；未触发重仓条件。
```

## 备选计划

```text
备选一：示例中军B，只看承接，不主动追。
备选二：老题材反抽票C，只做观察，不执行。
盘后处理：备选一保留到明日观察；备选二删除，不再占计划位。
```

## 当前数据缺口

```text
竞价辅助数据缺失，所以早盘判断更多依赖开盘后5-15分钟的承接和板块共振。该缺口没有阻止判断，但降低了先手仓位上限。
```
