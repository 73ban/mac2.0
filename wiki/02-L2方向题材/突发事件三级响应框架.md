# 突发事件三级响应框架

> 把"看情况处理"升级为"触发即执行"——一套针对新闻冲击的分层响应系统，保护趋势底仓，防止情绪交易。

## 为什么需要这个框架

当前的交易系统主要覆盖**市场内生信号**：[[概念/华泰拥挤度框架|拥挤度]]监控情绪热度，[[概念/ROE-TTM高位拐点|ROE拐点]]追踪盈利能力，[[概念/分歧日判断框架|分歧日]]处理多空博弈。但对外部冲击——突发政策、地缘冲突、黑天鹅——的处理分散在各页面中，缺乏统一的触发机制和执行流程。

**这个框架解决的核心问题**：

> 当新闻突发时，是该"不动"还是"行动"？如果要行动，怎么行动才不会破坏原有的趋势持仓逻辑？

答案是：**分层响应**。不是所有冲击都需要行动，该扛的不动，该逃的果断。

## 三级阈值定义

根据Schwab、SGT Markets等机构的研究，将市场冲击事件按三个维度评估：**冲击力**、**持续性**、**可预见性**。

### 1级：计划内冲击 — 维持原有框架

| 维度 | 特征 |
|------|------|
| **典型事件** | 经济数据公布（CPI/PMI/非农）、央行利率决议、财报发布、常规政策会议 |
| **市场反应** | 瞬时波动后回归基本面，核心在于**实际值vs预期值**的偏离 |
| **可预见性** | 高——时间、内容均可预期 |
| **持续性** | 低——通常日内消化完毕 |

**触发动作**：不进入任何特殊模式。继续执行原有计划（趋势底仓不动，短线按原规则运行），仅监控[[概念/分歧日判断框架]]是否因预期差而演变为分歧日。

### 2级：意外扰动 — 启用快节奏观察模式

| 维度 | 特征 |
|------|------|
| **典型事件** | 突发政策讲话（如关税调整）、局部军事冲突、行业监管变化、关键人物事件 |
| **市场反应** | 冲高或杀跌脉冲明显（ATR突破2倍均值），但持续性通常不强 |
| **可预见性** | 低到零——几乎无法提前预判 |
| **持续性** | 中——可能持续数小时到1-2个交易日 |

**触发动作**：启用[[概念/快节奏观察模式]]。核心操作：停止新开仓位 + 原有仓位挂止损/移损 + 等待15-30分钟稳定期 + 限价单替代市价单 + 当日短线仓位必须尾盘前平掉。

### 3级：系统性颠覆 — 全面防御

| 维度 | 特征 |
|------|------|
| **典型事件** | 大规模战争爆发、贸易体系崩溃、流动性危机、系统性金融风险 |
| **市场反应** | 可引发[[概念/风格撕裂]]甚至触发[[概念/抱团崩塌四条件]] |
| **可预见性** | 极低 |
| **持续性** | 极长——可能改变市场结构 |

**触发动作**：启用全面防御模式。立即对照[[概念/抱团崩塌四条件]]量化评分卡逐项检查。若崩塌条件触发——果断降低中期仓位，执行净资产三档仓位管理中的减仓纪律。若崩塌条件未触发——趋势底仓不动，暂停所有短线操作，以保护本金为首要任务。

## 阈值确认的判断标准

进入2级或3级的判断，不是靠"感觉冲击很大"，而是靠三个**可观测信号**的组合：

1. **量价异常扩大**：5分钟成交量突破过去20日均量的300%
2. **波动率显著攀升**：ATR(14)突破过去5日均值的2倍
3. **关键支撑/阻力瞬间突破**：价格穿透近期震荡区间上下沿

**三者齐备 → 确认2级起步。其中波动率突破3倍ATR或出现流动性枯竭迹象 → 升级至3级评估。**

> ⚠️ 在[[概念/华泰拥挤度框架|拥挤度]]极高时，利空冲击更容易引发踩踏，响应级别应提升。拥挤度极低时，利空冲击可能被快速消化，响应级别可下调。

## 核心操作纪律：多时间框架隔离

这是本框架中最关键的铁律：

> **快节奏模式仅作用于日内。2级模式下开立的任何短线仓位，必须在当日尾盘前平掉，不得越级进入趋势持仓池。**

这堵住了"短线情绪决策污染中长期持仓逻辑"的漏洞。与[[概念/交易脱钩原则]]（历史成本不绑架新决策）形成完整的时间-空间双维隔离：
- **交易脱钩原则** → 纵向隔离：昨天亏了≠今天不能买
- **多时间框架隔离** → 横向隔离：日内短线≠趋势中线

## 趋势底仓"不可交易期"

3级系统性事件是唯一可以动趋势底仓的场景，且必须走[[概念/抱团崩塌四条件]]的完整检查流程。2级意外扰动中，**趋势底仓不得动用**——这是保护中长期逻辑不被短期噪音干扰的机制性防火墙。

允许的例外：用浮动仓位（不超过总持仓的20%）参与快节奏对冲，对冲仓位同样遵守"当日尾盘平掉"规则。

## 与现有系统的分界

### 与分歧日判断框架的分界

| 维度 | 分歧日 | 突发事件 |
|------|--------|----------|
| 波动来源 | 内生的多空分歧 | 外生的新闻冲击 |
| 信号性质 | 分歧后的转向 | 冲击后的稳定 |
| 等待期 | 卖出看下午 | 15-30分钟稳定期 |
| 适用框架 | [[概念/分歧日判断框架]] | 本框架 |

**核心区分**：突发事件期间的"波动分歧"≠内生"分歧日分歧"。突发冲击的15-30分钟等待期是过滤脉冲噪音，分歧日的"卖出看下午"是等待多空博弈结果。两种场景不同，不可混淆。

### 与再参与决策框架的分界

突发事件期间，[[概念/再参与决策框架]]**暂停执行**。再参与框架处理的是"踏空/清仓后如何回到仓位"，而突发事件场景下，系统优先服从本框架的"不可交易期"和"趋势底仓不动"规则。

## 突发事件监控仪表盘

| 指标 | 用途 | 阈值 |
|------|------|------|
| ATR(14) | 波动率监测 | >5日均值2倍→2级观察；>3倍→评估3级 |
| 5分钟成交量 | 算法冲击识别 | >20日均量300% → 2级触发信号 |
| 博林格带宽 | 波动扩张可视化 | 带宽突破+放量 → 区分突破/脉冲 |
| RSI/MACD背离 | 反转预警（非交易信号） | 出现背离 → "不追高"提醒，不视为反转确认 |
| 订单流（LuxAlgo等） | 机构资金验证 | 大单流向与新闻方向相反 → 警惕陷阱 |

**参数状态**：所有阈值均为初始建议值，**未经过个人实盘压力测试校准**。需要收集3-5次历史突发事件数据进行回测后调整。

## 事后审计清单

每次触发2级及以上响应后，必须在复盘时完成：

1. [ ] 当日操作是否与盘前"突发事件响应清单"一致？
2. [ ] 是否有短线仓位被错误移入趋势持仓池？
3. [ ] 趋势底仓是否在未触发崩塌条件的情况下被动了？
4. [ ] 对比"完全不动"的假设结果，操作是否增值？
5. [ ] 15-30分钟等待期是否被尊重？有几次？
6. [ ] 纳入[[进化/纪律保质期问题]]保质期检查

## 已知局限（有意取舍）

1. **15-30分钟等待期可能错过逆转后的趋势**——这是**有意牺牲**，非框架缺陷。对于以中长期趋势为核心的系统，错过一次微观反弹 < 被脉冲带进坑里。
2. **限价单在流动性枯竭时可能无法成交**——框架接受"无法参与"的结果。2级模式的根本目标是保护趋势底仓，而非必须抓住每次日内机会。
3. **频率限制**：一周内进入快节奏模式超过2次 → 不是框架问题，是市场本身极度不明，应缩仓。

## 下一步

- 将ATR、成交量突发指标固化为交易软件面板
- 收集3-5次历史重大突发事件（如2024年关税冲击、地缘事件等）做压力测试回测
- 回测后校准各项阈值参数
