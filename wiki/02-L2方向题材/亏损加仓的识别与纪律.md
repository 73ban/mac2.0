# 加仓的确定性纪律

## 核心原则

**加不加仓，看确定性有没有强化——不看你现在是赚还是亏。**

浮盈状态下没有新逻辑 → 不加。浮亏状态下逻辑强化、确定性提升 → 可以加。盈亏本身不是加仓的判断标准。

## 加仓的三个硬条件

满足任意一条即可考虑加仓，三条都满足 = 确定性极高的加仓点：

1. **新增强催化**：出现之前没有的重大利好（超预期订单、板块级别政策、龙头地位确认）
2. **盘面超预期**：分歧后强势回封、板块共振、资金回流确认
3. **确定性升级**：从"大概率"变成"几乎确定"——比如唯一龙头确认、板块全面发酵

## 禁止加仓的唯一场景

**不是因为亏损，而是因为锚定。**

- 加仓理由如果是"降低平均成本" → 禁止。这是在用新钱掩盖旧错。
- 加仓理由如果是"便宜了再买点" → 禁止。便宜不是逻辑。
- 加仓后总仓位超过确定性等级对应的上限 → 禁止。

## 2026-05-28 深科技案例

- 持仓成本44.87，现价42.65，浮亏-5.2%
- 操作：买入2600股→卖出1700股，净增900股
- **问题不在"浮亏加仓"，而在"没有新逻辑"**——当天没有出现新的强催化，加仓只是锚定在历史成本上的摊平行为
- 正确做法：要么在有新逻辑时重仓，要么在没有新逻辑时不动

## 加仓检查清单

每次加仓前回答：
- [ ] 这笔加仓的逻辑是什么？和首次买入的逻辑有本质变化吗？
- [ ] 仓位加完后，总仓位是否在确定性等级允许范围？
- [ ] 我加仓是因为逻辑强化了，还是因为"想摊低成本"？
- [ ] 如果现在没有持仓，我还会在这个位置买入吗？

> **最后一条是关键——如果答案是否定的，就不要加。**
