# 源文档格式规范

定义从原始资料（公众号、网页等）转换后进入知识库管道的 Markdown 文件应遵守的统一格式，确保下游解析稳定。

## 规范要求

### 1. 文件命名
- 保持与原始标题一致，或使用时间戳命名，便于检索。
- 扩展名统一为 `.md`。

### 2. 文件头部（必须）
每个源文档 MD 文件顶部应包含以下行：
```
# 原始标题
> 来源: 作者|公众号名或其他来源标识
---
```
- 转换脚本（如 [[批量HTML转Markdown脚本]]）会自动生成这两行。
- 若手动创建，需严格遵循此格式，因为后续 LLM 摄入可能依赖 `# 标题` 和 `> 来源:` 模式提取元数据。

### 3. 内容清洗
- 移除一切与内容无关的富文本标记（如公众号专用标签、隐藏样式）。
- 图片使用标准 Markdown 语法：`![](url)`，若无法获取有效 alt 文本，暂空。
- 超链接保留文字和 URL 以 Markdown 格式呈现。

### 4. 编码
- 推荐 UTF-8 无 BOM（当前脚本设置），以兼容 Linux 工具链。

## 作用

- 保证 [[数据摄入管道]] 的可靠性，减少因格式变异导致的解析失败。
- 使 [[数据管道健康检查]] 可基于格式合规率进行监控。
