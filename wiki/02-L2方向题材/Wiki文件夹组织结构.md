# Wiki文件夹组织结构

## 定义

**Wiki文件夹组织结构**是将交易知识管理系统中的所有文件，按其**内容类型**（是什么）和**调用用途**（什么时候用）分配到10个功能文件夹的分层分类体系。它是知识资产的索引基础设施，确保知识可维护、可查找、可调用。

## 核心原则

### 1. 内容类型决定归属

每个文件按其"是什么"归类——是认知概念？是股票速查？是错误记录？还是外部来源？

### 2. 调用用途决定层级

文件夹在导航中的优先级取决于其**复用频率**：

| 调用频率 | 文件夹 | 场景 |
|:---|:---|:---|
| 高频 | 概念/ | 每次交易决策前调用认知模块 |
| 中频 | 股票/、模式/、策略/ | 选股、操作、仓位管理时调用 |
| 低频 | 错误/、来源/、总结/、进化/ | 复盘、系统升级时调用 |
| 待扩展 | 数据/ | 量化监控（未来可能升级为高频） |

### 3. 有灰度的边界用链接桥接

当文件在多个文件夹都合理时（如T+1锁仓既是错误也是操作规则），不做强行归属，而是用跨文件夹的链接建立桥接。

## 10文件夹体系

```
wiki/
├── 概念/          # 55+｜可复用的认知模块｜系统引擎
├── 股票/          # ~25｜持仓与候选速查｜快速工具箱
├── 个股档案/      # 7  ｜已完成深度分析｜深度存档
├── 分析/          # 5  ｜专题与市场研究｜方向判断
├── 模式/          # 3  ｜可执行操作规则｜做什么
├── 策略/          # 2  ｜仓位与资金管理｜开仓约束
├── 错误/          # 5  ｜已识别教训库｜避免什么
├── 来源/          # ~16｜外部文档归档｜溯源支撑
├── 数据/          # 1  ｜量化扫描监控｜定量驱动
├── 进化/          # 3  ｜系统迭代记录｜元认知
└── 总结/          # 2  ｜定期复盘｜跨日专题
```

## 关键设计决策

### 概念/是引擎

新摄入的外部信息，最终归宿是生成新的概念文件或更新已有概念。**概念/的规模直接反映系统的认知深度**。

### 股票/与个股档案/的两层结构

| | 股票/ | 个股档案/ |
|:---|:---|:---|
| 内容深度 | 基础信息+关键信号 | 归因+估值+交易计划 |
| 文件来源 | 日常积累 | 触发深度分析后 |
| 升级条件 | — | 完成完整三项分析 |

### 模式/与错误/的双轨

- 模式/ = 正向操作框架（在什么条件下做什么）
- 错误/ = 反向警示系统（什么条件触发什么错误）
- 两者共同构成个人交易DNA的完整画像

## 使用场景

- **新文件摄入后**：按内容类型自动归入对应文件夹
- **交易前准备**：按频率分层快速定位所需文件
- **系统维护时**：通过进化/日志追踪组织变更
- **知识审计时**：检查来源/→概念/的转化率
