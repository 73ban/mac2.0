# 持仓逻辑断裂

## 定义

**持仓逻辑断裂**是指当连续多个月的交易数据缺失时，持仓的演变路径——从建仓逻辑、加减仓动机到清仓决策——的因果链完全中断，导致无法追溯错误根源或验证策略有效性。

## 典型表现

| 症状 | 示例 |
|------|------|
| 标的A在某月出现，但无建仓记录 | 5/15持仓快照出现数据港/美利云，无4月建仓数据 |
| 标的B在某月买入，后续无卖出记录 | 3/6融资买入兆易创新，4‑5月全无数据 |
| 持仓线切换无迹可寻 | 从存储线到IDC线的切换没有中间记录 |
| 模式漂移无法定位起点 | 5/22诊断为模式漂移，但不知是4月还是5月开始 |

## 为什么危险

1. **错误无法追溯**：不知道亏损是建仓逻辑错误、持仓管理错误、还是卖出时机错误
2. **诊断沦为猜测**：5/22的精复诊断缺乏前因，如同分析电影的最后一幕
3. **重复错误无法识别**：如果4月已经犯过某个错误但无记录，到5月重复时无法预警
4. **策略优化失去基础**：无法区分“策略本身错误”还是“执行偏差”

## 4月案例：断裂最严重的一环

- 3/6 → 4月 → 5/15 之间存在 **46天连续空白**
- 持仓从存储线变为IDC线，逻辑完全不可追溯
- 这是3‑5月66天缺失中最关键的断裂点

## 预防措施

- 建立**每周至少导出一次交割单**的固定流程
- 设置日历重复提醒（如每周日导出本周交割单）
- 即使数据不完整（部分委托缺失），也比完全空白强
- 考虑自动化方案：券商API导出 → 自动归档
