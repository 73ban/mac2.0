# 产业链扩散逻辑

## 定义

产业链扩散逻辑是指在AI算力等主线行情中，行情从核心环节逐步向外扩散的路径和规律。理解扩散逻辑有助于识别行情的阶段性、判断跟风标的的参与时机和风险。

## Rubin链完整扩散路径

```
GPU/ASIC芯片
  ├── 光模块（800G→1.6T）         ← 核心环节
  │     ├── 光芯片（EML/CW）
  │     └── 磷化铟衬底
  ├── PCB（高阶HDI）                ← 中军环节
  ├── MLCC（高端化→01005/008004）   ← 被动元件
  ├── 设备（封装/测试）              ← 资本开支
  ├── 液冷散热（GB300认证）          ← 必选环节
  └── 存储（HBM→HBF）               ← 远期叙事
```

## 🆕 代际切换：扩散过程中的最大不确定性过滤机制

产业链扩散不是均匀的、线性的。每一次技术平台代际切换（如B100→B200→GB300→Rubin），都会对扩散路径上的所有供应商进行一次**重新洗牌**：

- **已卡位者**：扩散中的每一轮行情都会强化其地位
- **卡位滞后或悬案者**：在旧代际中的"存在感"可能只是暂时的，新代际切换后将面临淘汰

**平台认证代际是扩散过程中最核心的筛选机制**。

## 扩散节奏的监控

| 扩散阶段 | 信号 | 操作含义 |
|---------|------|---------|
| 核心环节先行 | 光模块一线龙头领涨 | 参与一线，不追二线 |
| 中军跟进 | PCB环节放量 | 趋势确认，可补仓 |
| 弱扩散 | MLCC/被动元件补涨 | 扩散末端，谨慎跟风 |
| 叙事扩散 | HBF等远期概念活跃 | 资金溢出，不参与 |
