# 冷板式液冷系统架构

## 定义

冷板式液冷系统架构是由**CDU（冷液分配单元）、Manifold（歧管/分水器）、快接头（UQD）、[[冷板]]、管路**组成的系统级热管理方案。其核心不在单个组件的性能，而在于各组件间的**流量匹配、流阻优化和系统协同**。

## 系统组成

```
┌─────────────────────────────────────┐
│                外冷源                │ ← 冷却塔/干冷器
└──────────────┬──────────────────────┘
               │
        ┌──────▼──────┐
        │     CDU     │ ← 泵力+热交换+控制
        └──────┬──────┘
               │
        ┌──────▼──────────┐
        │    Manifold     │ ← 流量分配
        └──┬──────────┬───┘
           │          │
      ┌────▼───┐  ┌──▼─────┐
      │快接头  │  │快接头  │ ← 盲插连接
      └────┬───┘  └──┬─────┘
           │          │
      ┌────▼───┐  ┌──▼─────┐
      │ 冷板   │  │ 冷板   │ ← GPU/CPU贴合
      └────────┘  └────────┘
```

## 全链条整合的技术壁垒

> **能同时做CDU + 冷板 + Manifold + 快接头 ≠ 简单叠加**

系统级优化的核心在于：

1. **流量-流阻匹配**：CDU的泵力特性必须与冷板的内部流阻精确匹配。若冷板流阻过高，CDU泵力不足则散热失效；若流阻过低则流量不均。
2. **多支路平衡**：Manifold的分配逻辑需根据各冷板的实际流阻特性设计，避免"短路"（某支路流量过大而另一支路不足）
3. **快接头兼容性**：快接头的口径和流阻曲线必须纳入全系统CFD仿真，否则会成为系统瓶颈

**拼凑供应商的死结**：若CDU来自A、冷板来自B、快接头来自C，各厂商只保证自己组件的性能，系统级流阻曲线的匹配成为集成商的责任——而集成商通常缺乏跨组件的热管理设计能力。

## 英维克的全链条优势

[[股票/英维克-002837]]是国内**唯一**同时覆盖上述所有环节的企业，这使得其：

- 可在设计CDU时同步优化冷板微通道，达到系统级效率最大化
- 对客户提供"单点责任"（单一供应商解决兼容性问题）
- 在英伟达GB300等高端项目中，能从冷板环节切入，连带锁定CDU和管路
