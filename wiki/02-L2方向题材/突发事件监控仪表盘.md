# 突发事件监控仪表盘

> 从"靠感觉判断冲击力"到"可量化触发响应"，将ATR、成交量突发、布林带宽等指标组合为一个监控面板。

## 仪表盘布局

在交易软件中维持一个独立面板（如TradingView多图表布局），包含以下四项核心指标：

```
┌──────────────────────────────────────────────┐
│  ATR(14)            │  博林格带宽              │
│  当前值 vs 5日均值  │  带宽扩张方向+力度       │
│  阈值线：2x / 3x   │  配合成交量解读          │
├────────────────────┼──────────────────────────┤
│  5分钟成交量        │  订单流（如有）           │
│  当前K线 vs 20日均量│  大单净流入/流出方向      │
│  阈值线：300%      │  与新闻方向的一致性       │
└──────────────────────────────────────────────┘
```

## 各指标阈值详解

### 1. ATR(14) — 波动率温度计

| 条件 | 含义 | 动作 |
|------|------|------|
| ATR < 5日均值1.5倍 | 正常波动 | 无特殊响应 |
| ATR突破5日均值2倍 | 显著冲击 | 进入快节奏观察模式 |
| ATR突破5日均值3倍 | 极端波动 | 评估是否升级至3级防御 |

**使用方法**：在K线图上叠加ATR(14)指标，同时显示5日均值线作为基准。当ATR向上穿越2倍线时，触发预警。

> ⚠️ ATR(14)的参数14和5日均值是行业内常见经验值，需要在压力测试中校准。不同品种的ATR敏感度不同——个人主要操作的中小盘标的可能天然ATR偏高，需要相对化（比率而非绝对值）。

### 2. 5分钟成交量突发 — 算法冲击识别

| 条件 | 含义 | 动作 |
|------|------|------|
| 5分钟量 < 20日均量200% | 正常/温和放量 | 无特殊响应 |
| 5分钟量 > 20日均量300% + 价格突破 | 算法/机构大单冲击 | 确认2级触发的充分条件之一 |
| 5分钟量 > 20日均量500% + 价格剧烈波动 | 可能流动性枯竭 | 暂缓所有计划，等待确认 |

**计算方法**：
- 计算过去20个交易日的5分钟K线成交量均值
- 当前5分钟K线成交量 / 该均值 → 百分比

### 3. 博林格带宽 — 波动扩张可视化

| 条件 | 含义 | 应对 |
|------|------|------|
| 带宽扩张+价格触及上轨+缩量 | 假突破概率高 | 不追，等待回档 |
| 带宽扩张+价格突破上轨+持续放量 | 可能转为趋势爆发 | 单独评估，不自动视为2级冲击 |
| 带宽急收 | 波动收敛，冲击消化 | 快节奏模式退出信号之一 |

**与成交量配合**是关键——带宽扩张+放量=趋势性突破，带宽扩张+缩量=脉冲概率高。

### 4. 订单流（可选，LuxAlgo或类似工具）

| 条件 | 含义 | 动作 |
|------|------|------|
| 大单净流向与新闻方向一致 | 新闻被机构认可 | 评估冲击持续性 |
| 大单净流向与新闻方向相反 | 存在陷阱概率 | **暂停所有操作** |
| 无明显大单方向 | 散户驱动为主 | 回归技术面判断 |

**注意**：订单流工具需要付费或特定平台。如果无法获取，则仅依赖前三项指标。不要用不熟悉的工具做决策。

## 响应触发流程

```
        触发信号？
    ┌──────┼──────┐
   ATR2x  量300%  区间突破
    └──────┼──────┘
       三项齐备？
     ├─ 是 → 确认2级 → 启动快节奏观察模式
     └─ 否 → 继续监控
             ↓
        ATR是否突破3x 且 流动性枯竭迹象？
         ├─ 是 → 评估3级 → 崩塌四条件检查
         └─ 否 → 维持2级响应
```

## 参数校准计划

所有阈值均为**初始建议值**（未经过个人实盘验证）：

| 参数 | 当前值 | 校准方法 | 预期校准窗口 |
|------|--------|----------|-------------|
| ATR倍数（2级） | 2x | 回测3-5次历史冲击，调整至命中率80%以上 | 回测后 |
| ATR倍数（3级） | 3x | 同上 | 回测后 |
| 成交量阈值 | 300% | 统计个人操作标的的放量分布 | 收集20次以上数据 |
| 稳定期时长 | 15-30分钟 | 测试不同时长下的假突破过滤率 | 对比分析 |
| 量回落比例 | 70% | 实际测试有效性 | 收集实盘数据 |

## 局限性

- 所有阈值在极端事件中可能被一次性冲破（如2020年疫情冲击），此时应跳过2级直接进入3级评估
- ATR是滞后指标，对大单冲击的反应有几根K线的延迟
- 订单流工具对散户可能不经济，前三项指标为最低可行版本
