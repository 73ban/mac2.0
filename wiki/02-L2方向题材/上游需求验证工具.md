# 上游需求验证工具

## 一句话定义

上游需求验证工具是指在分析国内某个产业链环节的景气度时，使用**独立于A股市场的、处于同一产业链上游或相关位置的外部实体数据**来交叉验证判断的方法论。

## 概念来源

该概念源于对[[entities/ncab-group|NCAB Group]]的研究。NCAB作为全球最大PCB贸易商，其月度供应链展望和价格指数为判断国内PCB环节（如[[股票/深南电路-002916]]、[[股票/沪电股份-002463]]）的景气度提供了一个**独立、高时效性**的外部视角。

## 核心原则

### 1. 独立性
验证工具必须独立于A股市场情绪。NCAB关注的是全球PCB供需，不受A股投机情绪影响，因此其行业判断具有更高的客观性。

### 2. 前置性
验证工具通常处于产业链更上游，其变化先于终端产品表现出来。成本端压力（如NCAB指数上涨）可能在1-2个季度后才反映在国内PCB厂商的财报中。

### 3. 唯一性
验证工具提供的是A股自身指标无法提供的信息维度。拥挤度、ROE等指标反映的是A股投资者的行为和上市公司财务，而上游验证工具反映的是**全球产业供需本身**。

## 如何使用

| 步骤 | 操作 |
|------|------|
| 1. 识别工具 | 在产业链上游找到公开可追踪的独立实体（如NCAB） |
| 2. 追踪数据 | 保持对其定期报告/指数的关注 |
| 3. 理解局限 | 明确该工具与A股标的之间的市场差异（如[[概念/HMLV高混合低量|HMLV vs 大批量AI基板]]） |
| 4. 交叉验证 | 将其数据与拥挤度、ROE、抱团崩塌评分卡等进行交叉验证 |
| 5. 方向判断 | 关注趋势方向而非绝对值，用于确认或质疑现有判断 |

## 使用边界

1. **不做独立决策**：单靠上游验证工具不应成为买卖决策的唯一依据
2. **注意传导时滞**：上游变化传导至A股公司基本面存在时间差
3. **确认市场差异**：必须明确验证工具的市场与A股标的的市场之间的差异程度
4. **定期校准**：验证工具的有效性需要定期重新评估

## 与现有监控框架的整合

NCAB数据可以作为一个**补充维度**嵌入以下现有框架：

- [[概念/中军不倒]]：增加"行业级景气度验证"维度
- [[概念/抱团崩塌四条件]]量化评分卡：可增加外部供需数据作为补充指标
- [[概念/华泰拥挤度框架]]：与拥挤度形成"全球产业基本面 vs A股投机情绪"对比
- [[概念/ROE-TTM高位拐点]]：NCAB价格指数的环比拐点可作为ROE拐点的先行指标

## 其他潜在验证工具

该方法论可以推广到其他产业链环节：
- **存储芯片**：跟踪全球DRAM/NAND现货价格指数
- **被动元件**：跟踪MLCC、电阻等交期数据
- **光模块**：跟踪LightCounting等第三方市场数据
- **新能源金属**：跟踪锂、钴等期货/现货价格
