# 趋势行情卖早与不敢追回循环

> 卖飞→踏空→不敢追回→继续踏空→终于追入时已是高点。完整循环模式与破解工具。🆕 趋势仓位"不可交易期"原则——突发事件中，不动底仓本身就是破解循环的机制性保护。

## 循环模式

```
趋势初期买入 → 小幅盈利后卖出（卖早）
    ↓
股价继续上涨 → 产生踏空感
    ↓
因卖飞而产生"不甘心追回"心理
    ↓
股价继续上涨 → 踏空感加剧
    ↓
终于在高位追入 → 回调即陷入亏损
    ↓
亏损后对趋势失去信心 → 再次卖早/止损
    ↓
循环重启
```

## 破解工具

1. **分批建仓+分批止盈** —— 永远不全仓进出，总有"锚"在仓位里
2. **机构估值锚定** —— 用[[概念/机构估值锚定]]量化判断是否真的"贵了"
3. **再参与决策框架** —— 用[[概念/再参与决策框架]]标准化再入场

## 🆕 趋势仓位不可交易期

这是[[概念/突发事件三级响应框架]]中为趋势持仓者设计的核心保护机制。

**规则**：
- 在2级意外扰动模式下，趋势底仓处于"不可交易期"
- 非3级系统性事件、非[[概念/抱团崩塌四条件]]触发 → 不许动趋势底仓
- 即使日间波动巨大，看起来"应该先跑再买回来"——不许动
- 允许的例外：用浮动仓位（不超过20%）做快节奏对冲，但必须在尾盘前平掉

**为什么这能防止卖飞？**
- 趋势票卖飞最常见的触发场景就是**突发利空冲击下的恐慌卖出**
- "不可交易期"用制度堵住了这个漏洞——不是靠意志力，而是靠规则
- 15-30分钟的等待期足以让噪音消退，让趋势逻辑重新浮出水面

**与现有破解工具的协同**：
1. 突发事件 → "不可交易期"防止恐慌卖飞（机制层）
2. 正常波动 → 分批建仓/止盈（策略层）
3. 已卖飞 → [[概念/再参与决策框架]]（修复层）

三层层层递进，从预防到修复形成完整闭环。
