# PUE与TUE——数据中心能效核心指标

> 用于评估液冷散热赛道经济价值的核心能效度量框架，填补[[概念/液冷散热赛道]]在技术指标层面的认知空白。

---

## 一、PUE（Power Usage Effectiveness）

### 定义
```
PUE = 总设施能耗 / IT设备能耗
```
由The Green Grid提出，已纳入ISO/IEC 30134-2国际标准。

### 行业基准
| 来源 | PUE |
|------|-----|
| Google全球机群TTM | 1.12 |
| Google全球机群季度 | 1.11 |
| AWS机群级 | ~1.15 |
| 先进液冷设施 | 可接近1.05 |

### ⚠️ 三大局限性（对液冷最关键）

**1. 液冷低估悖论** ← 本页面最大发现
```
服务器风扇能耗↓ → IT设备总能耗（分母）↓ 
→ 虽然分子也↓，但分母下降幅度比例更大
→ PUE反而可能上升
→ PUE系统性低估液冷的真实节能效果
```
Eaton直接断言："液冷正在淘汰PUE这一行业指标"。

**2. 不可用于跨数据中心横向比较**
- 负载类型、气候、测量方法不同
- 多数厂商使用建模数据而非实测数据

**3. 不覆盖IT设备内部损耗**
- 服务器内部风扇、内部电源转换损耗均不捕获

---

## 二、TUE（Total Power Usage Effectiveness）

### 定义
```
TUE = PUE × ITUE

ITUE = IT设备总输入能耗 / 计算元件实际消耗的可用能量
```

由Intel Patterson等人于2011年在EEHPC工作组提出。

### TUE vs PUE

| | PUE | TUE |
|------|------|------|
| 度量范围 | 市电→IT设备输入 | 市电→芯片级计算元件 |
| IT内部风扇 | ❌ 不包含 | ✅ 包含 |
| 冷却策略变化 | 可能反向扭曲 | 完整反映 |

### 对液冷赛道的意义
- 液冷替代风扇→ITUE显著改善→TUE完整捕捉
- 云厂客户将越来越关注TUE而非仅看PUE
- **云厂主动披露TUE是液冷加速的催化剂信号**

---

## 三、液冷低估悖论的交易含义

### 这是理解"强逻辑为何未充分定价"的关键线索

**逻辑链**：
```
GPU功耗密度↑ → 液冷需求↑
→ 服务器风扇能耗↓ → ITUE改善 → TUE改善
→ 真实能效提升
→ 但PUE可能因分母效应而"看起来"没改善/恶化
→ 市场缺乏完整的量化信号 → 可能低估液冷价值
```

### 监控信号
- **做多信号**：当云厂（Google/Meta/AWS）开始主动披露TUE/ITUE数据
- **做空信号**：当液冷厂商的投资者沟通仅依赖PUE数据（说明其对真实价值主张尚未建立完整框架）

## 五、实操应用

### 用于英维克/高澜的产品验证
监控这些公司是否：
- 在IR材料中披露PUE数据？测算标准是否公开？
- 是否开始使用TUE/ITUE来量化产品价值？
- 其客户（云厂）是否用TUE评估液冷产品？

### 用于液冷赛道催化剂监控
| 催化信号 | 强度 |
|----------|------|
| OCP发布液冷能效测试标准白皮书 | 中等 |
| Google/Meta年度报告首次披露TUE | ⭐ 强 |
| TUE进入ISO标准化流程 | ⭐ 强 |
| 液冷厂商停止仅引PUE，开始引用TUE | 边缘信号 |
