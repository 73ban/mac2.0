# 分级减仓触发体系

## 定义

将单一减仓决策拆解为**四个层级**，不同层级的警告信号对应不同幅度的仓位调整和执行节奏，避免"要么全仓要么清仓"的二元决策陷阱。

## 核心理念

此前所有监控框架（抱团崩塌、ROE拐点、缩量蓄力、中军不倒、分歧转一致）都是**诊断工具**——告诉你"出问题了"。

分级减仓触发体系是**处方**——告诉你"出了什么问题、该减多少、减哪个"。

## 四层架构

```
L1 宏观崩塌 → 战略减仓（降至≤30%）→ 无条件优先
L2 中军破位 → 方向减仓（该方向50%或全清）
L3 龙头退潮 → 短线清仓（该龙头全清）
L4 个股止盈 → 逐笔退出（动态止损止盈）
```

## 为什么需要分层

- **不同信号的严重程度不同**：风格撕裂反转 ≠ 单只股票破位 ≠ 龙头断板
- **避免过度反应**：L3信号出现时不必清仓L2中军
- **保留观察窗口**：L2触发后仍保留中军仓位作为行情延续的观察窗口

## 卖出顺序：先高波动后低波动

1. 短线情绪仓（龙头、连板）→ 流动性收缩时优先变现
2. 跟风波段仓（二线跟风）→ 逻辑支撑最弱
3. 中军方向仓 → 流动性最好，最后撤退

## 边界与限制

- **L1无条件优先**：宏观崩塌时不做个股幻想
- **所有量化阈值为初始设定**：需至少一轮完整行情验证
- **流动性调整**：两市成交额<7000亿时换手率阈值需下调
