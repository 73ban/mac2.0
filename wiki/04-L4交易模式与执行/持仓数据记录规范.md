# 持仓数据记录规范（SOP）

> 数据记录的**一致性**，应与**交易纪律**同等重要。

## 目的

本规范旨在消除"持仓快照缺失"这一系统性漏洞，确保每一次复盘都有完整的持仓数据输入。

## 适用范围

每个交易日收盘后 30 分钟内执行。

## 采集方式（三选一，按优先级排列）

### 方式一：券商软件直接导出（推荐）
1. 登录券商交易软件
2. 进入"持仓查询"或"资金股份"页面
3. 导出为 Excel 或 CSV 文件
4. 保存至 `raw/每日持仓/YYYY-MM-DD-持仓.xlsx`
5. ⚠️ 导出后立即打开验证数据完整性

### 方式二：标准化表格填写
| 字段 | 填写说明 |
|------|---------|
| 日期 | 自动生成 |
| 股票代码 | 6位数字 |
| 股票名称 | 中文简称 |
| 持仓数量 | 股数 |
| 持仓成本 | 元/股 |
| 最新市价 | 收盘价 |
| 浮动盈亏率 | % |
| 仓位占比 | % |
| 所属板块 | 如PCB、AI、消费电子 |

保存至 `raw/每日持仓/YYYY-MM-DD-持仓.md`

### 方式三：全屏截图（最低要求）
- 截取持仓页面完整截图
- 包含所有持仓品种、数量、成本、盈亏
- 保存至 `raw/每日持仓/截图/YYYY-MM-DD.png`
- ⚠️ 截图无法直接数据分析，仅作为备查

## 交接流程

收盘后遵循以下流程：

```
15:00 收盘
  ↓
15:00-15:30 获取当日交割单（自动或手动）
  ↓
15:30-15:45 采集持仓快照（按本规范执行）
  ↓
15:45-16:00 写入日复盘（含持仓状态评估）
  ↓
周末：持仓快照对照交割单交叉验证
```

## 异常处理

| 异常情况 | 处理方式 |
|----------|---------|
| 软件无法导出 | 改用方式二手动填写 |
| 当日忘记记录 | 次日开盘前补录（标注"补录，可能不精确"） |
| 文件为空/损坏 | 立即检查原始数据源，如不可恢复，使用交割单反推估算（标注置信度 D） |

## 2026-05-22 教训

`20260522 我的持仓.xlsx` 为空文件，导致：
- 无法评估当日收盘仓位健康度
- 无法量化 PCB 板块集中度
- [[2026-05-22-交割单复盘]] 的分析价值被削弱

此规范即为防止此类事件复发的制度化手段。**数据基建是复盘大厦的基石**。
