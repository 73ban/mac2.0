# 交易模式选择器

更新时间：2026-06-13

## 用途

这个文件解决一个问题：模式库很多，但每天只能用少数几个。作战室必须先判断市场状态，再从模式卡片库里选择当日可用模式。

模式来源：
```text
wiki/04-L4交易模式与执行/游资交易模式卡片库/
wiki/04-L4交易模式与执行/正式模式升级卡/
wiki/04-L4交易模式与执行/近期有效模式排行榜.md
```

## 总原则

1. 市场状态决定模式，不是看到某只票像某个模式就硬套。
2. 仓位权限由市场环境、模式等级、竞价确认共同决定。
3. 同一天主模式最多 2 个，避免脑子里什么都想做。
4. 观察模式只能验证，不能重仓。
5. 退潮期先保命，模式再好也要降仓。

## 买入前 10 问

每次买入前必须回答：

1. 现在是什么市场状态？
2. 今天主线是什么，还是没有主线？
3. 这只票是龙头、中军、前排、补涨、后排，还是杂毛？
4. 这笔交易对应哪个主模式？
5. 这个模式在今天市场状态下是否允许？
6. 9:15、9:20、9:25 竞价是否确认？
7. 风险锚有没有恶化？
8. 如果买错，退出条件是什么？
9. 仓位为什么是这个数字？
10. 这笔交易是否在作战室计划内？

只要第 4、5、8、10 任意一项答不出来，默认不买。

## 重仓许可条件

重仓不是看情绪，而是看条件同时满足：

| 条件 | 要求 |
|---|---|
| 市场分数 | 8-10 分 |
| 市场状态 | 题材主升、强趋势抱团、强连板接力之一 |
| 模式等级 | A 类或近期有效模式 |
| 股票角色 | 龙头、中军、换手核心、强辨识度前排 |
| 竞价确认 | 9:20 不撤，9:25 符合或超预期 |
| 风险锚 | 不恶化，不能出现高标 A 杀或核心补跌 |
| 退出条件 | 明确到价格、盘口、板块反馈或时间 |

满足 6 项以上才可以讨论重仓。少于 5 项，只能轻仓或标准仓。

## 模式降级规则

| 现象 | 动作 |
|---|---|
| 连续 2 次触发后亏钱 | 降为观察 |
| 连续 3 次无溢价 | 暂停使用 |
| 市场状态切换 | 重新评估，不沿用旧结论 |
| 样本盈利但回撤过大 | 降低仓位权限 |
| 只靠单只票成功 | 不升级为通用模式 |

## 作战室使用方式

作战室每天只写：

```text
今日市场状态：
今日主模式：
备选模式：
禁用模式：
可重仓条件：
必须降仓条件：
```

如果当日无法确定主模式，默认降低仓位。
