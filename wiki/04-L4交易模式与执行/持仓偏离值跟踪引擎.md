# 持仓偏离值跟踪引擎

> 建立：2026-06-09 | 维护：codex
> 
> **核心目的：系统化跟踪每一笔买卖的事后走势，做「卖飞了还是卖对了」的量化复盘。**

---

## 一、跟踪什么

你对最近30天买卖过的每一只票，每天更新：

| 字段 | 含义 | 计算方式 |
|------|------|----------|
| 买入日 | 建仓日期 | 交割单 |
| 买入价 | 建仓均价 | 交割单 |
| 买入理由 | 交易逻辑 | 用户口述 |
| 卖出日 | 清仓日期 | 交割单 |
| 卖出价 | 清仓均价 | 交割单 |
| 卖出理由 | 卖出逻辑 | 用户口述 |
| 持仓盈亏 | 这笔交易赚/亏多少 | (卖出价-买入价)/买入价 |
| D+3收盘价 | 卖出后第3天收盘价 | tdx_kline |
| D+5收盘价 | 卖出后第5天收盘价 | tdx_kline |
| D+10收盘价 | 卖出后第10天收盘价 | tdx_kline |
| D+20收盘价 | 卖出后第20天收盘价 | tdx_kline |
| 卖出后最高价 | 卖出后20天内最高 | tdx_kline |
| 最大卖飞幅度 | 如果多拿能多赚多少 | (最高价-卖出价)/卖出价 |
| 偏离值评分 | -100到+100 | 负=卖飞，正=卖对 |

---

## 二、偏离值评分规则

```
偏离值 = (卖出后D+5收盘价 - 卖出价) / 卖出价 × 100

正分 = 卖对了（卖出后跌了）
负分 = 卖飞了（卖出后涨了）
```

| 偏离值 | 结论 | 行动 |
|--------|------|------|
| **+10以上** | 卖得漂亮，躲过大跌 | 模式验证 ✅ |
| **+3 ~ +10** | 卖出正确 | 保持 |
| **-3 ~ +3** | 中性，无所谓 | 不纠结 |
| **-3 ~ -10** | 轻微卖飞 | 关注卖出时机 |
| **-10 ~ -30** | 明显卖飞 | 复盘卖出理由 |
| **-30以下** | 严重卖飞 | 模式调整 ⚠️ |

## 四、每日更新流程

```
1. 读 raw/偏离值数据库/持仓跟踪.json
2. 对每笔交易：
   a. 如果卖出后不足D+3/D+5/D+10/D+20 → 补查tdx_kline
   b. 更新对应字段
   c. 重新计算偏离值评分
3. 检查是否有新交易 → 追加到数据库
4. 检查是否超过30天 → 归档到月度报告，移出日常跟踪
5. 产出 04-持仓偏离值跟踪.md（当日报告）
```

## 五、月度偏离值报告

每月1号自动产出：
```
raw/偏离值数据库/月度报告/YYYY-MM-月度偏离值报告.md
```

| 指标 | 计算 |
|------|------|
| 本月交易笔数 | count |
| 平均盈亏% | avg(profit_pct) |
| 卖飞率 | 负偏离值笔数 / 总笔数 |
| 平均偏离值 | avg(deviation_score) |
| 最大卖飞 | min(deviation_score) |
| 最佳卖出 | max(deviation_score) |
| 按模式分组偏离值 | group by buy_reason |

**核心作用**：月度偏离值报告告诉你——哪个模式最容易卖飞？哪个卖点最可靠？

## 六、偏离值洞察 → 交易修正

| 偏离值模式 | 诊断 | 修正 |
|-----------|------|------|
| 连续正偏离（卖对） | 卖出时机成熟 | 保持当前卖点 |
| 连续负偏离（卖飞） | 卖得太早 | 延长格局周期，多等1-2天 |
| A类龙头负偏离大 | 龙头格局不够 | 龙头按规则可多格局 |
| B类低吸正偏离大 | 低吸卖点准确 | 模式验证 |
| C类消息催化偏离大 | 消息股持续性判断不足 | 提高消息催化持仓周期判断 |

---

> ⚠️ 偏离值跟踪是每日复盘第4项产出，依赖 tdx_kline 查询历史收盘价。
