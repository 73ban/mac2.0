# 统计看板字段标准-2026-06-13

## 用途

本文件把统计看板、模式验证闭环、错误成本账本统一到同一套字段。以后软件页面、表格、脚本、复盘模板都以本文件为准，避免“复盘写一套、统计看板算另一套”。

核心目标不是做流水账，而是回答：

1. 重仓到底值不值。
2. A+/A 确定性机会是否真的赚钱。
3. 1-3 只集中持仓是否比铺开更好。
4. 低吸、半路、打板、竞价、尾盘哪种方式更适合用户。
5. 哪些模式最近有效，哪些模式必须降权。
6. 最大错误成本来自哪里。

## 一、账户与时间范围字段

| 字段 | 类型 | 说明 |
|---|---|---|
| date | date | 交易日期 |
| account_equity | number | 当日账户权益 |
| start_date | date | 统计开始日期 |
| end_date | date | 统计结束日期 |
| initial_equity | number | 区间期初权益 |
| ending_equity | number | 区间期末权益 |
| net_profit_amount | number | 区间净利润 |
| net_profit_pct | number | 区间收益率 |
| max_drawdown_amount | number | 最大回撤金额 |
| max_drawdown_pct | number | 最大回撤比例 |

规则：
1. 支持自定义日期范围。
2. 盈亏计算不能只看范围内成交，还要能读取范围前形成的持仓成本。
3. 成本计算默认使用全历史交割单 FIFO 成本；统计展示只筛选当前日期范围。

## 三、重仓统计字段

| 指标 | 计算方式 | 用途 |
|---|---|---|
| heavy_trade_count | 重仓交易次数 | 看重仓频率是否过高 |
| heavy_win_rate | 重仓盈利笔数 / 重仓总笔数 | 判断重仓胜率 |
| heavy_profit_amount | 重仓总盈利 | 判断重仓贡献 |
| heavy_avg_profit_pct | 重仓平均收益率 | 判断重仓质量 |
| heavy_max_loss | 重仓最大单笔亏损 | 控制毁灭性亏损 |
| heavy_error_cost | 重仓错误成本合计 | 判断重仓是否乱用 |

重仓定义默认：

```yaml
heavy_position_ratio: 0.30
full_attack_ratio: 0.50
```

可在软件设置里调整，但统计字段不变。

## 四、确定性等级统计

| 等级 | 含义 | 仓位倾向 |
|---|---|---|
| A+ | 市场、题材、模式、个股、竞价全部共振 | 可讨论重仓 |
| A | 主要条件满足，有明确退出 | 标准仓到偏重 |
| B | 有逻辑但确认不足 | 轻仓或观察 |
| C | 试错或噪声 | 不买或极轻仓 |

每个等级统计：

1. 交易次数。
2. 胜率。
3. 总盈亏。
4. 平均盈亏。
5. 最大单笔亏损。
6. 重仓次数。
7. 非计划交易次数。

看板必须能回答：

```text
A+/A 是否真的更赚钱？
B/C 是否还在重仓？
重仓是不是只发生在 A+/A？
```

## 五、持仓集中度统计

| 字段 | 说明 |
|---|---|
| holding_count | 当日持仓数量 |
| top1_position_pct | 第一大持仓占比 |
| top2_position_pct | 前二大持仓占比 |
| top3_position_pct | 前三大持仓占比 |
| over_3_holdings | 是否超过 3 只 |
| concentration_score | 持仓集中度评分 |

默认目标：

```yaml
ideal_holding_count_min: 1
ideal_holding_count_max: 3
```

统计必须比较：

1. 1 只票时的收益和回撤。
2. 2 只票时的收益和回撤。
3. 3 只票时的收益和回撤。
4. 超过 3 只时是否明显降低效率。

## 六、模式与买入方式统计

按 `main_mode` 统计：

| 指标 | 说明 |
|---|---|
| mode_trade_count | 模式交易次数 |
| mode_win_rate | 模式胜率 |
| mode_profit_amount | 模式总盈亏 |
| mode_avg_profit_pct | 模式平均收益 |
| mode_profit_loss_ratio | 盈亏比 |
| mode_max_drawdown | 模式最大回撤 |
| mode_recent_5_result | 最近 5 次表现 |
| mode_status | A类/可用/观察/降级/禁用 |

按 `buy_method` 统计：

```text
低吸
半路
打板
竞价
尾盘
做T
```

每种方式统计胜率、平均盈亏、最大亏损、重仓表现。看板必须能回答“低吸、半路、打板哪种最适合当前用户”。

## 八、错误成本字段

错误成本账本必须与单笔交易字段打通。

| 字段 | 说明 |
|---|---|
| error_id | 错误编号 |
| trade_id | 对应交易编号 |
| error_type | 错误类型 |
| actual_loss | 实际亏损 |
| rule_based_result | 如果按规则执行的结果 |
| error_cost_amount | 错误成本 |
| repeated_error | 是否重复错误 |
| fix_action | 改进动作 |

每周看板必须输出：

1. 最大错误成本。
2. 错误成本最高的错误类型。
3. 是否重复发生。
4. 下周必须禁止的动作。

## 九、交易健康分 0-100

每日自动计算一个交易健康分，不评价赚亏，只评价执行质量。

| 项目 | 分值 |
|---|---:|
| 是否计划内交易 | 20 |
| 是否匹配 L4 模式 | 15 |
| 是否有竞价/盘口确认 | 15 |
| 仓位是否匹配确定性 | 15 |
| 是否持仓 1-3 只 | 10 |
| 是否有退出条件并执行 | 15 |
| 是否避免 S4 信息交易 | 10 |

扣分项：

| 行为 | 扣分 |
|---|---:|
| 非计划交易 | -20 |
| 无退出条件 | -15 |
| 退潮期重仓 | -20 |
| 后排当核心 | -15 |
| S4 信息当依据 | -20 |
| 重复同类错误 | -10 |

## 硬规则

1. 统计看板字段与 `模式验证闭环规则.md` 冲突时，以本文件和闭环规则的新版本为准。
2. 不记录 `main_mode` 的交易，不能进入模式统计。
3. 不记录 `certainty_level` 的交易，不能判断重仓是否合理。
4. 不记录 `planned_trade` 的交易，不能区分能力和冲动。
5. 不记录 `error_cost_amount`，错误库就没有改进价值。
