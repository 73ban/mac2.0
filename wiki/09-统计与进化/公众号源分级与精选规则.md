# 公众号源分级与精选规则

更新时间：2026-06-16

## 目标

把 `raw/05-研报新闻/公众号/` 里的内容分成两层处理：

1. 先识别公众号是不是游资号、工具号、媒体号、普通号。
2. 再从中挑出真正有方法论价值的文章，进入 `data/brain` 和 `wiki`。

## 目录原则

现有 RAW 已按公众号名分目录存放，不强制搬迁老文件。

以后统一按这套分类理解：

```text
raw/05-研报新闻/公众号/
  游资号/
  媒体号/
  工具号/
  普通号/
  待分类/
```

如果来源已经明确属于游资号，只需要在来源名下继续落 RAW，不必反复改历史文件。

## 游资号识别标准

一个公众号可归为游资号，通常满足至少两项：

1. 文章长期围绕超短、题材、龙头、反包、情绪周期。
2. 文章有明确实盘语言，不是纯资讯搬运。
3. 经常总结买点、卖点、仓位、节奏、盘面变化。
4. 文章里能抽出稳定的方法论，而不是只讲新闻。
5. 多次被盘面验证后有参考价值。

## 文章分级

### C 级

只保留在 RAW：

- 纯新闻搬运
- 广告/合集/泛内容
- 题材不清晰
- 没有方法论价值

### B 级

进入 `data/brain`：

- 有明确交易思路
- 能抽出一条规则
- 适合做样本
- 但还没到可直接指导作战室的强度

### A 级

进入 `wiki/09-统计与进化`：

- 规则清晰
- 可复用
- 经过盘面或历史样本验证
- 对次日预判、龙头演化、卖点处理有帮助

### A+ 级

进入作战室规则和候选体系：

- 多次验证有效
- 能直接影响盘前排序、盘中跟随、卖出后回看
- 在不同市场环境下都有稳定收益样本

## 处理流程

```text
公众号RAW -> 识别来源是否游资号 -> 抽方法论 -> 分级 -> 写入 data/brain / wiki -> 反哺作战室
```

## 抽取重点

每篇高质量文章至少抽这 6 项：

1. 适用市场环境
2. 适用题材阶段
3. 适用个股状态
4. 关键触发因子
5. 失效条件
6. 盘前/盘中/盘后对应动作

## 你手工补充的文章

你认为有用的文章，可以直接丢进公众号 RAW 目录。

我负责：

1. 先判断这是不是游资号内容。
2. 再抽规则。
3. 再决定是留 RAW、进 `data/brain`，还是进 `wiki`。

## 约束

1. 不把所有公众号文章都当作有效信号。
2. 不把写得好当成一定有用。
3. 不把一次成功样本当成长期规则。
4. 所有规则必须能被真实盘面验证。
