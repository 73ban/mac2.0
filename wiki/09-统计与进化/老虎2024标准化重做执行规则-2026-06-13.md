5. 旧复盘只能作为线索，不允许照抄旧复盘里的后视镜结论。
6. 原始 RAW 不删、不改、不覆盖。

## 四、交割单怎么做标准化派生

交割单原始 RAW 是券商事实，不重做、不改写、不覆盖。老虎只基于原始 CSV 做三层标准化派生。

### 1. 第一层：交易事实表

文件：

```text
raw/01-交割单/2024-标准化/2024-trades-fact.csv
```

作用：只保存不可争议的成交事实。

每一笔成交必须保留一行，不允许把同日同股多笔成交合并掉。若需要合并，只能另建汇总表。

字段：

```csv
trade_id,source_file,source_row,trade_date,trade_time,account_id,market,code,name,business_type,side,account_type,quantity_raw,quantity_abs,price,turnover,commission,stamp_tax,transfer_fee,other_fee,total_fee,settlement_amount,cash_flow,position_after_raw,remark,source_confidence
```

字段说明：

| 字段 | 规则 |
|---|---|
| `trade_id` | 唯一 ID：`YYYYMMDD-code-side-price-quantity-source_row` |
| `source_file` | 原始 CSV 路径 |
| `source_row` | 原始文件第几行 |
| `trade_date` | 成交日期 |
| `trade_time` | 原表为空就留空，不编 |
| `account_id` | 股东代码，可原样或脱敏，但全表一致 |
| `market` | `SH/SZ/BJ/UNKNOWN`，按代码规则判断，无法判断写 UNKNOWN |
| `business_type` | 原表业务名称，如证券买入、融资买入、证券卖出 |
| `side` | 统一成 `BUY/SELL` |
| `account_type` | `cash/margin/unknown`，融资买入写 margin |
| `quantity_raw` | 原始成交数量，保留正负 |
| `quantity_abs` | 绝对成交数量 |
| `turnover` | 成交金额 |
| `total_fee` | 手续费+印花税+过户费+其他费 |
| `settlement_amount` | 原表发生金额 |
| `cash_flow` | 买入为负，卖出为正，必须与发生金额方向一致 |
| `position_after_raw` | 原表“其他费”字段在部分券商导出里疑似持仓余额时，保留原值并标注，不强行解释 |
| `source_confidence` | 原始券商导出为 high；旧复盘反推为 low |

事实表禁止：

1. 禁止写 L1、主线、模式、错误成本。
2. 禁止合并成交。
3. 禁止补不存在的成交时间。
4. 禁止根据后面盈亏改成交价、数量、金额。
5. 禁止凭复盘文字新增券商交割单里没有的成交。

### 2. 第二层：交易回合表

文件：

```text
raw/01-交割单/2024-标准化/2024-trade-roundtrips.csv
```

作用：把多笔买卖匹配成一段交易，用于统计盈亏、持仓天数、卖飞、止损。

默认匹配规则：

```text
同一股票、同一账户类型，按 FIFO 先进先出匹配。
融资账户和普通账户分开匹配。
如果 2024-09-23 之前已有持仓成本缺失，开头仓位标记 opening_position_unknown，不强行算完整盈亏。
```

字段：

```csv
roundtrip_id,code,name,account_type,open_date,close_date,holding_days,buy_trade_ids,sell_trade_ids,buy_qty,sell_qty,avg_buy_price,avg_sell_price,buy_amount,sell_amount,total_fee,realized_pnl,realized_pnl_pct,position_status,open_cost_confidence,notes
```

规则：

1. 买入未卖完，`position_status = open`。
2. 卖出超过本期可匹配买入，标记 `opening_position_unknown`。
3. 盈亏计算必须能回溯到 `trade_id`，不能只写结论。
4. 同日 T+0 买卖要保留，不能因为看起来杂乱而删。
5. 回合表可以统计，但不能替代事实表。

### 3. 第三层：交易标签表

文件：

```text
raw/01-交割单/2024-标准化/2024-trade-tags.csv
```

作用：给每笔交易补交易大脑需要的市场环境和行为标签。

字段：

```csv
trade_id,trade_date,code,name,l1_market_env,secondary_env,dominant_profit_structure,main_theme,stock_role,is_in_mainstream_structure,planned_trade,buy_position,sell_position,original_reason,inferred_mode,inferred_confidence,certainty_tag,exit_condition,error_tag,error_cost,error_cost_method,usable_for_mode_validation,usable_for_error_analysis,lookahead_used_for_label,tag_confidence,evidence_files,notes
```

标签规则：

1. `original_reason` 默认 `unknown`，除非用户当时有明确文字记录。
2. `is_in_mainstream_structure` 只判断是否落在当日主流赚钱结构上。
3. `stock_role` 可写：龙头/中军/前排/后排/杂毛/风险锚/未知。
4. `planned_trade` 没有当日计划证明时写 `unknown`，不要写 yes。
5. `usable_for_mode_validation` 只有在当日理由、模式、买卖点都清晰时才能 true。
6. 低置信度推断只能用于 `usable_for_error_analysis = true`。
7. 如果标签使用了 T+1 后信息，必须写 `lookahead_used_for_label = true`，且 `usable_for_mode_validation = false`。

## 五、每日复盘怎么重新做

文件：

```text
raw/02-每日复盘/2024-标准化研究版/YYYY/MM/YYYY-MM-DD-复盘.md
```

每篇复盘顶部必须写：

```yaml
date:
review_version: V1-standard-research
data_cutoff: T日收盘后
lookahead_guard: true
label_frozen_before_dplus: true
old_review_used: yes/no
old_review_role: 仅作事实校验/线索参考，不作判断依据
source_files:
  - raw/01-交割单/2024/...
  - raw/02-每日复盘/2024/...
confidence: high/medium/low
```

每篇必须回答：

1. 今天是什么 L1 市场环境。
2. 今天主导赚钱结构是什么。
3. 今天核心方向、核心股、后排股分别是什么。
4. 今天最有确定性的交易是什么。
5. 今天最危险的交易是什么。
6. 我的交易是否站在主流结构上。
7. 如果亏钱，亏在模式错、环境错、股票角色错、买点错、卖点错、仓位错，还是纪律错。
8. D+1/D+3/D+5/D+10 如何验证，但 D+ 不能回头修改 T 日判断。

## 七、L1 标签表

文件：

```text
raw/04-市场数据/历史L1标签/2024-L1市场环境标签.csv
```

字段：

```csv
date,data_cutoff,l1_market_env,secondary_env,dominant_profit_structure,index_state,market_turnover,turnover_change,up_count,down_count,limit_up_count,limit_down_count,broken_board_rate,highest_board,promotion_rate,mid_board_feedback,trend_strength,crowding_strength,core_concentration,main_themes,core_stocks,risk_anchors,best_modes,forbidden_modes,lookahead_risk,confidence,evidence_files
```

规则：

1. 每天只能有一个 `dominant_profit_structure`。
2. 可以有副环境，但主环境必须明确。
3. 无明确主流时写混沌/轮动/退潮，不强行找主线。
4. `lookahead_risk` 可写 `none/low/medium/high`。
5. 9.24 至 10.11 的标签必须人工复核。

## 八、旧资料怎么用

旧版每日复盘：

```text
raw/02-每日复盘/2024/
```

用途：

1. 校验当日主线和个股线索。
2. 查漏补缺指数、涨跌停、连板天梯。
3. 对照交易明细和持仓。

禁止：

1. 不允许直接复制旧复盘的结论。
2. 不允许复制“后续大龙头”“后来连板”“1/3 天地板”等未来信息到 T 日判断区。
3. 不允许把旧复盘里的情绪化评价当成规则。

如果旧复盘存在后视镜，V1 复盘写：

```yaml
old_review_lookahead_risk: high
old_review_used_for: fact_check_only
```

## 九、校验标准

交割单事实表完成后必须自检：

1. 原始 CSV 文件数是否一致。
2. 原始成交行数是否一致。
3. 每一行是否有唯一 `trade_id`。
4. 买卖方向是否全部标准化为 BUY/SELL。
5. 成交金额、手续费、印花税、过户费、发生金额是否保留。
6. 同日同股多笔成交是否未被误合并。
7. 融资买入是否和普通买入区分。
8. 2024-09-23 前缺失成本是否标记，而不是硬算。

每日复盘完成后必须自检：

1. 是否写 `data_cutoff: T日收盘后`。
2. 是否写 `lookahead_guard: true`。
3. T 日判断区是否没有 T+1 后信息。
4. D+验证区是否和 T 日判断区分开。
5. L1 标签是否和当日市场事实匹配。
6. 交易评价是否只评价“是否匹配当时主流结构”，没有编主观理由。
