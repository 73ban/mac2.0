# 模式验证闭环规则

更新时间：2026-06-13

## 用途

这个文件用于判断一个模式到底是在赚钱，还是只是看起来有道理。

## 闭环流程

```text
模式卡 -> 作战室候选 -> 竞价确认 -> 买入/未买 -> D+1/D+3/D+5跟踪 -> 统计胜率和盈亏 -> 升级/降级
```

## 样本来源

| 样本 | 是否计入模式验证 | 说明 |
|---|---|---|
| 用户真实买入 | 是 | 权重最高 |
| 作战室候选但未买 | 是 | 用于判断大鸟选股质量 |
| 盘中临时关注 | 低权重 | 需要标注非计划内 |
| 事后才发现的牛股 | 否 | 不能倒推验证 |
| 游资文章案例 | 参考 | 只能用于理解，不能替代当前市场验证 |

## D+ 判断标准

| 周期 | 看什么 |
|---|---|
| D+1 | 是否有溢价，是否符合竞价预期，是否坑人 |
| D+3 | 模式是否有持续性，是否走成主升/修复/退潮 |
| D+5 | 题材和个股是否仍有生命力 |
| D+10 | 是否能沉淀为阶段规律 |

## 模式升级规则

一个观察模式升级为可用模式，至少满足：

1. 最近 5 个以上有效样本。
2. 胜率不低于 55%。
3. 平均盈利大于平均亏损。
4. 最大回撤可控。
5. 亏损案例能解释清楚，不是随机亏损。
6. 与当前市场状态匹配。

升级为可重仓模式，至少满足：

1. 最近 10 个以上有效样本。
2. 胜率不低于 60%。
3. 盈亏比大于 1.5。
4. 连续亏损次数不超过 2。
5. 回撤低于用户可承受范围。
6. 竞价确认规则清晰。
7. 退出条件明确且能执行。

## 模式降级规则

| 触发条件 | 处理 |
|---|---|
| 连续 2 次亏损 | 降低仓位权限 |
| 连续 3 次亏损 | 暂停使用 |
| 连续 3 次无溢价 | 从可用降为观察 |
| 市场状态切换 | 重新验证 |
| 出现一次不可接受大亏 | 立即复盘并暂停 |

## 错误类型

| 错误类型 | 定义 |
|---|---|
| 买入时机错误 | 买早、追高、分歧未结束、竞价未确认 |
| 卖出时机错误 | 卖飞、格局过度、未按退出条件卖 |
| 仓位管理错误 | 不该重仓时重仓，该重仓时犹豫 |
| 模式误用 | 市场状态不匹配，硬套模式 |
| 题材判断错误 | 把轮动当主升，把退潮当分歧 |
| 个股地位错误 | 把后排当核心 |
| 信息可信度错误 | 把 S4 小作文当 S1/S2 证据 |
| 非计划交易 | 不在作战室内临时买入 |

## 和统计看板的关系

字段标准以这里为统一入口：

```text
wiki/09-统计与进化/统计看板字段标准-2026-06-13.md
```

统计看板必须支持：

1. 按日期范围统计。
2. 按持股周期统计，而不是只按日胜率。
3. 按主模式统计胜率、盈亏额、买入次数、占比。
4. 按买入方式统计低吸、半路、打板。
5. 按错误类型统计错误成本。
6. 按重仓阈值统计重仓交易表现。
7. 按确定性等级 A+/A/B/C 统计。
8. 按计划内/计划外统计。
9. 按候选票 D+1/D+3/D+5/D+10 统计。
10. 生成每日交易健康分 0-100。

如果统计看板和本文件字段不一致，以本文件字段为准。
