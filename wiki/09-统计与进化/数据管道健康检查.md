# 数据管道健康检查

## 背景

当前数据管道从 `raw/` HTML 转换到 `wiki/sources/` MD 后进入 LLM 摄入，但缺乏自动监控。基于 `go.txt` 脚本的分析，管道可能存在静默失效风险，需要建立健康检查机制。

## 检查维度

1. **转换覆盖率**
   - 统计 `raw/公众号` 下 HTML 数量与已生成 MD 数量的比率。
   - 若比率 < 90%，可能新 HTML 未被转换或产生大量跳过。

2. **错误率**
   - 脚本输出的 `Errors` 计数应记录并设定阈值（如 >5% 需告警）。

3. **来源完整性**
   - 检查输出的 MD 是否包含 `> 来源:` 行，缺失比例可作为数据溯源损失指标。

4. **时效性**
   - 记录最后一次脚本运行的时间戳，若超过一周未运行，可能积压大量未处理文件。

5. **内容合规性**
   - 抽样检查 MD 是否仍残留未清洗的 HTML 标签或乱码实体。

## 实施建议

- 在脚本末尾增加日志写入功能（记录时间、统计、文件清单）。
- 开发一个简单的仪表板脚本或定期汇总报告，集成到复盘流程中。
