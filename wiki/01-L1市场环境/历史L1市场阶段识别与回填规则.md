# 历史L1市场阶段识别与回填规则

更新时间：2026-06-13

## 用途

本文件给老虎回填 `2024-09-01` 至今的交割单和每日复盘使用。目标不是复原当时主观买卖理由，而是识别每个时间段的主流赚钱结构。

历史交易对交易大脑的价值在于：

```text
当时是什么 L1 市场环境
当时主流赚钱结构是什么
我的交易是否落在主流结构上
如果没落在主流结构上，亏损成本是多少
```

## 回填原则

1. 不编当时买卖理由，忘记就写 `original_reason: unknown`。
2. 不用事后大牛股反推“当时一定是主升”。
3. 每天先识别 L1，再识别主流赚钱结构，再给交易打标签。
4. 同一天可以有主 L1 和副 L1，但只能有一个主流赚钱结构。
5. 历史阶段按周/月聚合，但交易标签必须落到具体日期。
6. 严禁未来函数：T 日复盘只能使用 T 日收盘前后当时可获得的数据。
7. T+1 之后数据只能用于 D+ 验证，不能回头修改 T 日 L1、主线、股票角色和买卖评价。

## 主流赚钱结构分类

| 主流赚钱结构 | 识别标准 | 适合模式 | 典型错误 |
|---|---|---|---|
| 连板赚钱结构 | 连板高度打开，晋级率高，空间板有溢价，断板反馈不差 | 连板空间破局、换手龙回封、竞价弱转强、前排确认打板 | 买后排、退潮后接力、中位股硬扛 |
| 机构趋势赚钱结构 | 容量中军持续新高，成交额核心有趋势，连板弱但趋势强 | 强趋容量抱团、趋势回踩均线、主线核心低吸 | 用连板打法追趋势票，或趋势破位还低吸 |
| 抱团赚钱结构 | 资金集中在少数核心方向，核心强、后排弱、非主线失血 | 强趋容量抱团、题材主升核心持股、核心抗跌穿越 | 买非主线、买后排、分散持仓 |
| 题材主升赚钱结构 | 主线清晰，核心/中军/前排/补涨梯队完整，分歧后修复 | 题材主升核心持股、主线核心低吸、强板块前排半路 | 高潮后追补涨，核心补跌还做后排 |
| 轮动套利结构 | 多方向快速切换，日内龙多，隔日持续性弱 | 首板新题材试错、日内龙套利、缩量轮动套利 | 格局、重仓、隔日幻想 |
| 防守/退潮结构 | 高标 A 杀、连板压缩、后排大面、核心补跌 | 风险锚减仓、计划外交易禁止、亏损后降频 | 不空仓、补仓、退潮期重仓 |

## 防未来函数规则

历史复盘分两步，顺序不能反：

```text
第一步：只用 T 日当时可见数据，冻结 L1、主流赚钱结构、股票角色、交易是否匹配主流。
第二步：再读取 T+1/T+3/T+5/T+10，写 D+ 验证结果。
```

例子：

```text
2025-01-03 是周五，2025-01-04 是周六，不是交易日；下一交易日通常是 2025-01-06。
复盘 2025-01-03 时，不能因为 2025-01-06 某票走强，就把 2025-01-03 改写成确定性主升。
正确写法是：2025-01-03 当日按当时数据判断；2025-01-06 的表现只进入 D+1 验证。
```

如果发现使用未来数据影响 T 日标签，必须标记：

```yaml
lookahead_violation: true
usable_for_mode_validation: false
usable_for_error_analysis: false
needs_rework: true
```

## 抱团行情识别规则

抱团行情至少满足 4 项：

1. 成交额前 20/50 集中在少数方向。
2. 少数核心票持续强于指数和大部分题材。
3. 非主线方向失血，后排没有持续溢价。
4. 龙头/容量核心连续新高或分歧后快速修复。
5. 市场开始出现“只买核心”的一致预期。
6. 风险信号开始集中在放量滞涨、拥挤、估值叙事全民化。

抱团行情的交易规则：

```text
只做核心，不做后排。
只做主流，不做非主流。
只做回踩/分歧/强承接，不做无确认追高。
抱团崩塌信号出现时先降仓，不和信仰对抗。
```

## 最终统计要回答的问题

```text
我在哪种 L1 环境赚钱最多？
我在哪种主流赚钱结构亏钱最多？
抱团行情里，我买的是核心还是后排？
连板行情里，我是否买在退潮后段？
机构趋势行情里，我是否过早卖出核心、追了后排？
题材主升行情里，我是否拿得住核心？
退潮行情里，我是否还在重仓或补仓？
计划外交易在哪类环境下错误成本最大？
```

## 禁止

1. 不允许凭记忆补理由。
2. 不允许把事后涨幅最大的票定义为当时主线。
3. 不允许把抱团行情当普涨行情。
4. 不允许用低置信度推断升级 A 类模式。
5. 不允许只统计盈亏，不统计是否匹配主流赚钱结构。
6. 不允许用 T+1 之后数据反向美化 T 日判断。
