# 20万小资金做大与 AI 超短进化总控

更新时间：2026-06-28

## 两个总目标

1. 20万小资金快速做大。
2. AI 通过你的实盘、复盘、D+验证、错误库和统计闭环，进化成超级短线高手。

## 核心原则

```text
小资金要快，不是乱快；AI要进化，不是堆资料。
```

所有 WIKI 内容必须服务于以下任一目标：

- 提高明日交易胜率。
- 降低单笔亏损和最大回撤。
- 提高主线识别速度。
- 减少重复错误。
- 验证一个模式是否能赚钱。
- 把一次实盘反馈转化为下一次更好的动作。

## 20万账户约束

| 项目 | 规则 |
|---|---|
| 资金定位 | 小资金进攻账户，重点做弹性和效率 |
| 首要风险 | 大回撤、连续亏损后上头、追后排、过度交易 |
| 仓位原则 | 只有 A 类模式 + 主线确认 + 竞价确认，才允许提高仓位 |
| 禁止 | 无计划交易、情绪补仓、退潮期重仓、资料逻辑代替盘面 |
| 复利来源 | 主线核心、强分歧修复、前排确认、错误减少 |

## AI 进化闭环

```text
RAW事实 -> WIKI规则 -> 作战室执行 -> 交割单反馈 -> D+验证 -> 错误库/模式库修正 -> 下一轮作战
```

AI 每日启动读取入口：

- `wiki/10-系统配置/AI每日启动读取清单.md`

当前交易日上下文入口：

- `wiki/07-作战室/2026-06-29-AI上下文包.md`

AI 每次回答交易问题时，优先级固定为：

1. 当前持仓和资金风险。
2. 今日/明日 L1 市场环境。
3. L2 主线阶段。
4. L3 个股核心性。
5. L4 买卖点和仓位权限。
6. D+验证和历史错误。
7. 外部资料和观点。

## 七部分整理任务

| 部分 | 目标 | 当前处理方式 |
|---|---|---|
| 个股档案 | 找到真正能帮小资金做大的核心票 | 核心池复核 + 正式候选档案 + D+验证 |
| 题材层 | 把 1000 多个题材页压成生命周期 | 高频主线总表 + 题材阶段判断 |
| 信息来源 | 防止资料仓库化 | S1-S4 可信度 + A-D 用途分级 |
| 模式执行 | 把游资心得变成可执行模式 | 模式总控 + D+验证 + 错误反馈 |
| 统计进化 | 用交割单训练 AI | 收益/回撤/胜率/模式/错误统计 |
| 99区 | 清理迁移残留 | 归档或风险隔离，不删除 |
| RAW散落 | 只保留原始输入 | 规则类文件迁入 WIKI 系统配置 |

## 每日最低闭环

1. 盘前：读取作战室、L1、L2、持仓、D+待验证。
2. 盘中：只按计划和触发条件行动。
3. 盘后：交割单先进 RAW，再复盘。
4. 复盘：写错误、模式、个股、题材变化。
5. D+：验证昨天、3天前、5天前、10天前、20天前、30天前判断。
6. AI：根据验证结果升级或降级规则。
