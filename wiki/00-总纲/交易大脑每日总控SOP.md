# 交易大脑每日总控SOP

更新时间：2026-06-13

## 用途

这个文件是每天使用 WIKI 的总控流程。目标是把信息、模式、作战室、执行、复盘、统计连成闭环。

## 一天的主流程

```text
前一晚信息沉淀
-> 早盘前作战室
-> 9:15/9:20/9:25竞价确认
-> 盘中只执行计划
-> 收盘后验证候选票
-> 晚间复盘和模式升级/降级
```

## 前一晚：信息沉淀

时间：收盘后到睡前。

输入：
```text
RAW新增内容
公告
研报
知识星球
公众号游资文章
淘股吧复盘
龙虎榜
美股/日韩早报
并购重组消息
```

输出：
```text
题材生命周期更新
重点个股作战档案更新
游资模式卡补充
高价值资料优先处理清单
```

判断重点：
1. 哪些信息是 S1/S2 硬证据。
2. 哪些只是 S3/S4 观点或小作文。
3. 哪些题材可能进入启动、确认、主升。
4. 哪些个股可能成为明日风险锚或进攻锚。

## 8:40：盘前作战室

前置生成：

```text
raw/07-系统脚本/codex_generate_daily_candidate_score.py --date YYYY-MM-DD
```

输出文件：

```text
wiki/07-作战室/YYYY-MM-DD-作战室候选票评分表.md
raw/03-每日交易计划/YYYY-MM-DD-竞价监控清单.md
```

必须输出：
```text
市场状态
市场情绪分 0-10
市场行情分 0-10
今日主线候选
今日主模式
候选票评分表
主计划 1 个
备选计划 1-2 个
禁做清单
竞价确认条件
退出条件
仓位权限
```

硬约束：
1. 重点股票最多 3 只。
2. 主计划只能 1 个。
3. 每只候选票必须有主模式。
4. 没有退出条件的票，不进主计划。
5. 不是计划内交易，默认错误。

## 9:15：一字定方向快照

看什么：
1. 大单一字涨停方向。
2. 大单一字跌停方向。
3. 作战室票和核心票是否符合预期。
4. 昨晚最强催化是否被市场认可。
5. 风险锚是否恶化。

处理：
```text
符合预期：继续观察，不急买。
超预期：保留主计划，等 9:20 真假过滤。
不及预期：降仓或取消计划。
风险锚恶化：暂停进攻。
```

## 9:20：真假过滤

9:20 后不能撤单，这一刻比 9:15 更真实。

看什么：
1. 9:15 大单一字是否撤单。
2. 热门方向是否仍然强。
3. 核心票是否继续强于板块。
4. 竞价弱转强是否真实。
5. 作战室票是否仍满足触发条件。

处理：
```text
撤单明显：降低对应题材评分。
核心仍强：提高作战室确认度。
孤立强：谨慎，不追后排。
风险锚继续恶化：取消主计划。
```

## 9:25：最终竞价确认

看什么：
1. 开盘价是否符合预期。
2. 量能是否匹配。
3. 板块是否扩散。
4. 主计划是否仍在模式条件内。
5. 禁止条件是否触发。

处理：
```text
竞价质量 8-10：计划继续，等开盘承接。
竞价质量 6-7：只观察，开盘确认后再说。
竞价质量 4-5：降仓或取消。
竞价质量 0-3：禁止买。
```

## 盘中执行

只做三件事：
1. 执行主计划。
2. 管理已有持仓。
3. 记录计划外冲动。

盘中禁止：
1. 临时看到拉升就追。
2. 用新故事推翻早盘规则。
3. 不写退出条件就加仓。
4. 弱市场里频繁试错。

## 收盘后验证

必须更新：
```text
候选票跟踪池
真实交易记录
模式验证闭环
错误成本账本
近期有效模式排行榜
```

收盘后先采集本地行情快照：

```text
raw/07-系统脚本/codex_fetch_eastmoney_market_snapshot.py --date YYYY-MM-DD
```

输出：

```text
raw/04-市场数据/东方财富/YYYY-MM-DD/market-snapshot.json
```

用途：

```text
73WIKI 软件候选票 D+ 跟踪页会读取本地快照，自动回填 D+1/D+3/D+5/D+10、最大涨幅、最大回撤和自动结论。
如果当天网络或接口不可用，不能编行情，只记录快照缺失。
```

每晚必须回答：
1. 今天作战室选的票，比随机票强吗？
2. 主模式是否有效？
3. 买入是否符合计划？
4. 卖出是否符合纪律？
5. 最大错误是什么？
6. 明天应该提高还是降低仓位？

## 周末进化

每周末只做 4 件事：
1. 统计本周最赚钱模式和最亏钱模式。
2. 找出本周最大错误成本。
3. 更新 A 类模式候选。
4. 删除或降级无效模式。
