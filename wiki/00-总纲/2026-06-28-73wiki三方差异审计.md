# 73wiki 三方差异审计

审计日期：2026-06-28

审计对象：

1. `Workspace/73wiki-source`：当前工具代码与自动化脚本。
2. `Desktop/拷贝MAC`：杨明杰原版框架与早期规划文本。
3. `wiki/73神话`：已迁移并持续生成的真实交易数据。

## 一句话结论

73wiki 不应继续做资料仓库，应收敛成“超短交易操作系统”。

`trading-review-wiki / 73wiki-source` 负责工具、自动化、采集、统计、审计、界面；`wiki/73神话` 负责稳定交易方法、纪律、错误、验证、进化。RAW 是事实入口，WIKI 是经过筛选、验证、结构化之后的交易决策层。

当前最大偏差不是功能太少，而是边界太混：工具层、资料层、交易认知层、自动产物层都在互相穿透，导致 WIKI 有继续膨胀成资料仓库的风险。

## 当前盘点

### 1. 工具层：`73wiki-source`

已具备的能力：

| 模块 | 当前状态 | 处理建议 |
|---|---|---|
| 项目打开、文件树、编辑器 | 可用 | 保留 |
| RAW watcher / ingest queue | 可用，但路径默认值与现项目不完全一致 | 重写配置与路由 |
| 交割单导入、FIFO、统计看板 | 可用 | 保留并升级为交易审计入口 |
| 搜索、图谱、聊天、Save to Wiki | 可用 | 保留，但降级为辅助工具 |
| Lint / Review / Plan / Dashboard | 有雏形 | 重写成操作系统视角 |
| `scripts/*.mjs` 市场、候选、验证、训练脚本 | 很多已经产生有效产物 | 保留，但需要统一总控 |
| 通用 LLM Wiki / Deep Research / 泛知识库描述 | 遗留明显 | 从主流程删除或隐藏 |

关键问题：

1. `README_CN.md` 同时保留“交易复盘软件”和“通用 LLM Wiki”两套叙事，不利于后续收敛。
2. `src/components/layout/icon-sidebar.tsx` 仍以“知识库、原始资料、搜索、关系图、统计看板、自动摄入、交易计划、知识体检、快速复盘”为通用应用导航，不是交易操作系统导航。
3. `src/lib/wiki-config.ts` 默认路径与现项目不一致：
   - 默认 `raw/研报新闻`，现项目是 `raw/05-研报新闻`。
   - 默认 `raw/交割单`，现项目是 `raw/01-交割单`。
   - 默认 `raw/日复盘`，现项目是 `raw/02-每日复盘`。
   - 默认 `raw/交易记录`，现项目未作为主目录出现。
4. 快速复盘模板写入 `raw/日复盘/`，与现项目目录 `raw/02-每日复盘/` 不一致。
5. 系统配置已经有 `73wiki.system.json`，但杨明杰原规划要求拆出 `73wiki.templates.json`、`73wiki.trading.json`、`73wiki.agents.json`，目前还没有完成。

### P2：摄入规则重写

当前风险：LLM 容易把资料直接写进 WIKI，导致 WIKI 膨胀。

新规则：

```text
RAW 新资料
-> 来源识别
-> S1-S4 可信度
-> A-D 信息命运
-> 交易影响分类
-> 写入以下之一：
   A 明日作战：进入作战室候选/观察池
   B 持续跟踪：进入题材/个股时间线
   C 归档留证：只保留 RAW 索引
   D 登记丢弃：不写 WIKI
```

只有以下内容允许写入 WIKI：

1. 影响明日交易计划。
2. 更新持仓条件或卖出条件。
3. 更新模式有效性。
4. 形成错误/纪律样本。
5. 通过 D+验证后升级为规则。

3. 今日可做模式清单：
   - 市场状态
   - 可做模式
   - 禁做模式
   - 仓位权限
   - 触发条件
   - 风险锚
4. 作战室：
   - 今日唯一交易入口
   - 候选最多 3-8 只
   - 主计划、备选、禁做、退出、盘中确认
5. 交易审计页：
   - 是否计划内
   - 是否符合 A 类模式
   - 是否违反退潮纪律
   - 是否卖点拖延

### P4：统计与验证重写

现有 `D+1/D+3/D+5/D+10/D+20/D+30` 验证应变成闭环：

1. 候选票入池必须有原始判断和证据路径。
2. 到期自动进入验证队列。
3. 验证只回答三件事：
   - T 日判断是否成立。
   - 模式是否增强/降级。
   - 下一次规则怎么改。
4. 结果写入：
   - `data/brain/validations.jsonl`
   - `wiki/09-统计与进化`
   - 对应个股/模式页摘要
