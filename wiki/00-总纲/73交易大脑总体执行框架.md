# 73交易大脑总体执行框架

更新时间：2026-06-18

## 目标

把 `20万小资金快速做大` 拆成可执行、可验证、可复盘、可进化的日常流程。

## 总体链路

```text
RAW事实输入
-> 市场环境识别
-> 主流赚钱结构识别
-> 交易模式选择
-> 作战室生成
-> 竞价确认
-> 实盘执行
-> 盘后复盘
-> D+验证
-> 规则升级/降级
```

## 分工

### 你

- 提供交易意图、真实理由、现场感受。
- 提供交割单、委托单、持仓变化、关键截图。
- 负责最终拍板和下单。

### Codex = 大鸟

- 在飞书侧和电脑侧都是同一个执行体。
- 负责把材料落 RAW、补齐关键客观数据、正式复盘、写入 WIKI、生成作战室、主动汇报。
- 可以区分“先落 RAW”和“后复盘”两个动作，但不是两个主体。

### WIKI

- 保存稳定规则、验证过的经验、模式卡、错误库和统计结果。
- 不保存无证据臆断。

## 每日执行步骤

### 盘前

- 读取最新 RAW 和最新制度入口。
- 判断今天哪些模式可做，哪些不可做。
- 给出主计划、备选计划、禁做条件、退出条件。

### 竞价

- 优先使用已获得的竞价数据。
- 没有龙虾辅助数据时，大鸟直接接管或明确标注 `数据缺口`。
- 竞价只用于确认、降级或暂停计划，不单独替代完整交易逻辑。

### 盘中

- 围绕持仓、主计划、风险锚、异常强票更新判断。
- 发现重大变化时，不等收盘，直接主动汇报。

### 盘后

- 汇总全天事实。
- 复盘交易、仓位、纪律、错误、临盘偏差。
- 提炼需要写入 WIKI 的稳定规则候选。

## 约束

1. 没有 RAW，不下结论。
2. 没有证据，不写进 WIKI。
3. 没有验证，不升级为长期规则。
4. 没有计划内逻辑，不允许把临场发挥包装成系统能力。
