# 2026-07-05 公告事件与晚间提醒闭环接入记录

## 背景

用户明确要求：

- 公告不能只存摘要；
- 每条重要公告要变成可验证样本；
- 对应个股档案要记录公告日期、内容和后续股价表现；
- 后续要训练系统判断哪些公告能持续大涨，哪些只是一日游。

## 本次接入

### 1. 公告事件入库

新增脚本：

```text
raw/07-系统脚本/codex_register_announcement_events.py
```

职责：

- 读取龙虾/老虎写入的公告 RAW；
- 反查或标记股票代码；
- 生成公告事件样本；
- 更新个股公告事件档案；
- 写入公告事件 D+验证队列。

输出：

```text
raw/11-Codex分析产物/公告事件样本/YYYY-MM-DD/公告事件样本.json
raw/11-Codex分析产物/公告事件样本/YYYY-MM-DD/公告事件样本.md
wiki/03-L3个股档案/公告事件档案/
wiki/09-统计与进化/公告事件D+验证队列.md
```

### 2. 公告 D+自动回填

新增脚本：

```text
raw/07-系统脚本/codex_announcement_dplus_autofill.py
```

职责：

- 读取公告事件 D+验证队列；
- 到期后抓取对应股票日 K；
- 回填 D+0/D+1/D+3/D+5/D+10 表现；
- 给出持续强反馈、一日游风险、负反馈等标签；
- 回写个股公告事件档案。

输出：

```text
wiki/09-统计与进化/YYYY-MM-DD-公告事件D+自动回填.md
data/facts/announcement_event_validation_results.jsonl
```

### 3. 晚间提醒接入公告事件样本

更新脚本：

```text
raw/07-系统脚本/codex_evening_stock_news_radar.py
```

修正点：

- 加入 `raw/11-Codex分析产物/公告事件样本/` 扫描源；
- JSON 不再按全文扫，只按单条股票记录匹配，避免串味；
- Markdown 表格不再取上下文，避免相邻股票串味；
- 同一股票同一来源文件只保留最高分线索；
- 用户决策区按股票去重，每只票只展示一条最重要线索。

## 路径原则

```text
raw/05-研报新闻/公告/
```

只保留龙虾/老虎事实层写入的原始或轻标准化公告。

```text
raw/11-Codex分析产物/
```

存放 Codex 派生分析、公告事件样本、晚间线索和回传产物。

原因：Mac 端 `raw/05` 是 Syncthing receiveonly 目录，Codex 不应在里面写派生产物，否则会污染事实层原始目录。
