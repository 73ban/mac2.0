# 2026-06-19 73交易大脑能力升级蓝图

更新时间：2026-06-20

## 最高目标

这份蓝图只服务两个最高目标：

1. 辅助你把 20 万小资金更快、更稳地做大。
2. 让 73交易大脑持续自学、验证、进化，最终形成明显强于你的超短辅助能力。

## 当前正式架构

自 2026-06-19 起，73交易大脑正式采用六层结构：

1. `公共数据主链`
2. `RAW 学习主链`
3. `iFinD 低频增强层`
4. `原因卡层`
5. `自动纠偏层`
6. `飞书主动提醒层`

## 已正式落地

### 1. 主数据链替换完成

正式主链已经从通达信 MCP 切换为：

1. `AKShare`
2. `腾讯财经`
3. `财联社实时流`
4. `同花顺热榜`
5. `iWenCai / pywencai`
6. `Ashare / mootdx`
7. `iFinD 低频补证`

补充规则：

- 周末或收盘后允许预生成下一个交易日的盘前预案。
- 只要同一份产物里同时混有“下个交易日预案”和“当日收盘证据”，就必须同时写出 `计划交易日` 与 `证据交易日`。

### 2. 腾讯分时回填层已落地

腾讯财经现在不再只是报价快照，而是正式进入验证链：

1. `09:25` 开盘价回填。
2. `09:31` 之后分钟分时回填。
3. `15:05` 收盘快照补齐。
4. 本地名称优先覆盖腾讯原始乱码名称。

结果：

`market-validate` 已从“经常数据不足”升级为“核心票可完成完整分时验证”。

### 3. iFinD 低频增强层已落地

当前固定用途：

1. 近端新闻补证。
2. 风险词识别。
3. 原因卡增强。
4. 自动纠偏辅助。
5. 飞书提醒证据支持。

边界固定：

- 不做主摄入。
- 不做高频轮询。
- 不覆盖 RAW 学习层。

### 4. 自动纠偏层已落地

当前正式输入：

1. `market-watchlist`
2. `stock-reason-cards`
3. `ifind-enhance`
4. `correction-alerts`

当前正式判断逻辑：

1. 官方风险优先。
2. 媒体风险降级处理。
3. 原因卡与价格冲突独立计分。
4. 只把真正影响交易判断的项目升级成飞书候选。

日期展示规则：

1. `planTradeDate` 表示本次纠偏服务的是哪一个计划交易日。
2. `evidenceTradeDate` 表示本次纠偏引用的最新盘面/新闻证据属于哪一天。
3. 两者不一致时，不允许只显示一个日期。

### 5. 纠偏证据分级已制度化

自动纠偏现在分三档证据强度：

1. `strong`
   说明：交易所/公告/官方异常波动，或高置信原因卡与价格大幅冲突。
2. `medium`
   说明：重复媒体风险，或原因卡偏强但价格明显转弱。
3. `weak`
   说明：单条媒体线索、弱冲突、只适合记录观察。

执行标准：

1. `strong` 证据优先进入飞书候选。
2. `medium` 证据只有在确实影响你当前作战室判断时，才允许升级为飞书候选。
3. `weak` 证据只落本地记录，不主动打扰你。

### 6. 飞书主动提醒层已落地

当前已经形成正式闭环：

1. `correction-alerts` 生成候选。
2. `correction-push` 做推送决策。
3. 飞书桥 `send-correction-alerts` 负责实际发送。

结果：

大鸟不再只是系统内发现问题，而是能把高价值纠偏主动推给你。

### 7. 游资方法学习闭环已明确

大鸟学习游资方法时，必须区分三层：

1. `学习观察`
2. `已验证方法`
3. `已纳入执行规则`

禁止混层。

所有学习目标最终都要落到一件事：

`是否真正提升你的小资金做大效率`

## 下一阶段继续推进

### 1. 竞价层继续补强

当前分时层已稳定，但 `09:15 / 09:20` 的真实竞价梯度还不够完整。

下一步重点：

1. 增加更强的竞价补证源。
2. 把竞价强弱写成固定评分。
3. 让竞价结果直接作用于作战室优先级。

### 2. 用户错误模型

要把你高成本、重复性的错误沉淀成系统模型：

1. 易冲动环境。
2. 易卖飞模式。
3. 易追高模式。
4. 易误判节奏。

目标不是记录错误，而是减少再次犯错。

### 3. 游资方法自动化验证

继续推进：

1. 自动提炼游资说法。
2. 自动找对应盘面样本。
3. 自动比对是否真实有效。
4. 有效后再进入“已验证层”。

### 4. 市场状态机 V1

固定输出四项：

1. 当前市场状态。
2. 今日仓位权限。
3. 今日优先模式。
4. 今日禁止模式。

## 模型升级态度

未来模型更强时，只会让这套系统更快、更深、更稳。

但当前原则不变：

不是等模型升级后再工作，而是现在就把制度、数据、验证、纠偏、飞书闭环全部先搭好。

`主链摄入 -> 盘中验证 -> 自动纠偏 -> 飞书提醒 -> 学习验证 -> 认知进化`
