# 淘股吧实盘赛样本学习规则

## 定位

淘股吧公开实盘赛不是用来抄票，而是用来学习高质量选手的真实操作模式。

重点关注：

- 千万组；
- 同一比赛的千万组前10名；
- 人气高、风云值高、每日公开信息较完整的选手；
- 连续多日能看到持仓、买入、卖出和收益曲线的选手。

核心判断口径：

```text
比赛名称 + 资金分组 + 组内排名
```

不能用全组排名、博客页总排名、人气排名替代千万组组内排名。

## 当前优先级

现阶段不追求全自动抓几百人，先把五个结构做准：

| 结构 | 要求 |
|---|---|
| 比赛白名单 | 只收淘股吧官方、定力、创世纪888888/创世纪88、镀金梦想，或用户明确加入的高公信力主办方 |
| 千万组前10选手池 | 必须是同一比赛千万组组内前10，当前前10和往期最终前10都要保留排名日期 |
| 公开交割单字段 | 持有、买入、卖出、仓位变化、交割单证据、是否问号/缺字段 |
| 每日分享字段 | 分享原文、观点摘要、题材判断、风险态度、是否空仓、来源 URL |
| D+验证字段 | D+1、D+3、D+5、验证指标、验证状态、最终结论 |

只有这五个结构稳定后，才扩大选手规模和抓取频率。

## 每日必须提取

| 字段 | 用途 |
|---|---|
| 比赛/分组/组内排名 | 判断样本质量，优先同一比赛千万组前10 |
| 全组排名/其他排名口径 | 保留原始口径，防止误判 |
| 选手 | 建立选手行为轨迹 |
| 选手主页 | 后续追踪同一选手 |
| 今日盈亏/总收益 | 判断当日操作是否被市场验证 |
| 持有 | 看隔夜方向和仓位态度 |
| 买入 | 看新开仓逻辑 |
| 卖出 | 看止盈、止损、切换和风控 |
| 交割单证据 | 区分事实动作和观点推测 |
| 当日分享观点 | 提炼买票思路和风控思路 |
| 题材标签 | 对照当日主线 |
| 是否空仓 | 判断高手对风险的态度 |
| 次日表现 | D+验证 |

## 结构化样本字段

`结构化样本.json` 中每条候选样本至少包含：

```text
采集日期、比赛日期、主办方、比赛名称、分组、组内排名、是否千万组前10、
选手、选手主页、排名证据URL、公开资料完整度、
公开交割单字段、每日分享字段、D+验证字段、样本偏差
```

## 模式提炼方法

每个选手不按单日评价，按连续行为评价：

```text
千万组前10选手
-> 每日公开交割单/持仓/买入/卖出
-> 当日分享观点
-> 股票与题材标签
-> 当日市场环境
-> 当日涨停/连板/板块/龙虎榜/热榜
-> 次日和三日表现
-> 候选模式
-> 用户交易记录对照
-> 有效模式沉淀
```

## 和用户交易记录怎么对照

目的不是模仿别人，而是帮用户给自己的思路命名、补全和降噪。

对照维度：

1. 用户是否也看到了同一条主线；
2. 用户是否比实盘赛选手更早/更晚；
3. 用户买的是先锋龙、趋势龙、资金龙、中军龙，还是杂毛跟风；
4. 用户卖点和高手卖点差在哪里；
5. 用户亏损是否来自环境判断错误、仓位错误、还是模式不完整；
6. 是否能把用户原本模糊的买票理由归类成稳定模式。

## 写入交易模式库的门槛

实盘赛样本进入模式库必须满足：

1. 来源是千万组前10或经过用户指定的高质量选手；
2. 有公开交割单、持仓或买卖动作证据；
3. 有当日分享、复盘或可解释的盘面背景；
4. 同类动作至少出现3次，或单次动作与重大市场拐点高度相关；
5. D+1/D+3/D+5 结果能验证，不是单纯事后归因；
6. 能和用户交易记录形成对照，帮助用户给自己的买票思路命名、补全或降权。

## 禁止事项

1. 不把比赛前排选手的买入直接当作买入建议。
2. 不把未完整公开的问号买入伪造成事实。
3. 不因单日收益高就升级模式。
4. 不抓登录后、积分、付费、私密交割单。
5. 不把比赛未结束的阶段排名等同于最终能力。
