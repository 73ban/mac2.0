# 竞价质量评分规则

更新时间：2026-06-18

## 目的

竞价评分不是预测器，而是执行过滤器。

它只回答一件事：当前竞价反馈，是否支持原计划继续、降级、暂停或禁止。

## 当前口径

1. 竞价辅助数据有则用，无则标注缺口。
2. 竞价评分由大鸟负责综合判断。
3. 本机脚本可以提供 9:15 / 9:25 辅助快照，但不是必需前置。
4. 用户导入RAW入口当前待命，不作为固定 9:20 主责。
5. tdxrs 已接入竞价快照辅助链路，但它不是独立集合竞价委托明细接口。

## tdxrs 竞价快照使用规则

输出目录：

```text
raw/04-市场数据/tdxrs竞价快照/YYYY-MM-DD/
```

使用口径：

| 时点 | 主要看什么 | 不能怎么用 |
|---|---|---|
| 09:15 | 初始价格、五档挂单、是否极端一致 | 不能直接判定买入 |
| 09:20 | 撤单后真实度、封单/买盘是否明显衰减 | 不能只看涨幅不看撤单 |
| 09:24:50 | 临近撮合前最终态度 | 不能忽略最后几十秒变脸 |
| 09:25:05 | 撮合后价格和成交额确认 | 不能当逐笔委托簿 |
| 09:30 | 开盘分时和逐笔成交承接 | 不能倒推盘前计划 |

评分原则：

1. tdxrs 快照只能给“盘口/竞价”维度加减分。
2. 同一只票必须比较多个时点，单点快照不作为强结论。
3. 09:20 后明显撤单、买卖量差转弱，竞价强度降级。
4. 09:25 后撮合价强但 09:30 承接弱，开盘确认价值降级。
5. 没有竞价快照时，继续按原规则标注 `竞价数据缺口`。

## 个股竞价评分

| 维度 | 分值 | 含义 |
|---|---:|---|
| 预期匹配 | 2 | 是否符合盘前预期 |
| 强度 | 2 | 封单、竞价额、价格反馈是否够强 |
| 板块支持 | 2 | 板块是否共振 |
| 风险锚 | 2 | 风险锚是否恶化 |
| 开盘确认价值 | 2 | 是否值得继续跟踪到开盘 |

## 使用规则

1. 8 分以上，说明值得重点确认，不代表直接买入。
2. 6-7 分，通常需要开盘后继续确认。
3. 5 分以下，计划应降级或暂停。
4. 风险锚崩溃时，即使目标股强，也要降低仓位权限。
5. 没有竞价数据时，不能伪造评分，只能标注 `数据缺口`。

## 09:25 竞价弱转强

竞价弱转强由Mac本机RAW入口在 09:25 执行，Codex 盘后验收。

统一 RAW 路径：

```text
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.json
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.md
```

硬规则：

1. 文件名和字段全部中文化。
2. 高开不等于弱转强。
3. 没有昨日弱的原因，不允许叫弱转强。
4. 没有今日转强证据，不允许叫弱转强。
5. 必须写竞价金额、竞价量占昨日成交额比例、板块对比、同身位对比。
6. 必须写 A/B/C/D 操作分级、买点、止损、骗炮风险。
7. ST 股不纳入。

盘后验收：

```bash
python3 raw/07-系统脚本/codex_check_auction_weak_to_strong.py --date YYYY-MM-DD --write
```
