# 公告事件入库与D+验证闭环规则

## 定位

公告不是简单资讯。每条重要公告都要变成可验证样本，用来训练系统判断：

- 哪些公告能刺激个股持续大涨；
- 哪些公告只能一日游；
- 哪些公告实际是风险压制；
- 哪些公告需要结合题材、市场情绪、竞价和资金承接才能成立。

## 分工

| 角色 | 职责 | 不做 |
| --- | --- | --- |
| 龙虾/老虎事实层 | 拉公告原始数据，写入 RAW | 不做最终交易判断 |
| Codex | 做公告事件入库、个股档案更新、D+验证队列、规律提取 | 不把公告本身当买入依据 |

## RAW输入

公告原始或轻标准化数据必须落在：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.json
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.md
```

要求字段：

| 字段 | 要求 |
| --- | --- |
| 公告标题 | 必须 |
| 公司名称 | 必须 |
| 股票代码 | 尽量必须；缺失时标记 `unknown` 或 `待补` |
| 公告日期 | 必须 |
| 分类标签 | 业绩/增减持/监管/重组/合同/其他 |
| 公告摘要 | 接口摘要即可，但必须标明不是公告全文 |
| 公告链接 | 必须尽量保留 PDF 或交易所链接 |
| 原始字段 | 必须保留，方便追溯 |

## Codex接力输出

Codex 每天读取公告 RAW 后，必须生成：

```text
raw/11-Codex分析产物/公告事件样本/YYYY-MM-DD/公告事件样本.json
raw/11-Codex分析产物/公告事件样本/YYYY-MM-DD/公告事件样本.md
wiki/03-L3个股档案/公告事件档案/{代码}-{名称}-公告事件档案.md
wiki/09-统计与进化/公告事件D+验证队列.md
```

边界：`raw/05-研报新闻/公告/` 只放龙虾/老虎事实层事实层抓取的原始或轻标准化公告；Codex 派生样本统一放 `raw/11-Codex分析产物/`，避免污染 Syncthing receiveonly 原始目录。

如果该股已有正式个股档案或 RAW 增量骨架，需要追加一条公告事件跟踪备注，指向公告事件档案。

## 验证节点

每条公告事件默认进入：

| 节点 | 看什么 |
| --- | --- |
| D+0 | 首个可交易日竞价、开盘、是否涨停、是否冲高回落 |
| D+1 | 是否继续强于板块，是否进入热榜/成交额前排 |
| D+3 | 是否形成持续主线或个股独立趋势 |
| D+5 | 是否仍强于板块，是否被资金持续认可 |
| D+10 | 最终归类：持续大涨/一日游/无效/负反馈 |

## 初始分类

| 类型 | 初始假设 | 必须验证 |
| --- | --- | --- |
| 业绩公告 | 正向，但要看是否属于行业景气和主线方向 | 高开后是否承接，是否放量滞涨 |
| 并购重组 | 正向或事件驱动 | 是否有资产质量、审核、停复牌、情绪周期支持 |
| 中标合同 | 正向 | 金额是否足够大，是否能改变业绩预期 |
| 增持回购 | 偏正向 | 是否只是护盘，是否有资金跟随 |
| 减持/问询/立案/异动澄清 | 风险或压制 | 是否低开低走，是否杀情绪 |

## 禁止误用

- 公告本身不等于买点。
- 没有 D+验证的公告，不能沉淀为规律。
- 没有股票代码的公告，不能进入自动股价验证，只能先入“代码待补”。
- 只看公告摘要不看股价反馈，会产生后视镜误导。
- 业绩高增但不是主线、没有承接，也可能是一日游。

## 执行脚本

```bash
python3 raw/07-系统脚本/codex_register_announcement_events.py --date YYYY-MM-DD --write
python3 raw/07-系统脚本/codex_announcement_dplus_autofill.py --today YYYY-MM-DD
```

入库脚本负责：

- 读取 `daily-announcements.json`；
- 反查或标记股票代码；
- 生成公告事件样本；
- 更新个股公告事件档案；
- 追加公告事件 D+验证队列。

回填脚本负责：

- 读取 `wiki/09-统计与进化/公告事件D+验证队列.md`；
- 按 D+0/D+1/D+3/D+5/D+10 找到到期事件；
- 拉取到期日股价表现；
- 写入 `data/facts/announcement_event_validation_results.jsonl`；
- 生成 `wiki/09-统计与进化/YYYY-MM-DD-公告事件D+自动回填.md`；
- 回写到 `wiki/03-L3个股档案/公告事件档案/`。

## 盘后闭环接入

每日闭环脚本已经接入公告链路：

```bash
python3 raw/07-系统脚本/codex_run_daily_closed_loop.py --date YYYY-MM-DD
```

其中会依次执行：

1. 公告事件入库；
2. 公告事件 D+自动回填；
3. AI 上下文刷新。

如果当天公告 RAW 还没同步到 Mac，公告事件入库允许失败或空跑，但必须在数据接口健康报告里标缺口。
