# 收盘事实层六模块采集验收规则

```yaml
type: 系统配置
date: 2026-07-04
owner: Codex
scope: Mac本机RAW入口 / 用户导入RAW入口 / Codex
status: active
```

## 定位

本规则用于验收每天收盘后的事实层数据是否够 Codex 做复盘、作战室、D+验证和模式进化。

事实层只拉数据，不写交易结论。

## 当前认可的收盘任务结构

15:10 收盘数据落盘可以作为主任务，避免重复创建 15:12 任务。

22:00 每日公告落盘可以作为公告主任务，覆盖并购重组、业绩预告、减持增持、异动问询、合同重组等类型，避免 23:00 冗余公告快讯。

## 六模块

| 模块 | 数据源 | 必须产出 | 用途 |
|---|---|---|---|
| 行情底座 | Mac tdxrs / 腾讯备份 | 指数、个股K线、盘口、D+窗口 | 判断环境和验证 |
| 全市场底表 | Mac tdxrs + 东方财富股票池缓存 | 全市场排名、成交额、市值、板块基础字段 | 中军、容量、全市场结构 |
| 热度预期差 | Mac同花顺热榜 + 通达信热榜 | Top100、热度分、热榜变化、概念标签、部分原因 | 发现预期差和散户关注度 |
| 涨停原因 | 通达信 ztfx + 同花顺原因 | 涨停原因原文、公告/新闻/概念来源 | 6维分析事实源 |
| 资金流向 | 通达信 zjlx / 成交额Top100 | 主力净额、超大单、大单、成交额 | 资金龙、中军龙、D+验证 |
| 监管异动 | K线预计算 + 异动公告 | 10日/30日涨幅、异动窗口、出监管日 | 龙头卖点和仓位权限 |

## 同花顺热榜口径

Mac 端已经能抓同花顺 Top100：

```text
raw/04-市场数据/同花顺热榜/YYYY-MM-DD/ths-hot-top100.json
raw/04-市场数据/同花顺热榜/YYYY-MM-DD/ths-hot-top100.md
```

龙虾/老虎事实层不必重复爬同花顺，优先通过 Syncthing 读取 Mac RAW。

如果本机RAW入口没有这份文件，只能说“同步缺口”，不能说“同花顺热榜未接入”。

## 涨停原因验收

不能只补连板票。

每日涨停全景必须覆盖全部非 ST 涨停票，并且每只票至少有一个原因字段：

```text
涨停原因原文
原因来源
所属板块
数据缺口
```

如果 L2 涨停全景已有完整原因原文，可以不逐票重复调 ztfx；如果原因缺失，必须用 ztfx 补齐。

首板是新主线发现入口，不能因为不是连板就不写原因。

## 资金流向验收

资金流向不要求每只首板都逐票深拉，但必须覆盖两个层级：

1. 连板票：全部覆盖。
2. 首板重点池：必须覆盖。

首板重点池定义：

- 同花顺热榜 Top100 与首板交集。
- 通达信热榜 Top100 与首板交集。
- 成交额 Top100 与首板交集。
- 板块强度前 10 里的首板。
- 20cm、30cm、北交所首板。
- 封单金额前 20 或成交额前 20 的首板。

资金字段至少包括：

```text
主力净额
超大单净额
大单净额
成交额
主力净额占成交额比例
数据来源
数据缺口
```

如果接口支持批量拉全量涨停资金流，可以全量覆盖；如果只能逐票拉，先按上述重点池覆盖。

## 监管异动验收

监管异动是核心交易规则，必须从 K 线预计算，不能只靠人工感觉。

每天至少对这些股票计算：

- 连板票。
- 作战室候选票。
- 成交额 Top100 与热榜 Top100 共振票。
- 当日涨停全景里的高辨识度票。
- 当前持仓票。

输出字段：

```text
股票代码
股票名称
10日涨幅
30日涨幅
是否接近10日100%
是否接近30日200%
最近异动公告日期
预计出监管日期
是否出监管日
数据来源
数据缺口
```

输出路径：

```text
raw/04-市场数据/监管异动/YYYY-MM-DD/tdx-regulatory-move.json
raw/04-市场数据/监管异动/YYYY-MM-DD/tdx-regulatory-move.md
```

## 22:00 公告任务验收清单

按 [[公告全量RAW与Codex提取规则]] 执行。

22:00 每日公告落盘至少覆盖：

- 并购重组。
- 业绩预告。
- 减持增持。
- 异动公告。
- 问询函/关注函/监管函。
- 重大合同。
- 定增/可转债/融资事项。
- 停复牌。

输出路径：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.json
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.md
```

Mac本机RAW入口只负责原始公告 RAW；Codex 读取后输出：

```text
raw/11-Codex分析产物/公告催化/YYYY-MM-DD/公告催化初筛.json
raw/11-Codex分析产物/公告催化/YYYY-MM-DD/公告催化初筛.md
wiki/07-作战室/YYYY-MM-DD-公告催化初筛.md
```

如果公告实际披露晚于 22:00，23:00 不另建重复任务；由 22:00 任务内部重试或次日早盘补拉。

## 不合格情况

以下都算不合格：

- 只拉连板原因，不拉首板原因。
- 首板催化只写空壳或只有题材名，没有原因原文。
- 资金流只覆盖连板，不覆盖首板重点池。
- 同花顺热榜 Mac 已有，却在本机RAW入口报“未接入”。
- 监管异动没有预计算。
- 公告只拉并购重组，不拉异动/减持/问询/业绩预告。
- 用模型分析替代事实字段。
