# 数据接口统一总控

```yaml
type: 系统配置
date: 2026-07-04
owner: Codex
registry: .system/data-interface-registry.json
status_script: raw/07-系统脚本/codex_data_interface_control.py
```

## 定位

本页是 73神话所有数据接口的统一入口。

以后新增接口、改接口、停接口，必须先改这里对应的登记表，不允许只散落在某个脚本或某个飞书对话里。

机器可读登记表：

```text
.system/data-interface-registry.json
```

接口状态检查脚本：

```text
python3 raw/07-系统脚本/codex_data_interface_control.py --date YYYY-MM-DD --write
```

运行后输出：

```text
.system/data-interface-health.json
wiki/09-统计与进化/YYYY-MM-DD-数据接口运行总控报告.md
```

## 总原则

1. 所有数据先写 RAW，不直接写 WIKI 结论。
2. Mac 能自动拉的接口，由 Mac 本机 LaunchAgent 或 Codex 脚本执行。
3. 通达信/市场事实层接口优先由 Mac 本机脚本执行；暂未自动化的由用户手动导入 RAW，不再依赖 Windows、云服务器或外部事实层。
4. 用户可见字段必须中文化，不用 `limitUpCount` 这类英文表头。
5. JSON 和 MD 尽量同目录双写。
6. ST 股不进入连板天梯和作战室主分析。
7. 缺数据时标注缺口，不伪造。
