# 编码治理与高频页面写入规则
更新时间：2026-06-20

## 目标

保证 73 交易大脑当前最常看的页面和主链 JSON 不再因为编码问题影响工作。

## 当前规则

1. 高频入口文件统一按 `UTF-8` 写入。
2. JSON、JSONL、Markdown 一律按 `utf8` 写入，不允许混用本地代码页。
3. 终端显示乱码，不能直接等同于文件损坏；文件真实性以编辑器打开结果和结构校验为准。
4. 高频入口优先治理，历史全库治理排后。

## 高频治理范围

当前纳入最小闭环的高频文件：

1. `wiki/00-总纲/总索引.md`
2. `wiki/10-系统配置/2026-06-18-现行制度总目录.md`

8. `.llm-wiki/system-health/latest-system-health.json`

## 验收方式

固定命令：

```text
npm run audit:encoding -- --project /Users/qixinchaye/wiki/73神话
```

通过标准：

1. 文件存在
2. 无明显 replacement char
3. 无异常控制字符
4. JSON 文件可解析

现在先保高频入口不坏，再谈历史全库清洗。
