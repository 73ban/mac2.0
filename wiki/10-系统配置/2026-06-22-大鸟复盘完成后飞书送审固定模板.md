# 2026-06-22 大鸟复盘完成后飞书送审固定模板

更新时间：2026-06-22

## 目的

统一大鸟在完成今日复盘或历史复盘后的飞书送审格式。

目标不是写漂亮消息，而是让用户能快速审核：

1. 这是什么复盘
2. 用了哪些资料
3. 有没有数据缺口
4. 是否存在后视镜风险
5. 当前版本能不能入 WIKI

## 适用范围

1. 今日复盘
2. 历史复盘

## 总原则

1. 必须发 `全文`
2. 不得只发摘要
3. 必须明确写当前状态：`待审核，未写入 WIKI`
4. 用户未明确确认前，不得入 WIKI

## 模板 A：今日复盘送审

```text
今日复盘已完成，先发你全文审核。

复盘类型：今日复盘
目标日期：YYYY-MM-DD
原始资料：
1. 交割单：____
2. 口述：____
3. 截图：____
4. 当日计划：____
5. 其他证据：____

数据缺口：无 / 有，缺口如下：____
当前状态：待你审核，未写入 WIKI

以下是复盘全文：
____
```

## 模板 B：历史复盘送审

```text
历史复盘已完成，先发你全文审核。

复盘类型：历史复盘
目标日期：YYYY-MM-DD
reviewTradeDate：YYYY-MM-DD
evidenceCutoffDate：YYYY-MM-DD
lookaheadForbidden：true

原始资料：
1. 交割单：____
2. 当日计划：____
3. 当日复盘原文：____
4. 聊天记录：____
5. 截图：____
6. 外部补证：腾讯财经 / AKShare / 热榜 / iWenCai / iFinD / 财联社

数据缺口：无 / 有，缺口如下：____
后视镜检查：已通过 / 未通过
当前状态：待你审核，未写入 WIKI

以下是复盘全文：
____
```

## 模板 C：用户要求修改后再次送审

```text
已按你的意见修改，重新发你全文审核。

复盘类型：今日复盘 / 历史复盘
目标日期：YYYY-MM-DD
本轮主要修改：
1. ____
2. ____
3. ____

数据缺口：无 / 有，缺口如下：____
当前状态：待你审核，未写入 WIKI

以下是修改后全文：
____
```

## 模板 D：用户通过后回执

```text
已按你确认版本写入 WIKI。

复盘类型：今日复盘 / 历史复盘
目标日期：YYYY-MM-DD
写入时间：____
更新位置：
1. ____
2. ____
3. ____
```

## 必须出现的字段

每次送审消息，最少必须出现：

1. `复盘类型`
2. `目标日期`
3. `原始资料`
4. `数据缺口`
5. `当前状态：待你审核，未写入 WIKI`
6. `复盘全文`

历史复盘额外必须出现：

1. `reviewTradeDate`
2. `evidenceCutoffDate`
3. `lookaheadForbidden：true`
4. `后视镜检查`

## 禁止事项

1. 只发一句“复盘好了”
2. 只发结论，不发全文
3. 未写明数据缺口
4. 历史复盘未写后视镜检查结果
5. 用户未确认就默认入 WIKI

## 一句话

大鸟复盘完成后，飞书发你的内容必须是：

`有日期、有资料来源、有缺口说明、有状态标记、有全文正文的待审稿`
