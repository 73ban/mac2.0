# 2026-06-28 GPT Pro / Codex 外部处理协议

## 定位

73wiki 不再被当成资料仓库，而是交易操作系统。

GPT Pro / Codex 的角色不是接入 73wiki 内置 LLM API，而是作为外部智能体直接读写本地工作目录：

- 工作目录：`/Users/qixinchaye/wiki/73神话`
- 原始资料入口：`raw/`
- 结构化结论入口：`wiki/`
- 运行中间产物：`.llm-wiki/`

## 成本规则

- 默认不使用 OpenAI API。
- 默认不使用 `gpt-5.5` API。
- 不把完整 API key 写入任何 WIKI 正文。
- GPT Pro / Codex 通过当前会话直接处理本地文件，作为主力处理方式。
- 73wiki 桌面应用主要负责浏览、展示、目录、看板和人工检查。

## 固定任务入口

用户临时任务、待处理材料、口头指令整理稿，统一放入：

`raw/00-GPTPro任务/`

Codex 内部运行记录、队列、归档，统一放入：

`.llm-wiki/gpt-pro-codex/`

## 默认处理顺序

1. 先读 `wiki/10-系统配置/` 的制度与当前任务约束。
2. 再读相关 `raw/` 原始材料。
3. 只把可复用、可验证、可执行的内容写入 `wiki/`。
4. 资料型内容优先留在 `raw/`，不要把 73wiki 写成资料仓库。
5. 涉及交易方法、纪律、错误、验证、进化的内容，落到对应 L1-L4、错误库、统计与进化目录。

## 固定产出类型

- 每日复盘：写入 `raw/02-每日复盘/` 与必要的 `wiki/` 结构化更新。
- 交易计划：写入 `raw/03-每日计划/` 与 `wiki/07-作战室/`。
- 个股沉淀：写入 `wiki/03-L3个股档案/`。
- 模式沉淀：写入 `wiki/04-L4交易模式与执行/`。
- 错误归因：写入 `wiki/05-错误库/`。
- D+1 / D+3 / D+5 / D+10 / D+20 / D+30 验证：写入 `wiki/09-统计与进化/`。
- 系统规则：写入 `wiki/10-系统配置/`。

## 执行纪律

- 不为凑内容而总结。
- 不把未验证观点写成规则。
- 不把一次交易经验写成稳定模式。
- 所有模式升级必须有样本、反例、适用市场环境和失效条件。
- 涉及未来判断时，必须标注计划日期、验证日期和触发条件。
- 发现资料重复、错位、过时，先标记审计结果，再决定合并、保留、删除或迁移。
