# 73神话目录规则

更新时间：2026-06-28

## 总原则

73神话不是资料仓库，而是超短线交易操作系统。

目录规则只有一个核心：

```text
raw 只放原始输入
wiki 只放加工后的交易操作系统成果
data 只放底层事实、学习层和训练材料
项目配置只放系统入口文件
```

任何文件进入 WIKI 前，都必须回答一个问题：

> 它是否已经从原始资料变成了可执行、可验证、可复盘、可进化的交易材料？

如果答案是否定的，先放 RAW，不直接写 WIKI。

## 文件树顺序

桌面 Wiki 文件树固定显示：

```text
raw
wiki
项目配置
data
```

`项目配置` 显示根目录下的核心入口文件：

- `purpose.md`
- `schema.md`
- `73wiki.config.json`
- `73wiki.system.json`

## RAW 目录规则

RAW 是事实入口，保存原始材料、待处理资料和采集结果。

```text
raw/00-GPTPro任务
raw/01-交割单
raw/02-每日复盘
raw/03-每日计划
raw/04-市场数据
raw/05-研报新闻
raw/06-规则原文
raw/07-系统脚本
raw/08-截图
raw/09-短线知识
```

| 目录 | 用途 |
|---|---|
| `raw/00-GPTPro任务` | 给 Codex/GPT Pro 的任务单、批处理要求 |
| `raw/01-交割单` | 成交、持仓、资金、原始交易凭证 |
| `raw/02-每日复盘` | 每日复盘原稿、心理记录、纪律记录 |
| `raw/03-每日计划` | 明日计划、盘前计划、作战计划原稿 |
| `raw/04-市场数据` | 热榜、涨停原因、指数、题材、行情数据 |
| `raw/05-研报新闻` | 公众号、财联社、知识星球、央媒、公告、纪要 |
| `raw/06-规则原文` | 杨明杰规则、旧规则、外部方法论原文 |
| `raw/07-系统脚本` | 脚本、模板、工具，默认不自动进入交易知识库 |
| `raw/08-截图` | 待 OCR、交易截图、热榜截图 |
| `raw/09-短线知识` | 飞书输入、游资公众号、截图 OCR、手工摘录等短线学习原始材料 |

RAW 规则：

- 新资料先进 RAW。
- RAW 保留原貌，不强求格式完美。
- RAW 文件尽量带日期、来源、主题。
- RAW 不能替代 WIKI 结论。
- 自动采集工具只能先写 RAW，不能直接写正式交易规则。

## WIKI 目录规则

WIKI 是加工后的交易操作系统成果。

```text
wiki/00-总纲
wiki/01-L1市场环境
wiki/02-L2方向题材
wiki/03-L3个股档案
wiki/04-L4交易模式与执行
wiki/05-错误库
wiki/06-持仓与资金管理
wiki/07-作战室
wiki/08-信息来源
wiki/09-统计与进化
wiki/10-系统配置
```

| 目录 | 用途 |
|---|---|
| `wiki/00-总纲` | purpose、总规则、纪律、目标、禁止事项 |
| `wiki/01-L1市场环境` | 指数、情绪、周期、风险、空仓条件 |
| `wiki/02-L2方向题材` | 主线、题材生命周期、事件催化、产业逻辑 |
| `wiki/03-L3个股档案` | 核心股、备选股、个股作战卡 |
| `wiki/04-L4交易模式与执行` | 买点、卖点、仓位、模式、执行规则 |
| `wiki/05-错误库` | 错误、亏损归因、违规交易、重复问题 |
| `wiki/06-持仓与资金管理` | 持仓、资金曲线、仓位变化、交割单加工结果 |
| `wiki/07-作战室` | 今日作战、明日计划、候选池、临盘决策 |
| `wiki/08-信息来源` | 外部资料卡、资料可信度、来源索引 |
| `wiki/09-统计与进化` | 交易统计、D+验证、自学习、月度进化 |
| `wiki/10-系统配置` | 目录规则、脚本说明、提示词、系统制度 |

WIKI 规则：

- WIKI 不存未经加工的原始资料。
- WIKI 不存重复新闻流水账。
- WIKI 页面必须有明确用途：规则、复盘、错误、验证、作战、统计、进化。
- 新规则必须说明适用环境、触发条件、退出条件和验证方式。
- 不确定内容先进入 `99-待删除审核`、`99-迁移` 或 RAW，不直接混入正式目录。
