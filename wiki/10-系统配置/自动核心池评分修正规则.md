# 自动核心池评分修正规则

更新时间：2026-07-01

## 定位

本页约束 `RAW增量个股核心池清单`、作战室候选票评分和后续自动选股脚本。自动分数只能作为候选线索，不能直接变成买入权限。

核心目标：

- 服务 20 万小资金快速做大，优先控制一次大亏。
- 让 AI 从 D+验证、实盘盈亏和错误库中反向修正，不重复高分低质信号。

## 评分优先级

自动分数的使用顺序必须是：

1. L1 市场环境是否允许进攻。
2. L2 主线是否仍在主升、分歧修复或容量抱团。
3. 个股是否是真核心、弹性核心、容量核心或补涨核心。
4. 来源质量是否可靠，是否存在低信号来源、乱码、泛题材映射。
5. D+1/D+3/D+5/D+10/D+20/D+30 是否验证通过。
6. 是否和当前持仓、仓位、风控冲突。

## 双分制：基本面纯度与短线情绪权重

所有消息催化票必须拆成两套分数，禁止用基本面纯度直接覆盖短线情绪。

```yaml
fundamental_purity: 0-100
emotion_weight: 0-100
market_validation: 一字板/大单一致/涨停/连板/板块扩散/热榜跃迁/暂无
```

处理原则：

- 基本面纯度高、情绪权重高：优先复核，可能进入作战室。
- 基本面纯度高、情绪权重低：保留中线或等待盘口确认。
- 基本面纯度低、情绪权重高：定义为情绪票或补涨票，不能因为“不纯”直接低估。
- 基本面纯度低、情绪权重低：降级观察或删除。

一字板、大单一致、连板晋级、板块扩散出现时，必须优先上修 `emotion_weight`；只有在消息映射明确、板块扩散成立、核心地位确认后，才同步上修 `fundamental_purity` 或题材级别。

## 消息权重的市场回填规则

Codex 初评消息后，必须用当天和次日市场表现回填分数。

| 市场验证 | 修正动作 |
|---|---|
| 一字板 | 消息级别不得低于“重大/高权重”，短线情绪权重至少 85 |
| 大单封死 | 短线情绪权重至少 80，记录封单和题材映射 |
| 连板晋级 | 个股地位上调一档，进入连板/补涨核心观察 |
| 板块多股涨停 | 题材级别上调一档，寻找核心和补涨梯队 |
| 热榜快速跃迁 | 注意力权重上调，进入预期差复核 |
| 无资金响应 | 消息权重下调，除非属于滞后兑现型催化 |
| 高开低走/炸板失败 | 保留消息，但盘口质量扣分，等待 D+3 修复 |

若 AI 文本判断为“一般/弱”，但市场给出一字、连板或大单封死，必须承认文本低估，并写入验证队列。

一字板补充口径：

- 一字板只强制上修 `emotion_weight`，不自动上修 `trade_permission`。
- 复牌、重组、停牌补涨、极小流通盘、无板块扩散的一字，必须额外打 `execution_risk`。
- 连续一字没有分歧买点时，作战室权限默认为观察；只有开板承接、回封、板块扩散或核心带动被验证后，才允许重新评审为可执行候选。
- 一字开板后高开低走、封单衰减或板块跟随失败，必须回写为“强一致兑现风险”，不能继续按高权重惯性打分。

## 强制降级规则

### D+失败降级

满足以下任一条件，自动核心池不得直接进入买入计划：

- D+1 收盘弱于预期，且相对所属板块核心明显偏弱。
- D+3 未修复 D+1 弱势。
- D+5 仍不能重新站回主线强度。
- D+10/D+20/D+30 证明只是事件脉冲，不具备持续性。

处理：

```yaml
validation_status: 降级
use_grade: C
trade_permission: 观察
score_penalty: -80
required_repair: D+3/D+5 强修复后重新评审
```

### 高分低信号降级

满足以下任一条件，自动分数必须打折：

- 催化摘要里出现 `low-signal`、`etf-generic`、`facts-only source`。
- 来源主要来自泛热榜、ETF映射、公众号跟风、乱码抽取。
- 题材数很多，但没有明确个股核心地位。
- 来源数很多，但证据只证明“被提到”，不能证明“能交易”。

处理：

```yaml
source_quality_floor: B
trade_permission: 观察
score_penalty: -40
must_have: 独立催化 + 板块地位 + 盘口验证
```

### 持仓冲突降级

若候选股和当前持仓方向冲突，或需要换仓但旧仓没有明确卖点，不能因为新候选高分而加仓。

处理：

```yaml
trade_permission: 禁止直接换仓
required_action: 先处理旧仓，再评估新仓
linked_error: 2026-06-10 外围利空下卖点拖延与换仓过度
```

## 新易盛样本

`300502 新易盛` 是第一条正式反向修正样本：

- 原始信号：A+，自动分数 634.8。
- 问题：存在 `Downgraded low-signal source: etf-generic`，且来源里有 `market-community`、`wechat-post`、`cls-news` 扣分。
- D+1 结果：2026-06-26 收盘 566.00，跌 7.29%，价格验证弱于预期。
- 处理：保留 CPO/光模块长期观察身份，短线作战权限降级为观察，等待 D+3/D+5 修复。

## 作战室使用规则

- A+ 自动分数只代表优先复核，不能代表可以买。
- D+失败后的 A+ 必须按 C 级观察处理，直到出现修复证据。
- 小资金只能把高分候选当成观察池，不能把分数当成仓位理由。
- 每次 D+失败都要回写一条扣分规则；连续成功才允许提高仓位权限。

## 与两个总目标的关系

- 对 20 万小资金：宁可错过高分票，也不能让高分低质票制造大回撤。
- 对 AI 超短进化：每次预测失败都必须沉淀为可执行的降级规则，而不是只写总结。
