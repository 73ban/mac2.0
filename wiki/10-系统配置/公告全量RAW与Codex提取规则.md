# 公告全量 RAW 与 Codex 提取规则

```yaml
type: 系统配置
date: 2026-07-04
owner: Codex
status: active
```

## 定位

22:00 公告任务由Mac本机RAW入口负责原始数据落盘，Codex 负责二次分析。

Mac本机RAW入口不判断公告好坏，只交原始公告。

Codex 不相信单条公告本身，必须和持仓、作战室、热榜、涨停结构、成交额、资金流和龙虎榜交叉。

## Mac本机RAW入口 22:00 拉什么

用 `wenda_notice_query` 分 4 批关键词拉：

| 批次 | 关键词 | 分类标签 |
|---:|---|---|
| 1 | 业绩预告、业绩快报、业绩修正 | 业绩 |
| 2 | 减持、增持、回购 | 增减持 |
| 3 | 异动公告、问询函、关注函、立案调查 | 监管 |
| 4 | 重大合同、中标、定增、重组、并购 | 重组 |

输出：

```text
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.json
raw/05-研报新闻/公告/YYYY-MM-DD/daily-announcements.md
```

## 原始字段要求

每条公告必须保留：

| 字段 | 用途 |
|---|---|
| 公告标题 | 判断公告类型和核心事件 |
| 公司名称 | 关联个股 |
| 股票代码 | 关联持仓、作战室、热榜、涨停票 |
| 公告日期 | 判断是否当晚新公告 |
| 分类标签 | 业绩/增减持/监管/重组 |
| 命中关键词 | 判断为什么被拉入 |
| 公告摘要 | 作为初筛文本 |
| 公告链接 | 可复核原文 |
| 原始字段 | 保留工具原始返回 |
| 数据缺口 | 缺字段必须明示 |

## Codex 提取什么

Codex 读取公告 RAW 后，提取 5 类结果：

| 类别 | 说明 |
|---|---|
| 持仓公告 | 当前持仓相关公告，优先提醒 |
| 作战室公告 | 作战室候选票相关公告，影响次日计划 |
| 热榜公告 | 同花顺/通达信热榜 Top100 相关公告，判断预期差 |
| 涨停公告 | 当日涨停、首板、连板相关公告，验证涨停原因 |
| 风险公告 | 减持、立案、问询、异动澄清、亏损修正等风险 |

输出：

```text
raw/11-Codex分析产物/公告催化/YYYY-MM-DD/公告催化初筛.json
raw/11-Codex分析产物/公告催化/YYYY-MM-DD/公告催化初筛.md
wiki/07-作战室/YYYY-MM-DD-公告催化初筛.md
```

## 评分边界

公告本身只给线索，不直接给买入权限。

必须看三层确认：

1. 公告是否实质改变预期。
2. 是否已经有涨停/热榜/成交额/龙虎榜验证。
3. 次日竞价和开盘承接是否确认。

## 高优先公告

优先级高：

- 持仓票重大利好或重大风险。
- 作战室候选票公告。
- 连板高标的异动公告、澄清公告、监管函。
- 热榜Top100里突然出现的重大合同、并购、定增、业绩超预期。
- 涨停原因与公告互相验证。

优先级低：

- 常规董事会、股东大会、章程修订。
- 无金额、无客户、无进度的泛泛合作协议。
- 已经被市场充分反应的旧公告。

## 不允许

- 不允许把公告标题直接当交易结论。
- 不允许把“重大合同”一律当利好。
- 不允许忽略减持、问询、立案、异动澄清。
- 不允许只看持仓，不看热榜和涨停票。
- 不允许在公告没有市场验证前提高仓位权限。
