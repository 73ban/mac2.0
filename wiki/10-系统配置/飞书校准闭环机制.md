# 飞书校准闭环机制

更新时间：2026-07-05

## 目标

用户在飞书里纠正模型时，系统立即把纠正变成训练信号，而不是只当普通聊天。

注意：飞书交易沟通不全部等于短线知识。消息重要性提问、分类提问、交互偏好、任务沟通，先按原始沟通写入 `raw/10-飞书交易沟通/YYYY/MM/DD/`；只有经过用户确认或市场验证后，才提炼到 `raw/09-短线知识/`。

## 飞书消息四类

1. 通知型：系统状态、缺口、Watchdog、任务完成。必须写明“无需回复/不用回复”，不要求用户判断。
2. 校准型：重大消息、题材、个股、游资学习样本。必须让用户只回复 `第几条 + 有效/一般/无效/反向 + 原因`。
3. 复盘型：用户口述交易、情绪、买卖理由。只保真写入 RAW，不抢先写成正式结论。
4. 命令型：重跑、暂停、切模式、查看状态。直接执行或回报不能执行的原因。

任何飞书不能混淆“判断消息”“判断股票”“判断学习样本”“系统告警”。如果混淆，发送前必须隔离。

## 校准型消息固定合同

校准型飞书每条必须包含：

- 判断对象：明确这次是在判断消息、题材、个股、学习样本，还是系统规则。
- 当前结论：有效/一般/无效/反向，或我当前判断为待理解。
- 我的判断逻辑：我为什么给这个等级，最多 3 条人话。
- 可能价值：它可能如何影响选股、题材理解、情绪周期或交易规则。
- 不确定点：我哪里可能高估、低估或还没理解。
- 用户动作：只回复编号和判断，不要求用户读长篇原文后猜我想问什么。

发送前由 `raw/07-系统脚本/codex_feishu_protocol_lint.py` 检查。缺少判断对象、回复格式、当前判断、判断逻辑、不确定点，或超过 10 条，直接移入 `.system/feishu-notify-quarantine/`，不发送。

## 自动动作

命中后不调用大模型，桥接器本机直接完成：

1. 原话写入 RAW。
2. 结构化写入 `data/facts/feishu_calibration_events.jsonl`。
3. 追加到当日 `wiki/09-统计与进化/YYYY-MM-DD-消息权重人工校准表.md`。
4. 需要验证的追加到 `wiki/09-统计与进化/消息权重人工校准待验证队列.md`。
5. 规则或回复偏好追加到 `wiki/10-系统配置/记忆进化日志.md`，先作为待验证规则，不直接覆盖稳定记忆。

其中回复偏好现在会直接写入当前飞书会话的持久偏好，后续回答自动生效，不用每次重复说。
另外，用户反复强调的要求会写入“会话记忆笔记”，回答前先和最近对话一起读取，避免出现“明明聊过却说缺少前文”的情况。

统一落库脚本：

```bash
python3 raw/07-系统脚本/codex_ingest_feishu_message.py --type auto --stdin
```

脚本会把普通沟通写入 `raw/10-飞书交易沟通/`，把复盘口述追加到 `raw/02-每日复盘/YYYY-MM-DD-飞书复盘RAW.md`，把校准类事件写入 `data/facts/feishu_calibration_events.jsonl`。

当用户按编号回复时，系统额外写入：

- `data/facts/feishu_calibration_events.jsonl`：逐条结构化事件，包含 item_index、user_judgement、reason、weight_change、needs_validation。
- `wiki/09-统计与进化/YYYY-MM-DD-飞书条目校准表.md`：当天校准表。
- `wiki/09-统计与进化/消息权重人工校准待验证队列.md`：只有有效、反向、高估、低估等需要 D+验证的条目进入。

## 消息判断类问题

当用户问：

```text
这个消息重不重要？
这个算什么？
这条能不能刺激涨停？
这条是不是重大消息？
```

这不是普通短线知识，而是“消息权重判断请求”。默认处理：

1. 原始问答写入 `raw/10-飞书交易沟通/YYYY/MM/DD/`。
2. 如果用户纠正判断，再写入 `data/facts/feishu_calibration_events.jsonl`。
3. 如果用户确认“以后按这个判断”，再提炼到 `raw/09-短线知识/` 和 `wiki/10-系统配置/记忆进化日志.md`。

回复必须包含：

```text
结论：重要/一般/不重要/反向/待验证
归类：政策/产业/订单/涨价/供需/公告/澄清/市场信号/情绪信号/个股材料
理由：2-3条
验证点：涨停、热榜跃迁、板块扩散、连板晋级、竞价强度
```

## 会话模式

飞书里还可以直接切模式：

```text
切到快答模式
切到分析模式
切到复盘模式
切到盯盘模式
恢复默认模式
当前是什么模式
当前会话记住了什么
查看上下文
```

模式含义：

- 快答模式：优先短答，能秒回的先秒回。
- 分析模式：优先走深任务，回答更完整。
- 复盘模式：适合连续口述、总结、整理。
- 盯盘模式：优先高信号、短判断，不拉太长。
- `当前会话记住了什么` / `查看上下文`：本机直回当前模式、会话要求笔记、最近对话摘要，用来检查助手有没有真正承接上下文。

## 回复规则

飞书只回短结果：

```text
已降权。
类似消息先降权，除非出现涨停、热榜跃迁、板块扩散或连板晋级验证。
后续用涨停、热榜跃迁、板块扩散、连板晋级来验证。
```

不默认展示：

```text
job_id
内部路径
桥接过程
后台日志
```

但有一个例外：

- 只要是在判断消息、题材、个股权重，不能只回“有效/一般/无效/反向”。
- 必须同步说出简短判断理由，不少于 2 条，不多于 3 条。
- 理由必须是可校准的，比如“消息纯度不够”“映射票太多”“缺少涨停验证”“一字板说明被资金确认”。

## 进化原则

- 用户判断优先级高于模型判断。
- 单次校准先进待验证；重复命中或用户明确确认后，再升级为稳定记忆。
- 消息强弱最终接受资金验证：涨停、一字板、大单封死、热榜跃迁、板块扩散、连板晋级。
- 回复偏好也进入校准闭环，因为飞书体验本身就是系统的一部分。
- 每次偏好更新后，必须回显“改了什么、当前模式是什么、后续怎么生效”，避免用户不知道系统有没有真正记住。
- 用户要校准我的前提，是先看见我为什么这样判断。所以权重判断默认输出“判断对象 + 结论 + 理由 + 验证点”，不允许只扔结论。
