# Codex每日启动检查清单

本页是 Codex 执行交易系统任务前看的内部检查清单。

它不是给用户日常浏览的首页，也不替代 `00-每日工作台`。

## 零、先读最新交接

新 CLI 窗口或上下文不连续时，先读：

```text
AGENTS.md
wiki/10-系统配置/2026-07-05-Codex新窗口交接单.md
MEMORY.md
wiki/10-系统配置/截图OCR自动化规则.md
wiki/10-系统配置/淘股吧实盘赛样本学习规则.md
```

如果用户只说“继续”，默认先按最新交接单的 P0 任务推进。

## 一、先确认当前任务类型

| 用户意图 | 处理方式 |
| --- | --- |
| 问怎么看 | 先给结论，再说明依据 |
| 让我继续干活 | 直接查文件、补流程、写入 WIKI 或 RAW |
| 让我验收数据到没到 | 先看 Mac 本机任务、RAW目录和用户是否已提供原始资料，再判断质量 |
| 让我做复盘 | 先读交割单、口述、市场数据、作战室、D+验证 |
| 让我改系统 | 先确认是否 Mac-only，不再沿用旧 Windows、云服务器或外部事实层路径 |

## 二、先看系统状态

- 数据接口健康：[[2026-07-05-数据接口运行总控报告]]
- 数据接口职责：[[数据接口统一总控]]
- 截图 OCR：[[截图OCR自动化规则]]，状态见 [[OCR截图识别能力体检]]
- 淘股吧高手样本库：[[淘股吧实盘赛样本学习规则]]，先验结构再扩规模。
- Syncthing 状态：从数据接口健康报告确认，不凭口头判断。
- 当日是否交易日：非交易日不强行要求交割单、收盘数据、当日复盘。

## 四、作战室任务必须检查

- 候选票是否有来源。
- 候选票是否有触发条件。
- 候选票是否有失效条件。
- 是否能写入 D+验证队列。
- 是否有盘中重大消息需要推翻原计划。

## 五、不能再犯的错误

- 不能把“Mac 没收到文件”直接说成“外部事实层没做”；当前先排查本机任务、RAW入口和用户是否已提供原始资料。
- 不能把 RAW 空壳当作已完成。
- 不能把旧服务器、旧 Windows 路径、旧目录口径当作当前事实。
- 不能只写结论不写可验证依据。
- 不能漏掉交割单和复盘必须写入正式 WIKI 这条硬规则。
- 不能把脚本执行成功当成业务完成，改完系统文件必须看变更摘要或 diff。
- 不能让高风险自动执行污染主链路，批量删除、同步改向、历史口径重写、新工具替换稳定链路都要先隔离或 dry-run。

## 六、输出要求

默认给用户短结论。

需要写文件时，直接写，不只给方案。

需要验收时，必须区分：

- 文件是否存在；
- 文件是否同步到 Mac；
- 文件是否为空壳；
- 字段是否完整；
- 结论是否能支持交易决策。
