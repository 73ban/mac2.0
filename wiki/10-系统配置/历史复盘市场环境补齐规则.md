# 历史复盘市场环境补齐规则

```yaml
type: 系统配置
date: 2026-07-04
owner: Codex
status: active
scope: 历史复盘 / 月度统计 / 错误库 / 模式进化
```

## 定位

历史复盘不能只用事后涨跌判断对错。

每一笔交易必须放回当时的市场环境里判断：当时是主升、分歧、退潮、修复，还是混沌轮动；当时谁在主导市场；当时赚钱效应在哪里；当时监管和政策环境是否宽容。

否则很容易出现两个误判：

1. 事后看到某只票走成龙头，就倒推当时应该买。
2. 事后看到某只票下跌，就忽略当时它在盘面里的合理性或危险信号。

## 三层复盘结构

| 层级 | 负责回答 | 主要来源 | 是否允许推断 |
|---|---|---|---|
| 交易事实层 | 当天买了什么、卖了什么、仓位多少、盈亏多少 | 交割单、持仓、委托、用户口述 | 不允许 |
| 市场环境层 | 当天市场处于什么周期、主线在哪里、赚钱亏钱效应是什么 | 指数、成交额、涨跌家数、涨跌停、连板、板块、热榜、龙虎榜、新闻 | 允许低置信推断，但必须标注 |
| 归因判断层 | 这笔交易是好模式、坏模式、执行问题还是环境错配 | 前两层证据 + 系统规则 | 允许，但必须给证据 |

历史复盘必须先补市场环境层，再写归因判断层。

## tdxrs 能满足什么

`tdxrs` 可以满足历史复盘的一部分要求，但不能单独满足全部要求。

| 需求 | tdxrs 是否适合 | 用法 |
|---|---|---|
| 个股日 K / 分钟 K | 适合 | 补交易票、候选票、龙头票的量价轨迹 |
| 指数 K 线 | 适合 | 判断指数环境、是否系统性风险 |
| 五档盘口 / 开盘承接 | 适合向前采集 | 用于今后盘中和竞价快照；旧历史若未采集，不强行补 |
| D+1 / D+3 / D+5 验证 | 适合 | 验证作战室和复盘判断是否兑现 |
| 全市场涨停/跌停/连板结构 | 不完整 | 优先用通达信事实层、每日涨停全景、连板天梯 |
| 同花顺热度 / 预期差 | 不适合 | 必须用同花顺热榜、公众号热门榜或通达信热榜补充 |
| 涨停原因文本 | 不适合 | 必须用通达信涨停原因、同花顺原因、新闻研报交叉验证 |
| 龙虎榜席位 / 机构游资 | 不适合 | 必须用龙虎榜全量数据 |
| 政策宽容度 / 监管环境 | 不适合 | 用公告、监管动态、交易所问询、财联社、研报新闻 |

结论：tdxrs 是量价和盘口底座，不是市场情绪和催化底座。

## 历史市场环境快照字段

每个关键交易日补齐一份市场环境快照，字段统一如下：

| 字段 | 说明 | 首选来源 |
|---|---|---|
| 日期 | 交易日 | 文件名 |
| 指数表现 | 上证、深成指、创业板、科创50等涨跌幅 | tdxrs / 东方财富 / 腾讯 |
| 全市场成交额 | A股总成交额 | 复盘原文 / 东方财富 / 通达信 |
| 涨跌家数 | 上涨、下跌、平盘 | 复盘原文 / 东方财富 |
| 涨停跌停 | 非ST涨停、跌停、炸板、封板率 | 通达信事实层 / 每日涨停全景 |
| 连板结构 | 最高板、2板以上数量、断板反馈 | 通达信连板天梯 |
| 主线板块 | 当天最强板块和持续板块 | 板块强度 / 热榜 / 复盘原文 |
| 板块涨跌前十 | 涨幅前10、跌幅前10 | 板块强度 |
| 赚钱效应 | 连板、趋势中军、20cm、低价、题材轮动等 | 涨停全景 / 热榜 / 复盘 |
| 亏钱效应 | 跌停、天地板、冲高回落、退潮杀跌 | 涨停全景 / 分时 / 复盘 |
| 主导资金判断 | 游资、机构、量化、散户情绪谁主导 | 龙虎榜 / 成交额Top100 / 中军走势 / 盘口 |
| 政策监管环境 | 监管、政策、停复牌、异动规则影响 | 公告 / 财联社 / 研报新闻 |
| 当日交易限制 | 当天哪些模式应升权或禁用 | 系统规则 + 环境判断 |
| 置信度 | 高/中/低 | 证据完整度 |
| 数据缺口 | 缺哪些数据，不得隐藏 | 人工列出 |

## 主导资金判断代理变量

| 主导力量 | 可观察证据 | 注意事项 |
|---|---|---|
| 游资主导 | 连板高度、首板晋级、龙虎榜游资席位、换手回封、小盘题材密集涨停 | 不能只看涨停数量，必须看晋级和断板反馈 |
| 机构主导 | 成交额Top100、中军趋势、机构龙虎榜、研报催化、行业龙头放量 | 机构票不一定涨停，中军承接更重要 |
| 量化主导 | 高频冲高回落、炸板多、盘口撤单快、尾盘异动、分时反复拉扯 | 没有逐笔委托时只能低置信判断 |
| 散户情绪主导 | 热榜高位、公众号/股吧热度、无机构资金但情绪扩散 | 热度高不等于可以买，需要封板和承接验证 |

## 数据源优先级

1. 已存在 RAW：交割单、每日复盘、盘中笔记、市场数据。
2. tdxrs：指数、个股、候选票、龙头票的 K 线、量价、D+验证。
3. 龙虾/老虎通达信事实层：涨停全景、连板天梯、涨停原因、板块强度、龙虎榜。
4. 同花顺 / 东方财富 / 腾讯：热榜、全市场底表、行情交叉验证。
5. 财联社 / 公众号 / 研报新闻：政策、事件催化、外围风险。
6. 用户口述：只用于当时真实主观理由；历史缺口不能由 AI 代编。

## 禁止事项

1. 禁止把事后走强票直接倒推成当时必买票。
2. 禁止用 tdxrs 价格数据替代热榜、涨停原因、龙虎榜。
3. 禁止把历史缺失的口述理由补写成用户当时想法。
4. 禁止没有市场环境就给交易贴最终错误标签。
5. 禁止只看个股涨跌，不看当天周期位置和赚钱效应。

## 执行顺序

1. 找出关键交易日：大赚、大亏、重仓、模式升级/降级、作战室候选。
2. 读取当天交割单和复盘原文。
3. 用 tdxrs 补指数、交易票、候选票、龙头票 K 线和 D+走势。
4. 用通达信事实层补涨停全景、连板、板块、龙虎榜。
5. 用新闻和公告补政策监管与重大催化。
6. 生成历史市场环境快照。
7. 再回写错误库、模式胜率、月度统计。

## 当前结论

tdxrs 能解决“当时价格和量能到底怎么走”的问题。

tdxrs 不能解决“当时市场为什么这样走、谁在主导、热度从哪里来”的问题。

历史复盘要做准，必须把 tdxrs、涨停全景、连板天梯、板块强度、龙虎榜、热榜和新闻放在一起看。
