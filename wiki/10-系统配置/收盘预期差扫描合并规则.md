## 结论

17:00 独立的“收盘预期差扫描”任务不保留为Mac本机RAW入口单独任务。

原因：

1. 它不是原始数据，而是分析产物。
2. 需要消耗模型 token。
3. 它依赖的数据已经由其他 RAW 覆盖。
4. Codex 盘后闭环会做更深的预期差分析。
5. 如果没有固定盘前预期基准，单独扫描容易变成事后解释。

## 保留什么

保留“收盘预期差分析”这个分析模块，但合并进 Codex 盘后闭环，不让Mac本机RAW入口单独跑。

输出位置：

```text
raw/11-Codex分析产物/预期差/YYYY-MM-DD/收盘预期差分析.md
raw/11-Codex分析产物/预期差/YYYY-MM-DD/收盘预期差分析.json
wiki/07-作战室/YYYY-MM-DD-收盘预期差与次日验证.md
```

## 必须有盘前基准

预期差不是“今天谁涨得多”，而是“实际走势超过了什么预期”。

因此 Codex 做收盘预期差时，必须读取或生成盘前基准：

```text
raw/03-每日计划/YYYY-MM-DD-盘前预期基准.md
raw/03-每日计划/YYYY-MM-DD-竞价监控清单.md
wiki/07-作战室/YYYY-MM-DD-作战室候选票评分表.md
```

如果没有盘前基准，输出必须标注：

```text
数据缺口：缺盘前预期基准，本次只能做“收盘强弱扫描”，不能严格称为预期差验证。
```

## 分析字段

收盘预期差分析必须输出：

| 字段 | 说明 |
|---|---|
| 方向/板块 | 机器人、黄金、算力、半导体等 |
| 盘前预期 | 盘前怎么看 |
| 收盘表现 | 指数、板块、涨停、热榜、成交额表现 |
| 超预期/不及预期 | 必须有证据 |
| 核心股票 | 龙头、中军、先锋、趋势票 |
| 催化来源 | 公告、新闻、热榜、涨停原因 |
| 次日验证点 | 周期能否延续 |
| 数据缺口 | 缺什么写什么 |

## Codex 负责

Codex 在盘后闭环里做：

1. 读取基础 RAW。
2. 读取盘前预期基准。
3. 生成收盘预期差分析。
4. 输出次日验证点。
5. D+1/D+3/D+5 复盘验证。
