# 学习层健康检查与制度候选池规则
更新时间：2026-06-20 14:10

## 目的

学习层不只是堆 RAW 和假设，必须持续回答 4 个问题：

1. 假设有没有到期未验证。
2. validation 能不能回链到 hypothesis。
3. corrections 有没有形成纠偏闭环。
4. 哪些方法已经具备进入“制度候选池”的资格。

## 固定产物

学习层固定产物：

1. `.llm-wiki/hypothesis-validation/latest-hypothesis-validation.json`
2. `.llm-wiki/brain-health/latest-brain-health.json`
3. `.llm-wiki/learning-layers/latest-learning-layers.json`
4. `wiki/09-统计与进化/当前学习层分层报告.md`

这些产物只做审计、分层、候选和提醒，不直接改写现行制度页。

## 固定命令

1. `npm run hypothesis:validate -- --project /Users/qixinchaye/wiki/73神话`
2. `npm run brain:health -- --project /Users/qixinchaye/wiki/73神话`
3. `npm run learning:layers -- --project /Users/qixinchaye/wiki/73神话`
4. `npm run brain:push -- --project /Users/qixinchaye/wiki/73神话`

## 学习层状态

只允许以下状态：

1. `学习观察`
2. `待验证`
3. `初步有效`
4. `已验证可用`
5. `冲突待提醒`

## 制度候选池门槛

只有同时满足以下条件，才允许进入制度候选池：

1. 来源明确。
2. 有真实样本。
3. 有失败样本复核。
4. 至少完成应到期的 `D1 / D3 / D5` 验证。
5. 能解释对“小资金超短做大”的实际价值。
6. 不与当前高优先级纠偏项正面冲突。

未满足以上条件前，只能留在学习层，不能冒充现行制度。

## 飞书提醒口径

只允许两类内容主动飞书提醒：

1. `冲突待提醒`
2. `制度候选`

不把全部学习噪音推给你。

## 检查点口径

假设验证不是只看最终 D5：

1. 先按数据地平线判断哪些检查点已经到期。
2. 到期的 `D1 / D3 / D5` 必须自动回填。
3. 没到期的不算逾期。
4. 到期但证据不足的，先留在 `待验证`，不得硬升层。

## 一句话

学习层要进化，就必须有健康页；制度要升级，就必须先过候选池。
