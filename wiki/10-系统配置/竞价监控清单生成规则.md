# 竞价监控清单生成规则

更新时间：2026-06-18

## 目的

竞价监控清单的作用，是让大鸟在竞价前先明确今天真正该盯哪些票、哪些风险锚、哪些异常变化会推翻计划。

Mac本机RAW入口如果在线，可以按清单做辅助采集；如果不在线，大鸟按同一套清单自行判断。

## 输出位置

```text
raw/03-每日计划/YYYY-MM-DD-竞价监控清单.md
```

## 必填分组

| 分组 | 来源 | 用途 |
|---|---|---|
| 作战室核心票 | 当天作战室 | 判断主计划能否执行 |
| 用户持仓票 | 持仓记录 | 判断是否需要保护或处理 |
| 人气热榜票 | 同花顺 / 开盘啦 / 东方财富 / 通达信 | 判断注意力和预期差 |
| 连板高标 | 连板天梯 | 判断短线情绪 |
| 题材中军 | L2 题材和 L3 个股档案 | 判断题材是否有容量支撑 |
| 并购重组票 | 并购重组 RAW | 判断是否有一字连板预期 |
| 风险锚 | 昨日高标、负反馈核心、风险方向 | 判断是否应降仓 |

## 每只票必须写明

```yaml
code:
name:
group:
theme:
role:
expected_open:
watch_at_0915:
watch_at_0925:
below_expectation:
above_expectation:
action_if_below:
action_if_above:
```

## 执行原则

1. 清单先服务大鸟自己判断，不只是给Mac本机RAW入口。
2. 本机快照可用时，按清单采集 9:15 / 9:25 辅助数据。
3. 本机快照缺失时，大鸟继续执行，只标注 `竞价数据缺口`。
4. 没有证据路径的数据，大鸟拒收。

## 硬规则

没有竞价监控清单，盘前判断视为准备不足。

没有完整竞价辅助快照时，大鸟的开盘判断必须标注“竞价数据缺失”，不得假装完整。
