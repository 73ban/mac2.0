# 四层嵌套决策体系 Schema

## 总体架构

```
L1 市场环境 → 今天能不能买？
  ↓ 通过
L2 方向概念 → 买什么方向？
  ↓ 通过
L3 个股档案 → 买哪只？
  ↓ 通过
L4 执行控制 → 买不买？仓位多少？什么时候卖？
```

## 目录映射

| 层级 | 目录 | 内容 |
|:----:|------|------|
| L1 | 01-市场环境/ | 大盘评估、情绪周期、仓位建议、市场数据 |
| L2 | 02-方向概念/ | 板块/概念/题材的逻辑链、核心标的、L4权限 |
| L3 | 03-个股档案/ | 所有个股档案（公司简介+炒作逻辑+风险+L4） |
| L4 | 04-交易模式/ + 06-策略仓库/ | 买入/卖出模式、执行SOP、仓位规则、止损纪律 |
| — | 05-错误库/ | 每次亏钱必须记录，闭环反馈 |
| — | 07-信息摄入/ | 每日复盘、交割单、早报、竞价快照 |
| — | 09-持仓管理/ | 当前持仓决策、持股态度卡 |
| — | 00-系统配置/ | 本文件、卡片模板、L4权限说明 |

## 决策流程

```
1. 看 L1 市场环境
   → 退潮日？→ 不买，空仓等信号
   → 正常日？→ 继续到L2

2. 看 L2 方向概念
   → 找到最强主线方向
   → 检查 l4_permission: 🟢/🟡/🔴
   → 🔴 方向不参与
   → 🟡 只观察不建仓
   → 🟢 继续到L3

3. 看 L3 个股档案
   → 找到方向内核心标的
   → 检查个股 l4_permission
   → 评估风险提示
   → 确定买入标的

4. 看 L4 执行控制
   → 检查模式触发条件
   → 执行仓位规则
   → 设定止损线
   → 执行交易
```

## 反馈闭环

```
交易 → 结果 → 记入 05-错误库/ → 修正 04-交易模式/ → 优化 06-策略仓库/
```
