# 2026-06-22 历史复盘数据调用与后视镜隔离规则

更新时间：2026-06-22

## 目的

明确历史复盘是否允许大鸟主动调取外部与本地数据，以及调用时的边界。

结论：

1. 历史复盘可以交给 `大鸟 = Codex` 主责完成。
2. 大鸟允许调用正式公共数据源补证。
3. 但历史复盘必须受 `时间封口` 约束，严禁后视镜污染。

## 主责口径

历史复盘的正式主责仍为：

`Codex = 大鸟`

职责包括：

1. 读取交割单、口述、截图、旧复盘、旧计划。
2. 调用允许的数据源补齐当日市场事实。
3. 在不使用后验信息污染当日判断的前提下，完成结构化复盘。
4. 把结果写入 RAW / WIKI。

## 核心铁律

### 一、允许调数据，但不允许开后视镜

历史复盘的本质，不是事后解释，而是还原当时条件下最合理的判断与纠错。

因此：

1. 可以查数据。
2. 不能用未来数据反推当天判断。

### 二、历史复盘必须遵守时间封口

若复盘目标日为 `T`，则：

1. `T` 日复盘的 `当时事实层` 只能使用 `T` 日收盘前已存在，或 `T` 日之前已存在的资料。
2. `T+1` 及以后才出现的走势、新闻、龙虎榜归纳、市场结论，不得写入 `当时事实层` 和 `当时判断层`。
3. 后验内容只能写入 `事后验证层`。

## 历史复盘专用日期字段

从本规则起，历史复盘建议统一写以下字段：

1. `reviewTradeDate`
   说明：本次历史复盘针对的交易日。
2. `evidenceCutoffDate`
   说明：本次复盘允许使用的最晚证据日期，默认等于 `reviewTradeDate`。
3. `evidenceTradeDate`
   说明：某条证据本身所属的交易日。
4. `validationAsOfDate`
   说明：事后验证层引用的结果观察截止日。
5. `lookaheadForbidden`
   说明：固定写 `true`。

示例：

```yaml
reviewTradeDate: 2024-10-18
evidenceCutoffDate: 2024-10-18
lookaheadForbidden: true
validationAsOfDate: 2024-10-25
```

## 历史复盘输出结构

历史复盘正文必须固定拆成三层：

### 1. 当时事实层

只允许出现：

1. 当日及之前的市场数据
2. 当日及之前的消息
3. 当日交割单与持仓
4. 当日截图、分时、分钟 K、计划、聊天记录

### 2. 当时判断层

只允许写基于当时证据，理论上在 `T` 日收盘后能够得出的判断：

1. 市场阶段判断
2. 主线判断
3. 龙头判断
4. 买卖理由
5. 证伪条件
6. 盘中纠错动作

### 3. 事后验证层

只能单独放在最后，不得反向改写前两层。

允许写：

1. 后来结果如何
2. 当时哪些判断被验证
3. 当时哪些判断被打脸
4. 应进入哪个错误库、模式页、统计页

## 允许调用的数据源

历史复盘允许调用的正式公共源，沿用现行主链口径：

1. `AKShare`
2. `腾讯财经`
3. `iWenCai / pywencai`
4. `同花顺热榜 / 热点`
5. `Ashare`
6. `mootdx`
7. `iFinD`
8. `财联社实时流`

说明：

1. 大鸟可调用这些源补证历史复盘。
2. 并非要求每次全用，而是按职责最小够用原则调用。

## 数据源职责边界

### AKShare

用于：

1. 涨停池
2. 炸板池
3. 跌停池
4. 市场广度
5. 情绪结构

### 腾讯财经

用于：

1. 基础报价
2. 开盘价 / 收盘价
3. 分时补证
4. 单日分钟 K
5. 收盘快照

### iWenCai / pywencai

用于：

1. 历史条件筛股
2. 题材归因辅助
3. 历史强势股回查

### 同花顺热榜 / 热点

用于：

1. 人气热度确认
2. 热度排序
3. 题材热度侧证

### iFinD

只做：

1. 重点票新闻补证
2. 风险词补证
3. 低频增强

不得做：

1. 全市场高频主摄入
2. 代替主链事实层

### 财联社实时流

用于：

1. 快新闻触发时间补证
2. 盘中突发事件时间线补证

## 历史复盘调用规则

### 一、默认低频调用

历史复盘不是盘中实战，不需要高频轮询。

因此：

1. 默认只拉关键日、关键票、关键时段数据。
2. 不做无意义全市场反复扫描。

### 二、重点票优先

优先调用以下标的数据：

1. 你的成交票
2. 当天最强龙头
3. 当天最高标
4. 对你判断有直接影响的前排票
5. 关键断板票、卡位票、分歧转强票

### 三、关键证据必须落 RAW

只要历史复盘中使用了外部或脚本数据补证，必须把关键证据写入 RAW 或保留证据路径。

最少保留：

1. 数据源名称
2. 查询时间
3. `reviewTradeDate`
4. `evidenceTradeDate`
5. 关键截图或关键文本摘要
6. 是否存在数据缺口

## 缺口处理

任一数据层缺失时：

1. 必须明确写 `数据缺口`
2. 可以继续复盘
3. 但不得假装证据完整
4. 当日结论可信度应降级

## 禁止事项

1. 把后续连板高度写回当天判断。
2. 把后续龙头地位写回当天已知信息。
3. 把 `T+1` 新闻、榜单、归纳结论，伪装成 `T` 日收盘已知。
4. 因为后来涨得好，就倒推当天“必须买”。
5. 因为后来走坏，就倒推当天“明显不能买”。
6. 在没有证据时脑补当时买卖理由。

## 一句话

历史复盘允许大鸟调数据，但必须是：

`受时间封口约束的证据型复盘`

而不是：

`知道结果后的分析型复盘`
