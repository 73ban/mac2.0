# 作战室候选事后验证与进化规则

更新时间：2026-07-02 20:45

## 原则

凡是进入作战室候选票评分表的股票，都必须进入事后验证。没有验证，就不允许说系统在进步。

验证对象包括：

1. 主计划票。
2. 备选票。
3. 持仓处理票。
4. 情绪锚票。
5. 盘中重大消息雷达建议调入/调出的票。

5. 更新月度准确率趋势。

## 验证节点

| 节点 | 看什么 | 目的 |
|---|---|---|
| D+0 | 当天涨跌幅、是否涨停/20cm、是否进热榜Top30 | 验证作战室当日排序是否有效 |
| D+1 | 是否延续、是否骗炮、是否弱于板块 | 验证次日可交易性 |
| D+3 | 是否仍在主线内，是否分歧修复 | 验证持续性 |
| D+5 | 是否走出阶段行情还是一日游 | 验证模式有效性 |

## D+0 判定

| 结果 | 条件 | 处理 |
|---|---|---|
| 加分 | 涨停、20cm、北交所强涨，或热榜Top30强反馈 | 相关题材、来源、模式保留加权 |
| 保留观察 | 涨幅大于等于 5%，但未涨停 | 看 D+1/D+3 是否延续 |
| 降权观察 | 小涨但弱于高分预期 | 降低追高权限 |
| 扣分 | 当日负反馈 | 相关来源、模式、评分因子必须降权 |
| 待补数据 | 无行情证据 | 不允许判定有效 |

## 进化规则

1. 高分候选 D+0 扣分：次日同类票不得直接给主计划权限。
2. 高分候选 D+0/D+1 连续扣分：对应题材或模式降级。
3. 低排名候选反而涨停：说明排序权重错了，必须复查遗漏因子。
4. 持仓票强于预期：只证明持仓处理有效，不等于新开仓模式有效。
5. 消息雷达高分但候选负反馈：消息关键词降权。
6. 用户认为模型误判时，以用户判断为准，并写入权重修正。

## 月度统计口径

| 指标 | 含义 |
|---|---|
| 严格命中率 | 只统计涨停级反馈、强于预期、符合预期 |
| 可用命中率 | 严格命中 + 弱命中，代表逻辑没有错但强度不足 |
| 失败率 | 下跌、证伪、扣分、归档 |
| 平均跑赢上证 | 个股当日涨跌幅减去上证指数当日涨跌幅 |

月报只接受已经入账的事实，不靠主观补写。D+1/D+3/D+5 到期但缺行情时，必须留在待补队列，不允许假装验证完成。

## 对龙虾/老虎事实层的新增要求

Mac本机RAW入口和用户导入RAW只负责提供事实层数据，不负责判定模式升级。为了让 D+验证能自动比较“个股是否跑赢板块”，每个作战室候选和涨停全景记录必须带上：

1. 股票代码、名称、日期。
2. `交易题材`：交易题材或催化主题。
3. `主板块`：用于 D+对比的主板块，只能填一个。
4. `主板块Code`：板块代码，拿不到可空，但不能乱编。
5. `板块来源`：`tdx` / `ths` / `eastmoney` / `manual` / `unknown`。
6. `备选板块`：多板块候选列表，rank=1 必须等于 `主板块`。
7. `行业归属`：行业归属，拿不到可空。
8. `概念标签`：相关概念标签数组。
9. `板块归属证据`：为什么归到这个板块，写证据来源。
10. 涨停原因或候选理由。
11. 数据来源：通达信、同花顺、财联社、公众号、公告、互动易等。

每天还必须交付板块强度文件：

```text
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.json
raw/04-市场数据/板块强度/YYYY-MM-DD/通达信板块强度.md
```

板块强度文件至少包含：

1. 板块涨幅全量排名；接口受限时最低 Top100。
2. 涨停家数 Top20。
3. 今日主线候选板块 Top10。
4. 每个重点板块的领涨股、涨幅、涨停数、催化摘要。

详细执行口径见：

```text
raw/07-系统脚本/发给Mac本机RAW入口和用户导入RAW-板块字段与板块强度执行规则.md
```

未交板块字段或板块强度文件，视为事实层验收不合格。

当前自动大盘对比已经跑通；板块字段稳定交付后，Codex 接入自动板块对比：个股涨跌幅减去 `主板块` 当日涨跌幅。
