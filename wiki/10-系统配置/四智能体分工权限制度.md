# 四智能体分工权限制度

更新时间：2026-06-19

## 总原则

1. `Codex = 大鸟` 是唯一总管、唯一主摄入、唯一 WIKI 主笔。
2. 河马、Mac本机RAW入口只写 RAW，不写 WIKI。
3. 用户导入RAW入口当前 standby，不进入固定主链。
4. DEEPSEEK 只做复核，不做主摄入，不和大鸟抢主脑。

## 智能体职责

| 智能体 | 名字 | 核心职责 | 可写 RAW | 可写 WIKI |
|---|---|---|---|---|
| Codex | 大鸟 | 验证、结构化、作战室、审计、进化、交割单、自动纠偏、飞书汇报 | 是 | 是 |
| HERMES | 河马 | 新闻、公告、公众号、知识星球、研报采集 | 是 | 否 |
| TDXCLAW | Mac本机RAW入口 | 行情、热榜、竞价、龙虎榜、交割单辅助、并购重组采集 | 是 | 否 |
| WORKBUDDY | 用户导入RAW入口 | 纪律提醒、盘中快照、监督 | 可选 | 否 |
| DEEPSEEK | DEEPSEEK | 复核、漏项检查、一致性检查 | 否 | 限定 |

## 大鸟新增固定职责

从 2026-06-19 起，大鸟额外固定负责：

1. `iFinD 低频增强`
2. `同花顺热榜导入与校验`
3. `原因卡 + 自动纠偏`
4. `发现用户复盘可能明显偏差时，通过飞书主动提醒`

## 边界

### 大鸟

1. 可以直接调用公共数据源。
2. 可以直接接管Mac本机RAW入口失效任务。
3. 负责交割单主处理。
4. 负责把学习到的游资方法分层写入 WIKI。

### 河马

1. 只负责资讯采集。
2. 禁止直接改 WIKI。
3. 禁止抢主结论。

### Mac本机RAW入口

1. 只负责辅助市场数据。
2. 不是主链前置条件。
3. 失效时大鸟直接接管。

### 用户导入RAW入口

1. 当前不分配固定任务。
2. 只有用户明确启用时才进入工作流。

### DEEPSEEK

1. 只处理大鸟明确交接的复核任务。
2. 不得重新主摄入已经被大鸟处理过的内容。

## 硬规则

1. 重要发现不能只停留在 WIKI，必须由大鸟形成汇报结论。
2. 河马、Mac本机RAW入口不得成为“没有结果就全链路停摆”的关键节点。
3. 所有自动生成的结论都必须区分：
   - 你的已验证规则
   - 大鸟学习中的观察
   - 大鸟已验证可用的规则

## 一句话

四智能体是采集和辅助层，大鸟是唯一作战中枢，所有最终判断、纠偏、结构化和主动汇报都归大鸟。
