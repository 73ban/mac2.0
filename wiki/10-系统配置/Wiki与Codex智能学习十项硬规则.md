# Wiki与Codex智能学习十项硬规则

更新时间：2026-07-06

## 定位

这页是 73神话 当前最高优先级记忆。用户要求先把这 10 个问题做完，再考虑其他任务。

两个总目标不变：

1. 20万小资金快速做大。
2. 把 Codex 训练成能持续自进化的超短交易助手。

## 十项硬规则

### 1. RAW 到 Wiki 必须主动处理

RAW 不是存档终点。每天必须执行：

```text
RAW -> 重要性分层 -> 个股/题材/模式/作战室/D+验证 -> Wiki沉淀
```

只把影响炒作逻辑、公司价值、风险证伪、模式胜率的信息写入正式 Wiki。普通噪音保留在 RAW，不污染个股卡和题材卡。

### 2. 历史交易必须补逐笔模式归因

所有历史复盘和交割单里的买入，最终都要补：

```text
主模式、辅助模式、模式来源、置信度、推定依据、行情环境、D+结果
```

优先顺序：

```text
最近30个交易日 -> 大亏日 -> 大赚日 -> 作战室重点票 -> 其他历史交易
```

### 3. D+验证必须反向改权重

所有作战室候选、消息判断、交易模式、高手学习样本，都必须进入 D+1/D+3/D+5 验证。  
验证失败必须降权、进入错误库或降级观察；验证成功也只能逐步升权，不能直接越权重仓。

### 4. 个股卡和题材卡必须每日刷新

持仓票、作战室 Top5、热榜前排、旧题材二次发酵票，必须每日刷新：

```text
最新催化、官方确认/辟谣、市场关注点、机构/价值逻辑、短线风险、历史炒作路径
```

如果没有正式卡片，先新建最低可用卡片，再后续补全。

### 5. 信息源必须分工

信息源不能混用：

| 来源 | 角色 | 使用边界 |
|---|---|---|
| 公告/互动易/上证e互动 | 官方确认或证伪 | 权重最高，但只写与当下炒作逻辑相关的信息 |
| 韭研公社 | 研报新闻、产业逻辑、题材解释 | 不归入短线知识；用于公司影响和题材逻辑 |
| 淘股吧 | 情绪、模式语言、资金关注、高手样本 | 不直接当买点，必须D+验证 |
| 同花顺/通达信/淘股吧三榜 | 热度、拥挤度、资金关注验证 | 不直接等于买入信号 |
| 公众号/游资观点 | 学习样本和情绪参考 | 先入RAW和验证队列，不直接升级规则 |

### 6. 作战室 Top5 必须像真实交易计划

每只 Top5 票必须写清：

```text
为什么入选、属于什么模式、怎么买、什么条件不能买、持仓怎么处理、如何卖、如何D+验证
```

Top5 变化必须写清调入/调出原因，不能只写“分数变化”。

### 7. 情绪周期必须多源合成

每日情绪周期不能只看一类文字。必须综合：

```text
连板高度、涨跌停、炸板率、核按钮/天地板、三榜迁移、淘股吧话题、实盘赛热门买入、持仓反馈
```

输出阶段：

```text
启动、发酵、高潮、分歧、修复、退潮
```

### 8. 飞书/通知必须降噪

所有校准通知必须带：

```text
判断对象、当前结论、我的判断逻辑、可能有用的点、风险/不确定点、需要用户纠偏什么
```

每天保留必要通知：

```text
消息Top10、学习Top10、22:30作战室、硬故障告警
```

不合格通知必须进隔离区，不发送。

### 9. 每条消息必须有公司价值判断

所有重要消息都要判断对公司意味着什么：

```text
短炒型、业绩兑现型、估值重估型、长期风险型、风险证伪型
```

并写清：

```text
证据强弱、兑现周期、收入/利润/估值映射、机构视角、短线视角、风险点
```

### 10. 必须建立学习成绩单

每天记录：

```text
我学了什么、判断了什么、哪些验证正确、哪些验证错误、哪些来源升权/降权、哪些模式升权/降权
```

半年后必须能回答：

```text
哪些模式赚钱、哪些模式亏钱、哪些信息源有效、哪些高手失效、Codex哪里进步、哪里仍需用户校准
```

## 每日固定执行顺序

```text
1. 数据源健康检查
2. RAW重要性分层
3. 个股卡/题材卡刷新
4. 互动易/公告官方证据入卡
5. 淘股吧/韭研/三榜更新
6. 作战室Top5刷新
7. D+验证清债
8. 交易模式逐笔归因和统计
9. 错误库候选与模式降级/升级
10. 学习成绩单和通知清晰度检查
```


## 禁止

1. 不把 RAW 数量当成系统进化。
2. 不把热榜排名当成买点。
3. 不把游资/名人堂观点直接升级成交易规则。
4. 不在 D+验证缺失时宣称模式有效。
5. 不覆盖历史记录，只追加修正和验证。
