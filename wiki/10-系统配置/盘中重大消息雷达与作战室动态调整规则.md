# 盘中重大消息雷达与作战室动态调整规则

更新时间：2026-07-06 20:10

## 目标

把财联社、公众号、同花顺热榜、淘股吧热榜、通达信热榜、成交额、异动、持仓票、作战室候选统一进一层“盘中重大消息雷达”，解决三个问题：

1. 新催化、新风险出现时，及时提示是否需要调整作战室。
2. 每轮输出最重要消息 Top10，避免信息堆积但没有排序。
3. 高优先级消息自动发飞书让用户校准权重，用户判断优先于模型判断。

## 1 分钟盘中检查口径

从 2026-07-06 起，盘中判断统一改为：

| 对象 | 检查频率 | 说明 |
|---|---:|---|
| 持仓票 | 1 分钟 | 先看风险，再看机会；持仓风险 S 级优先推送 |
| 作战室 Top20 / Top5 | 1 分钟 | 调入、调出、排名变化必须写原因，进入 D+验证 |
| 热榜 / 成交额 / 异动 | 1 分钟 | 读取本机 RAW 和已同步快照；源数据没刷新时记录“无新增” |
| 财联社 / 公众号 / 韭研 / 淘股吧 / 互动易 / 公告 | 1 分钟增量检查 | 只把高价值消息升级提醒，普通消息只入 RAW |

注意：这里说的是 Codex 盘中检查和判断频率，不等于每个外部网站都强行 1 分钟重抓。接口限频或重型网页仍按采集边界执行，但 Codex 每分钟检查有没有新 RAW、新快照、新变化。

禁用边界：

- 不调用付费 `tdx_quotes`。
- 不调用付费 `tdx_kline`。
- 需要量价事实时，只读取龙虾/本机已经落盘的 RAW、东方财富/腾讯等免费快照、收盘基础数据和成交额榜。

## 打分口径

雷达不是简单看新闻标题，而是综合：

1. 消息来源：财联社、公众号、市场数据。
2. 发布时间：默认看最近 18 小时，按发布时间而不是文件修改时间。
3. 催化词：政策、并购重组、涨价、订单、AI、算力、PCB、半导体、机器人等。
4. 风险词：外围大跌、A50、汇率、监管、问询、减持、亏损、澄清、退潮、补跌等。
5. 个股映射：是否命中同花顺 Top100、涨停/20cm、作战室候选。
6. 去重：同步冲突、`.stversions`、同标题热榜重复路径不重复计分。

## 动作分层

| 动作 | 含义 | 处理 |
|---|---|---|
| 防守/推倒重做复核 | 明确外围或市场风险触发 | 先降仓位权限，再重看主计划 |
| 风险降级复核 | 个股或题材风险显著高于催化 | 不得直接升级候选 |
| 作战室调入/调出复核 | 命中已有作战室票或强相关票 | 只调整权重，不直接买入 |
| 候选票加权复核 | 消息 + 热榜/个股共振 | 加入候选池，等连板/六维/竞价验证 |
| 题材权重上调观察 | 方向有催化但个股不够清晰 | 观察题材，不直接给票 |
| 记录观察 | 低信号 | 只留痕 |

## 飞书校准

默认只把高优先级消息发飞书，不把 Top10 全部刷屏。扫描可以 1 分钟一次，但飞书只推 S/A 级：

| 等级 | 推送规则 |
|---|---|
| S | 持仓重大风险、核心票澄清/监管/炸板、市场系统性退潮，立即推 |
| A | Top5调入调出、重要催化强共振、持仓处理方案变化，去重后推 |
| B | 普通排名变化、普通学习样本、单源热度，写 RAW 不打扰 |

用户回复格式：

```text
第1条 有效/一般/无效/反向，原因：...
```

后续处理原则：

1. 用户认为有效：相同关键词、来源、个股映射后续升权。
2. 用户认为无效：相同关键词、来源、泛化长文后续降权。
3. 用户认为反向：写入风险词或降级规则。
4. 用户判断与模型冲突时，以用户判断为准。
