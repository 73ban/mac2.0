# GPT Pro 持续工作规则

更新时间：2026-07-06

## 定位

GPT Pro 的 token 优先用于让 73神话持续进化，不用于无边界读资料。

## 优先级

1. 交易安全：持仓、仓位、回撤、禁止动作。
2. 明日胜率：作战室、竞价、主线、候选票。
3. D+验证：把预测变成反馈。
4. 错误修正：把亏损和失控动作写成限制。
5. 模式升级：只升级被真实市场验证过的模式。
6. 高手学习：淘股吧千万组、名人堂、大游资只作为样本和规则候选。
7. 资料整理：只保留能服务交易和训练的资料。

## 自动推进口径

当用户说“继续”“你自己推进”“全部执行到位”时，Codex 默认按以下顺序执行：

```text
每日总控驾驶舱
-> D+债务清零
-> 当前作战室新鲜度
-> 持仓证据检查
-> 三榜/淘股吧/公告/消息验证
-> 模式权限修正
-> 错误库和统计回写
-> Wiki资料瘦身
```

## 当前最高优先级记忆

用户已明确要求：先把 Wiki 与 Codex 智能学习能力的 10 个问题做完，再考虑其他任务。

必读规则：

```text
wiki/10-系统配置/Wiki与Codex智能学习十项硬规则.md
```

执行顺序固定为：

```text
数据源健康检查
-> RAW重要性分层
-> 个股卡/题材卡刷新
-> 互动易/公告官方证据入卡
-> 淘股吧/韭研/三榜更新
-> 作战室Top5刷新
-> D+验证清债
-> 交易模式逐笔归因和统计
-> 错误库候选与模式升降级
-> 学习成绩单和通知清晰度检查
```

## 盘中一分钟工作记忆

从 2026-07-06 起，盘中自动看盘统一按 1 分钟检查：

```text
持仓票
-> 作战室Top20/Top5
-> 三榜热度/同花顺/淘股吧/通达信热榜
-> 成交额Top100
-> 盘中异动
-> 财联社/韭研/公众号/淘股吧/互动易/公告
-> S/A级提醒去重推送
-> B级记录入RAW和D+验证队列
```

执行入口：

```text
raw/07-系统脚本/start_dynamic_warroom_top5.sh
raw/07-系统脚本/codex_intraday_minute_watch.py
raw/07-系统脚本/codex_intraday_alert_validation.py
raw/07-系统脚本/codex_feishu_task_inbox.py
wiki/10-系统配置/盘中一分钟自动看盘与提醒规则.md
```

边界：

- 不调用付费 `tdx_quotes` / `tdx_kline`。
- 只读取本机 RAW、已同步快照、免费行情/热榜产物和龙虾落盘结果。
- 扫描频率是 1 分钟，但飞书只推真正需要用户决策的 S/A 级变化。
- 飞书下任务必须带 `【Codex任务】` 或 `#Codex任务`，进入任务收件箱后再执行和回写。

## 禁止浪费 token

- 不为“看起来有用”的资料长篇总结，除非能写出验证点。
- 不把名人堂观点直接写成交易规则。
- 不在 D+验证未完成时宣称系统进化。
- 不覆盖用户交易事实。
- 不用历史后视镜倒填盘前计划。
