## 还没有做完的事项

| 优先级 | 缺口 | 为什么重要 | 当前改进动作 |
|---|---|---|---|
| P0 | 历史复盘/交割单很多买入没有模式归因 | 没有模式标签就不能统计胜率 | 新增交易模式归因缺口审计 |
| P0 | 模式胜率还没有按行情拆分 | 同一模式在主升和退潮成功率完全不同 | 模式模板已加行情分层胜率 |
| P0 | 模式页质量不统一 | 有些页缺失效条件、修正记录、统计字段 | 新增模式页质量检查 |
| P0 | 用户未口述模式时，系统还没有逐笔自动补标签 | 复盘不能依赖用户每次说完整 | 先文件级审计，后续升级逐笔归因 |
| P1 | 120 个候选模式未全部结构化进 JSON | 候选池不能直接用于脚本识别 | 后续按高频和交易相关度分批纳入 |
| P1 | 淘股吧新叫法还不能自动发现并入词典 | 市场叫法会变，旧词典会过期 | 后续做“新词候选发现” |
| P1 | 模式页案例和反例不足 | 没有案例就难以判断有效/失效 | 每日复盘和D+验证后追加 |
| P1 | 模式修正日志还没覆盖所有旧模式页 | 后期变化无法追溯 | 模式页 lint 会持续暴露缺口 |
| P1 | 作战室 Top5 尚未强制输出交易模式标签 | 选股逻辑和模式统计还没完全绑定 | 后续接入动态作战室脚本 |
| P1 | 飞书校准还没按模式维度稳定反馈 | 你无法每天校准系统学得对不对 | 后续把缺口Top10和新模式Top10发飞书 |
| P2 | 模式与个股卡、题材卡的反向链接不足 | 事后复盘难以从股票追到模式 | 后续给个股卡追加“涉及模式” |
| P2 | 没有模式别名冲突处理 | 同一个词可能在不同语境含义不同 | 后续加别名冲突表 |
| P2 | 没有模式退役机制 | 市场玩法失效后仍可能误用 | 后续加“降级/退役/观察”状态 |

## 下一步执行顺序

1. 先跑模式页质量检查，补齐 P0 缺字段。
2. 再跑交易模式归因缺口审计，找出最近复盘中没有模式标签的文件。
3. 优先补最近 30 个交易日、大亏日、大赚日、作战室重点票。
4. 把每笔买入统一标注：主模式、辅助模式、模式来源、置信度、推定依据。
5. D+1/D+3/D+5 后按模式和行情环境统计。
6. 每周把模式胜率、失效模式、新增玩法写入模式页修正记录。
