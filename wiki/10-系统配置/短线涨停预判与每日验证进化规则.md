# 短线涨停预判与每日验证进化规则

更新时间：2026-07-01

## 定位

本规则服务两个目标：

1. 20 万小资金快速做大。
2. Codex 通过 Top100、连板天梯、涨停原因、财联社、公众号、交割单和每日复盘，持续训练成超短交易高手。

这里不是资料归档规则，而是每日预判、次日验证、权重修正的闭环规则。

## 必看输入

每天盘后到次日盘前，Codex 必须读取以下数据：

| 数据 | 要求 | 用途 |
|---|---|---|
| 通达信热榜 Top100 | 必须全量读，不得只看前 20 | 判断散户注意力、人气扩散、预期差和一致性 |
| 每日涨停全景总表 | 必须每日读取，首板和 2板及以上放在同一个文件里 | 一个入口看全市场涨停票，比较首板催化、连板高度、20cm/30cm弹性和次日补涨机会 |
| 剔除 ST 后的 2板及以上连板天梯 | 必须完整写入复盘正文并读取，含主板10cm、创业板20cm、科创板20cm、北交所30cm；首板不得混入 | 判断情绪高度、梯队完整度、20cm/30cm弹性、亏钱效应和明日接力风险 |
| 首板涨停催化日报 | 必须单独写 RAW/JSON 并读取 | 识别新题材、新消息催化、最大预期差和次日补涨方向 |
| 通达信涨停原因 6 维度 | 每只 2板及以上连板票必须进复盘正文，首板在催化日报里写清原因和验证点 | 拆解题材、公告、产业、事件、资金、技术等催化 |
| 财联社 | 必须按个股/题材映射读取 | 识别能引发隔日交易冲动的硬催化 |
| 游资公众号和短线知识 | 必须读取高信号内容 | 识别情绪周期、模式语言、市场共识和风险提示 |
| 交割单和用户口述 | 必须优先读取 | 用于判断用户是否站在正确方向和正确买卖点 |
| 当日作战室 | 必须读取 | 用于验证盘前预判是否被市场兑现或证伪 |

Top100 是散户必看信息，也是短线注意力入口。Codex 不得再把 Top100 当作可压缩的边缘数据。

## 每日复盘正文要求

复盘正文不追求短。短线选手每天必看的结构，必须保留在人类可读正文里。

必须进入复盘正文：

1. 通达信热榜 Top100 全量表，或 Top100 附录。
2. 剔除 ST 后的 2板及以上连板天梯全量表，必须含主板10cm、创业板20cm、科创板20cm、北交所30cm；若北交所为 0，必须明写。
3. 首板涨停催化日报路径和摘要；首板不进入连板天梯。
4. 每只 2板及以上连板票的涨停原因 6 维度。
5. 高标、中位、低位首板催化池的分层判断。
6. 今日强主线、断板风险、亏钱效应和明日情绪锚。

可以压缩的是解释性废话，不能压缩核心市场结构。

## 次日涨停候选打分

Codex 每天盘后必须生成一张候选评分表，输出到：

```text
wiki/09-统计与进化/YYYY-MM-DD-次日涨停候选评分表.md
```

评分总分 100 分，扣分项可让总分低于 0。

| 维度 | 分值 | 说明 |
|---|---:|---|
| 题材与催化强度 | 0-20 | 财联社、公告、政策、产业事件、公众号高信号内容是否形成明确催化 |
| 板块共振与梯队 | 0-20 | 是否有涨停宽度、连板高度、中军容量、首板扩散 |
| 个股地位 | 0-15 | 是否龙头、空间板、换手核心、容量核心、补涨核心、辨识度核心 |
| 热榜注意力 | 0-10 | Top100 排名、排名变化、是否从低位进入大众视野 |
| 盘口与涨停质量 | 0-15 | 板型、封单、开板次数、成交额、换手、炸板修复 |
| 用户适配度 | 0-10 | 是否符合当前资金体量、模式权限、持仓处理优先级 |
| 次日触发清晰度 | 0-10 | 是否能写出竞价确认、买点、止损和放弃条件 |
| 风险扣分 | 0 到 -30 | ST、监管、减持、业绩雷、过度一致、连续加速、退潮期高位接力等 |

候选表必须包含：

```text
代码
名称
所属题材
候选类型
总分
核心加分证据
主要风险
次日竞价确认条件
可执行权限：主计划/备选/观察/禁止
D+1 验证点
```

## 每日 15 票预期差与重大催化训练

Codex 每天除作战室 3-8 只可执行候选外，必须额外输出 15 个训练样本，用于校准市场总结能力和消息权重判断能力。15 票不是买入清单，而是训练“哪些消息会被资金选择、哪些消息只是文字热闹”。

输出位置：

```text
raw/03-每日计划/YYYY-MM-DD-市场预期差与重大催化15票.md
```

15 票必须覆盖两类：

1. 市场最大预期差个股：文本消息不显眼、市场原本低预期，但被涨停、一字、连板、热榜跃迁、板块扩散验证。
2. 重大新闻催化个股：政策、产业、订单、并购、监管、技术突破、海外映射等具备明确增量，且能映射到具体标的和板块。

每只票必须写：

```text
股票：
题材/消息：
候选类型：最大预期差/重大新闻催化/两者兼具
消息级别：重大/一般/弱
映射纯度：高/中/低
资金验证：一字板/大单一致/涨停/连板/炸板回封/板块扩散/热榜跃迁/暂无
基本面纯度：
短线情绪权重：
预期差来源：
风险点：
次日验证点：
综合判断：
```

筛选优先级：

1. 先看当天一字板、大封单、连板晋级、强封板票，倒查背后逻辑。
2. 再看重大新闻映射到的核心票、补涨票、20cm 弹性票、中军容量票。
3. 再看热榜 Top100 中排名突然跃迁、但市场讨论尚未充分展开的票。
4. 最后看文本催化强但盘面暂未确认的备选样本。

若消息文本判断和市场表现冲突，以市场表现优先，必须写明“文本低估/盘面上修/等待验证”。

## 消息能否刺激涨停的判断

财联社、公众号和其他新闻不能只做摘要，必须回答：

```text
这条消息是否可能刺激股价涨停？
刺激的是哪个题材？
更可能刺激龙头、中军、补涨、20cm 弹性票，还是只刺激情绪？
有没有盘面确认？
如果没有盘面确认，为什么暂不进入作战室？
```

消息评分只代表候选优先级，不代表可以买。进入主作战室必须同时满足 L1 市场环境、L2 主线、L3 个股地位、L4 模式权限和竞价确认。

## 市场验证优先级

涨停、一字板、大单一致、连续封板，是消息权重的市场验证。Codex 对消息强弱的文本判断不得压过资金投票。

资金验证权重按以下顺序上修：

| 信号 | 解释 | 处理 |
|---|---|---|
| 一字板 | 市场一致预期最强，说明背后逻辑已形成共识 | 消息权重至少上调到高，进入 15 票训练样本，倒查逻辑来源 |
| 大单封死 | 资金态度明确，卖压被压制 | 上调短线情绪权重，记录封单和题材映射 |
| 连续涨停/连板晋级 | 逻辑持续被验证，或情绪持续发酵 | 上调题材级别和个股地位，区分龙头、补涨、容量核心 |
| 涨停后板块扩散 | 个股逻辑外溢为题材逻辑 | 上调板块共振分，寻找预期差补涨 |
| 炸板回封 | 分歧后仍被资金承接 | 保留高关注，但风险扣分 |
| 涨停但无扩散 | 可能是独立事件或筹码博弈 | 只上调个股情绪权重，不直接上调题材级别 |

反误判约束：

```text
一字板/大单一致 = 强资金共识验证，不等于无条件交易机会。
```

以下情况只能标记为“强一致待拆解”，不得直接推导为可买强逻辑：

1. 复牌、重组、重大资产注入、停牌补涨导致的连续一字。
2. 极小流通盘、封单好看但成交极低，无法验证真实承接。
3. 无板块扩散、无同题材跟随，只是独立事件或筹码锁定。
4. 高位连续加速后一字，次日开板可能成为情绪退潮点。
5. 封单快速衰减、反复撤单、开板后不能回封。

处理要求：

```text
先承认资金验证强，再拆分：
逻辑强度 / 筹码一致 / 买点可执行性 / 次日分歧风险
```

硬约束：

1. 一字板和大单一致必须倒查涨停原因，不能只按行业标签解释。
2. 基本面纯度低，不等于短线情绪权重低；必须拆成两套分数。
3. 低纯度补涨票若走出 3 板以上或一字加速，必须标记为情绪补涨核心。
4. 高位连续跌停只代表当下退潮，不得反向否定前期资金验证；应改写为“前期逻辑强、当前退潮、等待修复验证”。
5. 未被预测到的一字板、连板、强封板，必须进入漏判样本，回写低估原因。

## 每日验证闭环

次日收盘后，Codex 必须用本机统计的连板天梯、实际涨停池、Top100 变化验证前一日预判。

必须回写：

```text
wiki/09-统计与进化/YYYY-MM-DD-涨停预判验证与权重修正.md
```

验证字段：

```yaml
prediction_date:
validation_date:
candidate_count:
hit_limit_up_count:
hit_top3:
hit_top5:
hit_top10:
broken_board_count:
large_loss_count:
best_hit:
worst_false_positive:
missed_limit_up:
weight_adjustment:
next_rule_patch:
```

验证口径：

1. 预测票次日涨停，记为命中。
2. 预测票次日冲板炸板，记为半命中但必须分析炸板原因。
3. 预测票大跌或低开低走，记为失败并扣权重。
4. 未预测到但次日涨停的强票，必须进入漏判样本。
5. 连续 3 次同类失败，必须写入降级规则。
6. 连续 5 次同类成功，才允许小幅提高该类信号权重。

## 进化原则

Codex 的任务不是把所有信息写长，而是把每日市场结构变成可比较、可验证、可修正的数据。

每天必须做到：

1. 看全 Top100、首板涨停催化日报、2板及以上连板天梯、涨停原因、财联社、公众号。
2. 打分并排序次日涨停候选。
3. 给出可执行和不可执行边界。
4. 用次日真实涨停和连板晋级结果验证。
5. 把成功经验和失败样本写回统计与规则。

一句话：

```text
资料必须读全，正文保留必看结构，作战室只给可执行计划，统计层负责验证进化。
```
