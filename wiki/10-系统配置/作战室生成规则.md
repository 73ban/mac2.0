# 作战室生成规则
更新时间：2026-07-06 20:10

## 目标

作战室只服务今天和下一个交易日执行，不做资料堆砌，不做无出口长文。

## 核心原则

1. 先判市场状态，再判题材，再判个股。
2. 持仓处理优先级高于新开仓计划。
3. 主计划最多 1 个，重点票最多 1 到 3 只。
4. 没有触发条件、降级条件、退出条件、竞价确认条件的计划，不得进入主作战室。
5. 数据有缺口时允许继续输出，但必须显式标注 `数据缺口`。
6. Top100、首板涨停催化日报、剔除 ST 后的 2板及以上连板天梯、涨停原因 6 维度、财联社和公众号必须先进入评分层；天梯必须含主板10cm、创业板20cm、科创板20cm、北交所30cm，首板不得混入天梯，作战室只接收经过评分和风控过滤后的可执行候选。

1. `AKShare + 涨停池骨架`

7. `通达信涨停原因 6 维度`

## 自动骨架

现在作战室自动生成 3 类页面：

1. `wiki/07-作战室/当前作战室工作页.md`
2. `wiki/07-作战室/YYYY-MM-DD-作战室候选票评分表.md`
3. `raw/03-每日计划/YYYY-MM-DD-竞价监控清单.md`

自动骨架只负责起草，不替代你的拍板。

## 次日涨停候选

作战室生成前，必须先完成或读取：

```text
wiki/09-统计与进化/YYYY-MM-DD-次日涨停候选评分表.md
```

候选来源必须覆盖：

```text
通达信热榜 Top100
首板涨停催化日报
剔除 ST 后的 2板及以上连板天梯
通达信涨停原因 6 维度
财联社
游资公众号
当日主线与板块强度
```

作战室不能把所有高分票都列为买入计划。高分票先进入候选池，只有满足市场环境、主线、个股地位、用户模式权限和竞价确认条件时，才允许升级为主计划或备选。

## 盘中重大消息雷达

盘中自动链路必须至少每 1 分钟检查并在有变化时生成：

```text
wiki/07-作战室/YYYY-MM-DD-盘中重大消息雷达Top10.md
.llm-wiki/catalyst-radar/latest-catalyst-radar.json
raw/11-Codex分析产物/盘中一分钟看盘/YYYY-MM-DD/intraday-minute-watch.json
wiki/07-作战室/当前盘中一分钟看盘.md
```

雷达命中 `防守/推倒重做复核` 时，作战室必须先重判市场状态和仓位权限；命中 `作战室调入/调出复核` 或 `候选票加权复核` 时，只允许改变候选权重，不允许绕过连板、Top100、涨停原因六维和竞价确认直接给买入权限。

高优先级消息会生成飞书待判断通知，用户回复的 `有效/一般/无效/反向` 必须优先写入消息权重人工校准表。

## 事后验证

作战室生成后，同日必须运行：

```text
raw/07-系统脚本/codex_warroom_validation_pipeline.py
```

所有进入 `YYYY-MM-DD-作战室候选票评分表.md` 的股票必须写入：

```text
data/facts/warroom_candidate_predictions.jsonl
data/facts/warroom_candidate_validation_results.jsonl
wiki/09-统计与进化/YYYY-MM-DD-作战室候选验证回看.md
wiki/09-统计与进化/作战室候选票D+验证队列.md
```

没有 D+0/D+1/D+3/D+5 验证的作战室票，不能作为后续模式升级证据。高分候选当日负反馈，必须反向修正评分权重。

## 闭环自进化硬规则

作战室不是一次性选票页，必须形成闭环：

```text
盘前/盘后预判
→ 作战室候选评分
→ 盘中重大消息雷达调权
→ D+0 收盘验证
→ D+1/D+3/D+5 继续验证
→ 权重修正
→ 模式升级/降级/禁用
→ 次日作战室重新生成
```

每只进入作战室评分表的票，都必须有唯一验证链路：

```text
data/facts/warroom_candidate_predictions.jsonl
data/facts/warroom_candidate_validation_results.jsonl
wiki/09-统计与进化/YYYY-MM-DD-作战室候选验证回看.md
wiki/09-统计与进化/作战室候选票D+验证队列.md
```

闭环判断口径：

| 结果 | 处理 |
|---|---|
| 命中涨停/强于板块/热榜前排延续 | 保留或小幅升权，但仍需 D+1 延续确认 |
| 高分但 D+0 弱 | 降低该来源、题材或模式的追高权限 |
| D+1/D+3 连续证伪 | 候选来源降权，相关模式进入复盘 |
| 退潮日高分票批量失败 | 市场环境权重优先级上调，个股催化权重下调 |
| 用户人工纠正 | 写入消息权重人工校准表，优先级高于模型判断 |

禁止事项：

1. 只给作战室候选，不做验证。
2. 只记录命中，不记录失败。
3. 高分失败后第二天仍按原权重推荐。
4. 没有 D+ 结果就把模式升级为正式规则。
5. 退潮日仍用进攻日评分权重。

盘中一分钟链路必须把作战室验证状态写进健康报告；如果验证队列不增长、D+结果不回填，视为作战室闭环断裂。

## 必须输出

```text
市场预期
市场状态 / 仓位权限
今日优先模式
今日禁止模式
持仓处理
明日主线判断
候选方向 / 重点个股
触发条件
降级条件
退出条件
竞价确认条件
数据缺口
自动纠偏候选
```

## 竞价口径

1. 竞价只用于确认、降级、暂停，不单独决定重仓。
2. 本机快照可用时，可提供 `09:15 / 09:20 / 09:25` 快照。
3. 本机快照缺失或失效时，大鸟直接接管，不等待。
4. 没有竞价辅助数据时，仍可继续判断，但必须写明 `竞价数据缺失`。

## 自动纠偏层

作战室必须同步看：

1. `latest-warroom-watchlist.json`
2. `latest-stock-reason-cards.json`
3. `latest-ifind-enhance.json`
4. `latest-correction-alerts.json`

如果自动纠偏命中：

1. 异常波动
2. 问询
3. 澄清
4. 监管
5. 减持
6. 亏损
7. 其他高风险词

则对应个股必须标注 `纠偏待复核`，不能当作无风险强票直接推进。

## 一句话

作战室不是资料陈列页，而是大鸟基于公共数据、RAW、纠偏层、学习层复核后压缩出来的最小可执行计划。
