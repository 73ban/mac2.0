# WIKI 准入规则

更新时间：2026-06-28

## 目的

本规则用于防止 73神话再次变成资料仓库。

核心原则：

```text
RAW 收集事实
WIKI 保存可执行、可验证、可复盘、可进化的交易系统成果
```

任何资料进入 WIKI 主目录前，必须通过准入判断。

## 准入判断

新内容进入 WIKI 前，必须至少满足以下一类用途：

1. 影响作战室：能改变今日/明日候选、主线、风险或仓位。
2. 影响持仓：能改变当前持仓态度、卖点、止损、加减仓。
3. 影响模式：能验证、修正、升级或淘汰一个交易模式。
4. 影响错误库：能解释一次亏损、违规、重复错误或高成本行为。
5. 影响统计进化：能进入 D+验证、月度复盘、自学习闭环。
6. 影响总规则：能形成明确制度、纪律、禁止动作或执行标准。

如果只是资料原文、新闻摘要、公众号观点、研报摘录、未验证概念，默认不进 WIKI 主层，先进入 RAW 或资料索引。

## 必填字段

进入 WIKI 的资料型页面，必须尽量补齐以下字段。

```yaml
truth_grade: S1 | S2 | S3 | S4
use_grade: A | B | C | D
trade_impact: 作战室 | 持仓 | 模式 | 风险 | 规则 | 统计 | 无
evidence_path:
validation_status: 待验证 | 已验证 | 证伪 | 降级 | 归档
```

字段说明：

| 字段 | 含义 |
|---|---|
| `truth_grade` | 判断信息真不真 |
| `use_grade` | 判断信息怎么用 |
| `trade_impact` | 判断是否影响交易系统 |
| `evidence_path` | 指向 RAW、公告、交割单、截图、数据源 |
| `validation_status` | 标记后续验证状态 |

## truth_grade

| 等级 | 含义 | 例子 |
|---|---|---|
| S1 | 官方/硬事实 | 交易所公告、公司公告、交割单、原始成交、正式财报 |
| S2 | 可复核高可信资料 | 券商正式研报、权威媒体、可核验数据源、完整公开实盘记录 |
| S3 | 有参考价值但需验证 | 公众号、知识星球、淘股吧高手观点、调研纪要、产业链传闻 |
| S4 | 低可信/噪音 | 小作文、无来源截图、社群传言、无法核验观点 |

## use_grade

| 等级 | 用法 |
|---|---|
| A | 可进入作战室或持仓决策，但仍需结合盘面 |
| B | 可作为题材/个股/模式验证材料 |
| C | 只作为背景资料或观察样本 |
| D | 不进入 WIKI 主层，只留 RAW 或归档 |

## 目录准入

| 目标目录 | 准入标准 |
|---|---|
| `wiki/00-总纲` | 只收最高规则、总制度、明确禁止动作、交易系统总入口 |
| `wiki/01-L1市场环境` | 必须能判断市场阶段、情绪、风险、仓位环境 |
| `wiki/02-L2方向题材` | 必须能解释主线、催化、题材生命周期或证伪逻辑 |
| `wiki/03-L3个股档案` | 必须能明确个股角色、核心性、证据链、风险点 |
| `wiki/04-L4交易模式与执行` | 必须能形成买点、卖点、仓位、执行或退出规则 |
| `wiki/05-错误库` | 必须对应真实错误、亏损归因、纪律破坏或重复问题 |
| `wiki/06-持仓与资金管理` | 必须对应真实持仓、资金、交割单加工、仓位决策 |
| `wiki/07-作战室` | 必须服务于今日/明日交易计划或候选池 |
| `wiki/08-信息来源` | 只保留来源规则、来源分级、证据链和索引，不堆资料正文 |
| `wiki/09-统计与进化` | 必须进入统计、D+验证、模式有效性或自学习闭环 |
| `wiki/10-系统配置` | 只放制度、模板、提示词、脚本说明、运行规则 |

## 资料卡处理

资料卡不能默认进入 WIKI 主层。

处理顺序：

1. 原文进 RAW。
2. 提取来源、日期、主题、可信度。
3. 判断是否影响 L1/L2/L3/L4。
4. 能影响交易系统的，进入对应 WIKI 目录。
5. 暂时不能验证的，进入 `wiki/08-信息来源` 索引或 `data/brain` 候选。
6. 后续通过 D+验证或实盘反馈，再升级为正式模式、规则、错误或个股档案。

## 增量卡处理

以下目录属于重点瘦身对象：

```text
wiki/08-信息来源/RAW独立知识卡
wiki/08-信息来源/公众号-资料卡
wiki/08-信息来源/知识星球-每日上扬研报-资料卡
wiki/03-L3个股档案/RAW增量个股卡
wiki/02-L2方向题材/RAW增量题材卡
wiki/04-L4交易模式与执行/RAW增量模式卡
wiki/04-L4交易模式与执行/游资认知体系
```

这些目录暂不批量移动。后续处理必须先做审计：

- 是否有主档案可合并。
- 是否只是 RAW 摘要。
- 是否有 D+验证价值。
- 是否有重复页面。
- 是否有孤立页面或坏 wikilink。

## 禁止进入 WIKI 主层的内容

- 未核验小作文。
- 重复新闻摘要。
- 只有标题没有交易含义的研报。
- 不能影响主线、个股、模式、仓位、风险的资料卡。
- 未经验证的公众号观点。
- 自动摄入失败页、空 slug 页、模板回流页。
- 单纯缓存、脚本输出、中间 JSON 摘要。

## 升级路径

内容从 RAW 到 WIKI 的升级路径：

```text
RAW 原文
  → 资料索引/证据链
  → 候选题材/候选个股/候选模式
  → D+验证
  → 正式题材生命周期 / 个股档案 / 交易模式 / 错误库
  → 月度统计与系统进化
```

## 降级路径

WIKI 中不再符合准入标准的内容，应降级：

```text
正式 WIKI 页面
  → 信息来源索引
  → data/brain 候选
  → RAW 引用
  → 99-待删除审核
  → 归档或删除
```

删除必须人工确认，不允许自动批量删除。

## AI 执行要求

AI 写 WIKI 前必须先判断：

- 这是不是原始资料？
- 是否有证据路径？
- 是否会影响交易决策？
- 应进入哪个 L 层？
- 是否需要 D+验证？
- 是否只是资料卡，应该留在 RAW/索引？

如果不确定，默认不写正式 WIKI，先进入候选或审核区。
