# 竞价弱转强 RAW 与盘后验证规则

## 定位

竞价弱转强是 09:25 实时执行辅助，不是 Codex 盘前生成任务。

分工：

| 角色 | 责任 |
|---|---|
| Mac本机RAW入口 | 09:25 拉竞价数据、判断弱转强、写 RAW、必要时飞书提醒 |
| Codex | 盘后验收 RAW、做 D+验证、修正弱转强模式权重 |

## RAW 位置

```text
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.json
raw/04-市场数据/竞价弱转强/YYYY-MM-DD/0925-竞价弱转强.md
```

## 合格标准

合格的竞价弱转强必须同时具备：

1. 昨日确实有“弱”的事实。
2. 今日竞价确实出现超预期。
3. 竞价金额或竞价量能足够支撑，不是虚高。
4. 板块、同身位、核心票至少有一个维度确认。
5. 有明确买点、止损和骗炮风险。

不合格：

- 只写高开百分比。
- 只写“关注”。
- 没有昨日弱因。
- 没有竞价金额。
- 没有板块/同身位对比。
- 没有风险和止损。

## 操作分级

| 分级 | 含义 | 交易含义 |
|---|---|---|
| A可参与 | 昨日弱因清楚，今日竞价超预期，板块/同身位共振 | 可按计划参与，但必须有止损 |
| B观察 | 有转强迹象，但缺一个关键确认 | 开盘 3-5 分钟看承接 |
| C冲高卖或回避 | 竞价强但板块不跟，或持仓风险大于机会 | 不开新仓，持仓按卖点处理 |
| D骗炮风险 | 高开无量、无板块、风险锚恶化、昨日弱因没有修复 | 禁买 |

## Codex 盘后验证

盘后验收脚本：

```bash
python3 raw/07-系统脚本/codex_check_auction_weak_to_strong.py --date YYYY-MM-DD --write
```

验收输出：

```text
wiki/09-统计与进化/YYYY-MM-DD-竞价弱转强验收.md
```

验证维度：

- RAW 文件是否存在。
- 字段是否完整。
- 是否存在英文路径/英文字段。
- 是否有 A/B/C/D 分级。
- 是否写了可执行买点和止损。
- 是否写了骗炮风险。

后续 D+验证再看：

- D+0 是否高开低走。
- D+1 是否有溢价。
- D+1/D+3 是否强于板块。
- A 级候选是否真实可参与。
- D 级风险候选是否被市场证伪。
