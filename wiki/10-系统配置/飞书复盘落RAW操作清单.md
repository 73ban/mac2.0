# 飞书复盘落 RAW 操作清单

更新时间：2026-06-18

## 适用场景

你在飞书里先把交割单、截图、口述内容发给大鸟。

大鸟先按原样落 RAW，再由同一个执行体继续正式复盘。

正式复盘完成后，不得直接写入 WIKI，必须先把复盘全文通过飞书发给用户审核。

## 顺序

1. 先落交割单。
2. 再落口述补充。
3. 再归档关键截图。
4. 再进入正式复盘。
5. 正式复盘全文发飞书送审。
6. 用户确认后，再写入 WIKI。

## 交割单落库位置

```text
raw/01-交割单/YYYY-MM-DD/
```

## 口述补充位置

```text
raw/02-每日复盘/YYYY-MM-DD-口述补充.md
```

## 统一接收脚本

飞书桥接器或本机手工导入口述时，统一调用：

```bash
python3 raw/07-系统脚本/codex_ingest_feishu_message.py --date YYYY-MM-DD --type review --stdin
```

脚本会同时写入：

```text
raw/10-飞书交易沟通/YYYY/MM/DD/
raw/02-每日复盘/YYYY-MM-DD-飞书复盘RAW.md
raw/02-每日复盘/YYYY-MM-DD-飞书复盘RAW.jsonl
.system/feishu-notify-pending/
```

`YYYY-MM-DD-飞书复盘RAW.md` 是复盘口述主入口；`YYYY-MM-DD-口述补充.md` 只作为旧兼容路径或二次补充路径。

## 回执要求

每次收到口述后，只回四类信息：

```text
已按原话写入 RAW。
落点：...
识别：股票/动作/情绪分
待确认：...
```

不得在回执里抢先总结盘面、替用户判断对错、把“已入 RAW”说成“已完成复盘”。

## 截图位置

```text
raw/截图/盘中关键信号/YYYY-MM-DD/
raw/截图/学习内容/YYYY-MM-DD/
raw/截图/热榜/
```

## 禁止事项

1. 先分析，再落库。
2. 把你的原始口述改写成大鸟自己的观点。
3. 把“已入库”写成“已复盘完成”。
4. 未经用户审核，就把正式复盘直接写入 WIKI。
5. 把指代不清的口述写成确定结论。
6. 把用户提问误写成用户观点。
