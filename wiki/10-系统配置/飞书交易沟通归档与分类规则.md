# 飞书交易沟通归档与分类规则

更新时间：2026-07-03

## 核心结论

飞书里和交易有关的沟通，不等于“短线知识”。

默认先作为原始沟通记录保存，再按内容分流：

```text
原始聊天 -> raw/10-飞书交易沟通/YYYY/MM/DD/
校准判断 -> data/facts/feishu_calibration_events.jsonl
复盘口述 -> raw/02-每日复盘/
交割单/持仓 -> raw/01-交割单/
市场事实/消息材料 -> raw/04-市场数据/ 或 raw/05-研报新闻/
真正方法论 -> raw/09-短线知识/
```

本机统一入口：

```bash
python3 raw/07-系统脚本/codex_ingest_feishu_message.py --stdin
```

该入口只负责原文归档、低风险识别和歧义标记；深度分析、6维、作战室、D+验证由 Codex 主流程读取 RAW 后完成。

## 消息权重判断的标准响应

当用户问：

```text
这个消息重不重要？
这个算什么？
这条能不能刺激涨停？
这条是不是重大消息？
```

默认按四段回答：

```text
结论：重要/一般/不重要/反向/待验证
归类：政策/产业/订单/涨价/供需/公告/澄清/市场信号/情绪信号/个股材料
理由：2-3条
验证点：涨停、热榜跃迁、板块扩散、连板晋级、竞价强度
```

不能只回答“重要”或“不重要”。

## 原始记录和知识沉淀的区别

`raw/10-飞书交易沟通/` 保存原始对话，不代表已经成为规则。

`raw/09-短线知识/` 只保存已经有方法论价值的内容，例如：

1. 用户明确说“写成规则”。
2. 同类判断重复出现，并经过市场验证。
3. 对股票候选、消息强度、板块持续性、退潮防守有稳定指导意义。
4. Codex/Mac本机RAW入口复盘后确认可复用。

短线知识不能混入大量普通问答，否则会污染学习池。

## 消息重要性的五类

| 类别 | 含义 | 处理 |
|---|---|---|
| 重大产业消息 | 能推动整个行业上涨，具备持续催化 | 高优先级，进入作战室候选和 D+验证 |
| 个股独立消息 | 只影响单只票，如订单、公告、互动易确认 | 看纯度、位置、资金确认 |
| 市场信号 | 涨停全景、连板天梯、热榜跃迁、板块强度 | 不叫消息，但权重大 |
| 情绪信号 | 冰点、退潮、修复、一致、分歧 | 影响仓位和进攻/防守 |
| 噪音材料 | 没有映射票、没有资金验证、只是观点复述 | 降权，只留原文 |

## 写入原则

1. 用户原话必须保留，不改写。
2. 先落原始沟通，再做提炼。
3. 判断错了，要把用户纠正写成校准事件。
4. 是否升为短线知识，要看用户确认和后续市场验证。
5. “消息权重判断”不是短线知识本身，而是训练样本。
6. 用户口述和 Codex 理解必须分开保存；理解字段不能替代原话。
7. 指代不清、对象不清、买卖票不清时，先标 `待确认`，不能写入正式结论。
