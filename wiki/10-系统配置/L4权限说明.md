# L4权限说明

L4是四层嵌套决策体系的最底层——**执行控制层**。决定"买不买？仓位多少？什么时候卖？"

## 权限等级

| 等级 | 含义 | 操作规则 |
|:----:|------|----------|
| 🟢 | 可买 | 满足条件可建仓，按标准仓位 |
| 🟡 | 观察不买 | 可观察跟踪，但不新建仓。已有持仓按止损线管理 |
| 🔴 | 禁止建仓 | 不可新建仓，已有持仓尽快离场 |

## 权限判定维度

### 事实强度（facts_strength）
- **高**：已确认的事实，多源交叉验证（如公司公告、财报数据、海关数据）
- **中**：单源报道或分析师推测
- **低**：市场传闻、未验证消息

### 判定规则

| 条件 | L4权限 |
|------|:------:|
| 事实强度高 + 核心标的 + 估值合理 | 🟢 |
| 事实强度中 + 核心标的 | 🟡 |
| 事实强度高 + 核心标的 + 估值严重偏高 | 🟡 |
| 核心逻辑已被公司辟谣 | 🔴 |
| 收入占比极低（<1%）靠概念炒作 | 🟡~🔴 |
| 纯情绪票无基本面支撑 | 🔴 |

## 每张卡片必须标注

```yaml
l4_permission: 🟢  # 或 🟡 或 🔴
```

## 与其他层的关系

```
L1 市场环境（今天能不能买？）
  ↓  通过
L2 方向概念（买什么方向？）→ 每张概念卡有l4_permission
  ↓  通过
L3 个股档案（买哪只？）→ 每张个股卡有l4_permission
  ↓  通过
L4 执行控制（买不买？仓位多少？）
```

L4权限是**最后关卡**：即使L1-L3都通过，L4不通过也不能买。
