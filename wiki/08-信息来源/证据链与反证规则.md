# 证据链与反证规则

更新时间：2026-06-13

## 用途

这个文件防止 WIKI 被无效信息污染。任何进入作战室的观点，都必须有证据链和反证条件。

## 信息分级

| 等级 | 来源 | 用途 | 是否可作为买入依据 |
|---|---|---|---|
| S1 | 公告、交易所、公司正式披露、监管文件、官方数据 | 硬证据 | 可以，但仍需盘口确认 |
| S2 | 高质量研报、机构交流纪要、产业链调研、可信媒体 | 强参考 | 可以进入候选，需盘面确认 |
| S3 | 公众号、淘股吧高手、知识星球观点、复盘文章 | 认知参考 | 不能单独买入 |
| S4 | 小作文、截图、传闻、群聊消息 | 观察材料 | 禁止作为买入依据 |

## 作战室证据链格式

每条候选逻辑必须写：

```yaml
claim:
source:
truth_grade: S1/S2/S3/S4
evidence:
market_confirm:
counter_evidence:
recheck_time:
decision:
```

## 证据强度判断

| 证据组合 | 处理 |
|---|---|
| S1 + 盘口确认 + 热门榜共振 | 可进入主计划 |
| S2 + 板块走强 + 核心票确认 | 可进入候选 |
| S3 + 盘口确认 | 可观察或轻仓验证 |
| S3 无盘口 | 只沉淀，不交易 |
| S4 | 只放观察，不交易 |

## 反证条件

每条逻辑必须同时写反证。没有反证条件，说明逻辑不可交易。

常见反证：
1. 公告不及预期。
2. 研报逻辑被盘面否认。
3. 龙头不强，只有后排动。
4. 竞价高开低走。
5. 9:20 大单撤单。
6. 板块没有宽度。
7. 风险锚补跌。
8. 题材高潮后无修复。

## 小作文处理

S4 信息处理规则：
1. 不能进入主计划。
2. 不能提高仓位。
3. 只能设 `rumor_watch: true`。
4. 必须设置复核时间。
5. 如果盘面先走强，也只能按盘口和模式处理，不能把小作文当逻辑。

## 研报处理

研报不能直接等于买点。

研报进入作战室前必须回答：
1. 市场是否已经定价？
2. 个股是否处于高位？
3. 是否有短线资金认可？
4. 是否有板块共振？
5. 是否和当前主线一致？
6. 是否只是长期逻辑，短线无催化？

## 公众号/游资文章处理

公众号和游资文章主要用于学习模式和市场理解，不直接作为买入理由。

可提炼：
1. 模式。
2. 买卖点。
3. 市场环境判断。
4. 风控规则。
5. 情绪周期理解。

不可直接做：
1. 某人看好就买。
2. 复盘文章吹某票就追。
3. 用历史案例套当前市场。

## 淘股吧/实盘赛处理

淘股吧和实盘赛用于观察市场预期和高手行为。

可用：
1. 市场乐观/悲观程度。
2. 高手仓位变化。
3. 热门模式是否拥挤。
4. 核心票是否被反复讨论。

不可用：
1. 盲从某个实盘选手。
2. 事后复制持仓。
3. 不看自己作战室条件。

## 作战室硬门槛

候选票进入主计划至少满足：
1. 有 S1/S2 或明确盘面确认。
2. 有主模式。
3. 有竞价确认条件。
4. 有退出条件。
5. 有反证条件。

缺任何一项，只能放观察池。
